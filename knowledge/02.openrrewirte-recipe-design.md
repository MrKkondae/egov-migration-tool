# OpenRewrite Recipe 구성 방안

1순위 대상 라이브러리 분석 결과, OpenRewrite는 다음 3개 Recipe 영역으로 분리하여 관리하는 것을 원칙으로 한다.

```text
rules/
└── phase1-openrewrite/
    ├── 01.pom-dependency-migration.yml
    ├── java-import-migration.yml
    └── xml-config-migration.yml
```

## 1. pom-dependency-migration.yml

### 목적

Maven pom.xml 내 의존성 정보를 eGovFrame 4.3 기준으로 변경한다.

### 주요 작업

* groupId 변경
* artifactId 변경
* version 변경
* 제거 대상 라이브러리 삭제
* 신규 라이브러리 추가

### 적용 대상

| 구분        | 내용                                        |
| --------- | ----------------------------------------- |
| eGovFrame | egovframework.rte.* → org.egovframe.rte.* |
| Spring    | 3.2.9 → 5.3.37                            |
| Jackson   | 1.x → 2.x                                 |
| MyBatis   | 3.2.x → 3.5.x                             |
| OJDBC     | ojdbc14 → ojdbc8                          |
| Logging   | Log4j / SLF4J 버전 통일                       |
| JSTL      | standard.jar → taglibs-standard           |

### 특징

* 가장 안정적인 자동 변환 영역
* 소스 수정 영향 최소
* OpenRewrite 1차 적용 대상

---

## 2. java-import-migration.yml

### 목적

Java 소스 내 import 구문 및 클래스 참조를 변경한다.

### 주요 작업

#### eGovFrame Package 변경

```java
import egovframework.rte.psl.dataaccess.EgovAbstractMapper;
```

↓

```java
import org.egovframe.rte.psl.dataaccess.EgovAbstractMapper;
```

#### Jackson 변경

```java
import org.codehaus.jackson.map.ObjectMapper;
```

↓

```java
import com.fasterxml.jackson.databind.ObjectMapper;
```

#### Commons DBCP 변경

```java
org.apache.commons.dbcp.BasicDataSource
```

↓

```java
org.apache.commons.dbcp2.BasicDataSource
```

### 특징

* Java 소스 자동 변환 전용
* OpenRewrite 활용 효과가 가장 큰 영역
* 컴파일 검증으로 결과 확인 가능

---

## 3. xml-config-migration.yml

### 목적

Spring XML 설정 파일 및 Framework 설정 XML을 변경한다.

### 주요 작업

#### iBatis → MyBatis

```xml
<bean class="org.springframework.orm.ibatis.SqlMapClientFactoryBean">
```

↓

```xml
<bean class="org.mybatis.spring.SqlSessionFactoryBean">
```

#### Jackson Converter 변경

```xml
MappingJacksonHttpMessageConverter
```

↓

```xml
MappingJackson2HttpMessageConverter
```

#### Commons DBCP 변경

```xml
org.apache.commons.dbcp.BasicDataSource
```

↓

```xml
org.apache.commons.dbcp2.BasicDataSource
```

### 특징

* Spring Context XML 중심
* Java 소스보다 위험도가 높음
* 적용 후 기동 테스트 필요

---

## 적용 순서

```text
1. pom-dependency-migration.yml
          ↓
2. java-import-migration.yml
          ↓
3. xml-config-migration.yml
          ↓
4. Python Rule
          ↓
5. LLM 보정
```

현재 Library Matrix 기준으로 OpenRewrite 적용 범위는 다음과 같이 정의한다.

* pom.xml 의존성 변경
* Java Import 변경
* 단순 클래스명 변경
* XML 설정 변경

복잡한 API 변경, SQL 변환, Framework 구조 변경은 OpenRewrite 대상에서 제외하고 Python Rule 또는 LLM 보정 단계에서 처리한다.
