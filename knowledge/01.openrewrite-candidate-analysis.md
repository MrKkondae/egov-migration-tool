# OpenRewrite YML 후보 분석

Library Matrix 기준으로 **OpenRewrite YML 후보**는 아래 정도로 보면 됩니다. Matrix에는 `버전업`, `전환대상`, `검토` 기준이 정의되어 있고, OpenRewrite 자동변환 가능 여부도 보고서 양식의 검토 항목으로 잡혀 있습니다.  

## 1순위: OpenRewrite YML 작성 가능

|           No | 라이브러리                                                     | YML 유형                            |
| -----------: | --------------------------------------------------------- | --------------------------------- |
|        19~24 | `egovframework.rte.*-3.1.0` → `org.egovframe.rte.*:4.3.0` | 의존성 groupId/artifactId/version 변경 |
|        30~31 | Jackson 1.x → Jackson 2.x                                 | 의존성 + import 변경                   |
|    43~46, 55 | Log4j/SLF4J 계열                                            | 의존성 버전 통일, 충돌 제거                  |
|        48~49 | MyBatis / MyBatis-Spring                                  | 의존성 버전업                           |
|           50 | `ojdbc14` → `ojdbc8`                                      | 의존성 교체                            |
| 56~62, 65~68 | Spring 3.2.9 → Spring 5.3.37                              | Spring 의존성 일괄 버전업                 |
|           70 | `standard-1.1.2.jar` → `taglibs-standard-impl-1.2.5`      | 의존성 교체                            |

## 제외 또는 보류

`검토`, `확인`, `대상아님`, 공급사/솔루션성 라이브러리는 바로 YML로 만들기 어렵습니다.

예: `exportfile`, `jcaos-client-arccert`, `lib.dstspdfSig`, `lib.magic.tsaclient`, `Petra`, `ozenc_utf8`, `xdataset`, `xecure`, `OZ` 계열.

## 결론

우선 YML은 아래 4개 묶음으로 시작하는 게 좋습니다.

```text
1. egovframe-3.1-to-4.3.yml
2. spring-3.2-to-5.3.yml
3. logging-migration.yml
4. json-mybatis-common-libs.yml
```

그다음 `db-pool`, `ibatis-to-mybatis`, `quartz`, `pdf`는 별도 Phase2/Phase3 후보로 분리하는 게 안전합니다.
