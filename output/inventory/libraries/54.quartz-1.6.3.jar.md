## 2. 라이브러리 개요

### 주요 기능

* 작업(Job) 스케줄링
* Cron 기반 스케줄 실행
* Batch 작업 예약 실행
* 정기 업무 자동 실행
* 클러스터링 기반 스케줄 관리

### 사용 목적

정해진 시간 또는 주기에 따라 업무 프로그램을 자동 실행하기 위해 사용된다.

전자정부 프로젝트에서는 다음과 같은 업무에 자주 사용된다.

* 배치 작업 실행
* 통계 데이터 집계
* 데이터 정합성 점검
* 로그 정리
* 파일 정리 작업
* 야간 업무 처리

### 대표 사용 예시

```java
JobDetail job = new JobDetail(
    "sampleJob",
    Scheduler.DEFAULT_GROUP,
    SampleJob.class
);

CronTrigger trigger =
    new CronTrigger(
        "sampleTrigger",
        "0 0 * * * ?"
    );

scheduler.scheduleJob(job, trigger);
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

quartz-1.6.3은 매우 오래된 버전으로 현재 유지보수가 종료된 상태이다.

eGovFrame 4.3은 Spring Framework 5.3 기반 환경으로 운영되며, Quartz 1.x 계열은 최신 Spring 환경과의 호환성 및 보안 측면에서 업그레이드가 필요하다.

특히 Quartz 2.x에서는 API 구조가 변경되었으므로 전환 시 영향도 분석이 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 높음 |
| 설정파일    | 높음 |
| pom.xml | 높음 |
| 운영환경    | 중간 |

##### (16) 최종 정리

```text
Quartz 1.6.3 → 2.3.x 전환은 Java 소스 수정이 발생할 수 있다.

단순히 Job 인터페이스 또는 QuartzJobBean을 구현한 클래스는 대부분 유지 가능하지만,
Java 코드에서 JobDetail, CronTrigger, SimpleTrigger를 직접 생성하는 경우
Quartz 2.x의 Builder 기반 API로 변경해야 한다.

OpenRewrite는 Maven 의존성 변경, Spring XML Bean class 치환,
구버전 API 사용 위치 탐지에는 효과적이다.

다만 JobDataMap 처리, DB JobStore, Cluster 설정,
실제 스케줄 실행 여부는 자동변환 대상이 아니므로 수동 검토 및 실행 검증이 필요하다.
```
