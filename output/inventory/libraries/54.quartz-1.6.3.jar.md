## 2. 라이브러리 개요

### 주요 기능

* 작업(Job) 스케줄링
* Cron 기반 스케줄 실행
* Batch 작업 예약 실행
* 정기 업무 자동 실행
* 클러스터링 기반 스케줄 관리

### 사용 목적

정해진 시간 또는 주기에 따라 업무 프로그램을 자동 실행하기 위해 사용된다.

전자정부 프로젝트에서는 다음과 같은 업무에 자주 사용된다.

* 배치 작업 실행
* 통계 데이터 집계
* 데이터 정합성 점검
* 로그 정리
* 파일 정리 작업
* 야간 업무 처리

### 대표 사용 예시

```java
JobDetail job = new JobDetail(
    "sampleJob",
    Scheduler.DEFAULT_GROUP,
    SampleJob.class
);

CronTrigger trigger =
    new CronTrigger(
        "sampleTrigger",
        "0 0 * * * ?"
    );

scheduler.scheduleJob(job, trigger);
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

quartz-1.6.3은 매우 오래된 버전으로 현재 유지보수가 종료된 상태이다.

eGovFrame 4.3은 Spring Framework 5.3 기반 환경으로 운영되며, Quartz 1.x 계열은 최신 Spring 환경과의 호환성 및 보안 측면에서 업그레이드가 필요하다.

특히 Quartz 2.x에서는 API 구조가 변경되었으므로 전환 시 영향도 분석이 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 높음 |
| 설정파일    | 높음 |
| pom.xml | 높음 |
| 운영환경    | 중간 |

---

## 5. 전환 시 검토사항

### 변경 필요 사항

* Quartz 1.x → Quartz 2.x 업그레이드
* JobDetail API 변경 검토
* Trigger API 변경 검토
* SchedulerFactoryBean 설정 검토
* quartz.properties 검토
* Spring 연계 설정 검토
* 배치 프로그램 동작 검증

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | ----- |
| OpenRewrite | 일부 가능 |
| Python Rule | 가능    |
| LLM 보정      | 필요    |

### 예상 난이도

* 높음

* 일부 수정 필요
