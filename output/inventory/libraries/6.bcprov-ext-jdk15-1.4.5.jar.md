## 2. 라이브러리 개요

### 주요 기능

* 대칭키 암호화(AES, DES, SEED 등)
* 공개키 암호화(RSA, ECC 등)
* 전자서명(Signature)
* 인증서(X.509) 처리
* CMS/PKCS#7 처리
* PEM 파일 처리

### 사용 목적

Bouncy Castle은 Java 표준 암호화 라이브러리(JCE)를 확장하는 보안 Provider이다.

Java 기본 Provider에서 지원하지 않는 다양한 암호화 알고리즘과 인증서 처리 기능을 제공한다.

전자정부프레임워크 자체 의존성은 아니지만 공공기관 시스템에서 다음과 같은 목적으로 자주 사용된다.

* 전자서명
* 인증서 발급
* PKI 연계
* 암호화/복호화
* CMS 전자문서 처리

### 대표 사용 예시

```java id="1q0m7s"
import java.security.Security;
import org.bouncycastle.jce.provider.BouncyCastleProvider;

Security.addProvider(
    new BouncyCastleProvider());

Cipher cipher =
    Cipher.getInstance(
        "AES/CBC/PKCS7Padding",
        "BC");
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

전자정부프레임워크 4.3은 해당 라이브러리에 의존하지 않는다.

그러나 현재 사용 중인 버전(1.45)은 매우 오래된 버전이며 JDK17 환경과의 호환성 및 보안 측면에서 최신 버전 사용이 권장된다.

직접 사용하는 경우 버전 업그레이드 검토가 필요하다.

### 영향 범위

| 구분      | 영향          |
| ------- | ----------- |
| 소스      | 일부 영향 가능    |
| 설정파일    | 없음          |
| pom.xml | 변경 필요       |
| 운영환경    | JDK17 대응 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

#### 사용하지 않는 경우

```text id="c4o0rr"

```

#### 사용하는 경우

```text id="9rj4l2"
최신 버전 업그레이드 권장
```

권장 Maven Dependency

```xml id="j5f1uz"
<dependency>
    <groupId>org.bouncycastle</groupId>
    <artifactId>bcprov-ext-jdk18on</artifactId>
    <version>1.84</version>
</dependency>
```

PKIX, PEM, CMS 기능 사용 시

```xml id="9a6gpl"
<dependency>
    <groupId>org.bouncycastle</groupId>
    <artifactId>bcpkix-jdk18on</artifactId>
    <version>1.84</version>
</dependency>
```

추가 검토가 필요하다.
