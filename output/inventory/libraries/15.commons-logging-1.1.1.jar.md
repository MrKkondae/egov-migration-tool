## 2. 라이브러리 개요

### 주요 기능

* 다양한 Logging Framework 추상화
* Log4J, JUL(Java Util Logging), JDK Logging 연동
* 런타임 환경에 따라 자동 로그 구현체 선택
* 라이브러리와 애플리케이션 간 Logging 의존성 분리

### 사용 목적

Commons Logging(JCL, Jakarta Commons Logging)은 애플리케이션이 특정 로그 프레임워크에 종속되지 않도록 하기 위한 Logging Facade 라이브러리이다.

전자정부프레임워크 3.x, Spring Framework 3.x~4.x 시절에는 사실상 표준 Logging API로 사용되었다.

### 대표 사용 예시

```java
import org.apache.commons.logging.Log;
import org.apache.commons.logging.LogFactory;

public class UserService {

    private static final Log LOG =
        LogFactory.getLog(UserService.class);

    public void execute() {
        LOG.debug("Debug Message");
        LOG.info("Info Message");
        LOG.error("Error Message");
    }
}
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 제외

### 전환 사유

전자정부프레임워크 4.3은 Spring Framework 5.3 기반이며, Spring 내부에서도 Commons Logging API를 계속 사용한다.

따라서 commons-logging 자체는 전자정부프레임워크 전환 대상 라이브러리가 아니다.

다만 최근 Spring 5.x 환경에서는 실제 구현체로 Logback + SLF4J 조합을 사용하는 것이 일반적이며, Commons Logging은 Spring JCL(Spring-JCL)로 대체되어 동작한다.

### 영향 범위

| 구분      | 영향    |
| ------- | ----- |
| 소스      | 거의 없음 |
| 설정파일    | 없음    |
| pom.xml | 일부    |
| 운영환경    | 없음    |

## 5. 전환 시 검토사항

### 변경 필요 사항

* 직접 commons-logging.jar를 사용하는지 확인
* Spring-JCL이 이미 제공되는지 확인
* 중복 Logging 라이브러리 충돌 여부 확인
* Log4J 1.x 사용 시 Logback 또는 Log4J2 전환 검토

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | ----- |
| OpenRewrite | 가능    |
| Python Rule | 가능    |
| LLM 보정      | 불필요   |

### 예상 난이도

* 낮음

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
commons-logging 1.1.1
(2007년 출시)
```

### TO-BE 버전

```text
별도 유지 불필요

또는

Spring-JCL
(Spring Framework 5.3.x 포함)
```

### 주요 변경사항

* Spring 5.x에서 Spring-JCL 제공
* Commons Logging API 유지
* 실제 Logging 구현은 Logback 사용
* 별도 commons-logging.jar 제거 가능

### 소스 수정 여부

* 없음

직접 사용하는 코드도 대부분 수정 없이 동작한다.

예)

```java
private static final Log log =
    LogFactory.getLog(getClass());
```

위 코드는 그대로 사용 가능하다.

### 비고

* 전자정부프레임워크 4.3 환경에서는 commons-logging-1.1.1.jar를 별도로 유지할 필요가 없는 경우가 많다.
* Spring Framework 5.3이 제공하는 Spring-JCL이 동일 역할을 수행한다.
* AS-IS 소스에서 `org.apache.commons.logging.Log`를 직접 사용하더라도 대부분 수정 없이 동작한다.
* Migration Matrix 기준으로는 **"전환 대상 제외 / 제거 검토"** 분류가 적절하다.
* 다만 프로젝트에서 Log4J 1.x를 사용하는 경우에는 Logging 체계(Logback 또는 Log4J2) 전체를 함께 검토하는 것이 좋다.
