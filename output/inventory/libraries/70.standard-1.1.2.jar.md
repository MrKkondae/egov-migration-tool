## 2. 라이브러리 개요

### 주요 기능

* JSP JSTL 태그 제공
* 조건문 처리
* 반복문 처리
* 문자열 처리
* 국제화(i18n) 처리
* URL 처리

### 사용 목적

`standard-1.1.2.jar`는 JSP 화면에서 JSTL(JSP Standard Tag Library)을 사용하기 위한 구현체 라이브러리이다.

전자정부프레임워크 3.x 기반 시스템에서는 JSP 화면에서 다음과 같은 태그를 사용하기 위해 활용된다.

* `<c:if>`
* `<c:forEach>`
* `<c:choose>`
* `<fmt:message>`
* `<fmt:formatDate>`
* `<fn:length>`

### 대표 사용 예시

```jsp id="j1n8kc"
<c:if test="${user != null}">
    ${user.userNm}
</c:if>
```

```jsp id="8phf2g"
<c:forEach items="${userList}" var="user">
    ${user.userNm}
</c:forEach>
```

```jsp id="2qjv0x"
<fmt:formatDate
    value="${today}"
    pattern="yyyy-MM-dd"/>
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`standard-1.1.2.jar`는 JSTL 1.1 계열의 오래된 구현체이다.

eGovFrame 4.3은 Servlet 4.0 기반(Tomcat 9 기준) 환경을 사용하며, 일반적으로 JSTL 1.2 이상 또는 Maven 기반 JSTL 라이브러리 사용을 권장한다.

특히 기존 프로젝트에서는 다음과 같이 구성된 경우가 많다.

```text
jstl-1.1.2.jar
standard-1.1.2.jar
```

eGovFrame 4.3 전환 시에는 JSTL 라이브러리 정비 및 중복 라이브러리 제거가 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 낮음 |
| 설정파일    | 낮음 |
| pom.xml | 중간 |
| 운영환경    | 중간 |

##### (16) 최종 정리

```text
standard 1.1.2 → taglibs-standard-impl 1.2.5 전환은 JSP/JSTL 태그 소스 수정이 아니라 JSTL 구현체 교체가 핵심이다.

일반적인 <c:if>, <c:forEach>, <fmt:message>, <fmt:formatDate>, <fn:*> 사용 코드는 대부분 그대로 유지 가능하다.

OpenRewrite를 통해 Maven 의존성 변경, 구버전 Jar 제거 목록화,
taglibs-standard-impl/spec 의존성 추가, JSTL 중복 Jar 탐지는 자동화할 수 있다.

다만 JSTL Jar 중복 제거, WAS 제공 라이브러리와의 충돌 여부,
JSP 컴파일 및 전체 화면 호출 검증은 수동 검토 대상으로 관리해야 한다.

전환 후에는 JSP 컴파일 오류, taglib URI 인식 오류,
JSTL 태그 출력 결과, 한글 메시지 출력 여부를 반드시 검증해야 한다.
```
