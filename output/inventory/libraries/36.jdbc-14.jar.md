## 2. 라이브러리 개요

### 주요 기능

* Java 애플리케이션과 Oracle Database 간 JDBC 연결 제공
* `DriverManager` 또는 `DataSource` 기반 DB 접속 지원
* SQL 실행, PreparedStatement, CallableStatement, ResultSet 처리
* Spring JDBC, MyBatis, iBATIS, eGovFrame DataAccess 계층의 하위 DB 드라이버 역할 수행

### 사용 목적

`jdbc-14.jar`는 파일명만으로는 정확한 제조사와 버전을 확정하기 어렵다. 다만 Matrix에서는 `Persistence / Oracle JDBC 추정` 항목으로 분류되어 있으며, 50번 `ojdbc14-10.2.0.1.0.jar`와 중복 가능성이 큰 항목으로 정리되어 있다.

따라서 본 보고서에서는 `jdbc-14.jar`를 Oracle JDBC Driver 계열 또는 동일 목적의 DB 접속 드라이버로 추정하고, eGovFrame 4.3 전환 시 `ojdbc8` 단일 적용으로 정리하는 것을 기본 방안으로 한다.

Oracle JDBC 드라이버는 업무 소스에서 직접 사용하는 기능 라이브러리라기보다 DB 접속을 위한 런타임 필수 라이브러리이다. 드라이버 버전이 맞지 않거나 중복 배치되면 애플리케이션 기동 오류, DB 연결 실패, 트랜잭션 오류, 문자셋/날짜 처리 차이, 운영 환경별 동작 차이가 발생할 수 있다.

### 대표 사용 예시

```java
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;

public class JdbcSample {
    public void select() throws Exception {
        Class.forName("oracle.jdbc.OracleDriver");

        try (Connection conn = DriverManager.getConnection(
                "jdbc:oracle:thin:@//db-host:1521/service", "user", "password");
             PreparedStatement ps = conn.prepareStatement("SELECT 1 FROM DUAL");
             ResultSet rs = ps.executeQuery()) {

            while (rs.next()) {
                System.out.println(rs.getInt(1));
            }
        }
    }
}
```

실제 eGovFrame/Spring 기반 프로젝트에서는 위와 같이 직접 `DriverManager`를 사용하는 경우보다 `DataSource`, `SqlMapClient`, `SqlSessionFactory`, `JdbcTemplate`, MyBatis Mapper를 통해 간접 사용되는 경우가 일반적이다.

## 3. AS-IS 사용 현황

### 프로젝트 내 사용 여부

* 확인 필요
* `jdbc-14.jar` 파일명만으로는 정확한 제품명/버전 확정 불가
* Oracle JDBC Driver 또는 `ojdbc14`와 동일/유사 드라이버일 가능성 있음
* 50번 `ojdbc14-10.2.0.1.0.jar`와 중복 가능성 높음

### 확인 방법

#### JAR 파일 및 중복 드라이버 확인

```bash
find . -name "*jdbc*.jar"
find . -name "*ojdbc*.jar"
find . -name "jdbc-14.jar"
find . -name "ojdbc14*.jar"
find . -name "ojdbc6*.jar"
find . -name "ojdbc8*.jar"
```

#### JAR Manifest 및 클래스 확인

```bash
jar tf jdbc-14.jar | head -50
jar tf jdbc-14.jar | grep -E "oracle/jdbc|oracle/sql|META-INF/MANIFEST.MF"

unzip -p jdbc-14.jar META-INF/MANIFEST.MF
```

#### Maven/Gradle 의존성 확인

```bash
mvn dependency:tree | grep -i "ojdbc\|oracle\|jdbc"

grep -R "ojdbc" pom.xml **/pom.xml
grep -R "com.oracle" pom.xml **/pom.xml
grep -R "oracle.jdbc" pom.xml **/pom.xml
```

#### 설정 파일 확인

```bash
grep -R "oracle.jdbc.driver.OracleDriver" .
grep -R "oracle.jdbc.OracleDriver" .
grep -R "jdbc:oracle" .
grep -R "driverClassName" ./src ./config .
grep -R "validationQuery" ./src ./config .
```

#### 소스 사용처 확인

```bash
grep -R "DriverManager.getConnection" ./src
grep -R "Class.forName" ./src | grep -i oracle
grep -R "oracle.sql" ./src
grep -R "oracle.jdbc" ./src
grep -R "CLOB\|BLOB\|TIMESTAMP\|STRUCT\|ARRAY" ./src
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상
* 대체 / 검토 대상

### 전환 사유

Matrix에서는 `jdbc-14.jar`를 `ojdbc8` 단일 적용 대상으로 분류하고 있다. 또한 `ojdbc14-10.2.0.1.0.jar`와 중복 가능성이 크므로 JAR Manifest 확인 후 Oracle JDBC 드라이버를 하나로 정리해야 한다고 명시되어 있다.

전자정부프레임워크 4.3은 JDK 1.8 이상을 기준으로 하므로, 과거 JDK 1.4 시대의 `ojdbc14` 계열 드라이버 또는 그와 유사한 `jdbc-14.jar`를 유지하는 것은 적절하지 않다. Oracle JDBC 드라이버는 JDK, Oracle DB 버전, WAS, 커넥션 풀 설정과 직접 연관되므로, `ojdbc8` 계열로 표준화하고 중복 드라이버를 제거하는 것이 안정적이다.

다만 운영 WAS에서 Oracle JDBC 드라이버를 공통 lib로 제공하고 있을 수 있으므로, 애플리케이션 lib에 무조건 포함하기보다 운영 표준 배치 위치를 먼저 확인해야 한다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 대부분 없음. 단, `oracle.sql.*`, `oracle.jdbc.*` 전용 API 직접 사용 시 일부 수정 가능 |
| 설정파일    | `driverClassName`, JDBC URL, validationQuery, DataSource, WAS JDBC Provider 설정 확인 필요 |
| pom.xml | `jdbc-14.jar`, `ojdbc14` 제거 및 `ojdbc8` 단일 의존성 적용 필요 |
| 운영환경    | WAS 공통 lib, 애플리케이션 lib, 배치 classpath 중복 제거 및 DB 연결 테스트 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `jdbc-14.jar`의 실제 정체 확인
  * `META-INF/MANIFEST.MF` 확인
  * 포함 클래스 확인
  * 공급사/라이선스/버전 확인
* `ojdbc14-10.2.0.1.0.jar`와 중복 여부 확인
* Oracle JDBC 드라이버는 `ojdbc8` 단일 적용으로 정리
* WAS 공통 lib와 애플리케이션 `WEB-INF/lib` 중복 배치 제거
* 운영 배치 classpath 내 구버전 JDBC 드라이버 제거
* DB 버전, JDK 8, WAS 버전과 호환되는 `ojdbc8` 버전 선정
* 다음 설정 항목 회귀 검증
  * `driverClassName`
  * JDBC URL
  * DB 접속 계정/권한
  * 커넥션 풀 validation query
  * 트랜잭션 처리
  * CLOB/BLOB 처리
  * 날짜/시간/TIMESTAMP 처리
  * 문자셋/한글 처리
  * 대량 조회/배치 성능

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 제한적 |
| Python Rule | 가능 |
| LLM 보정      | 일부 필요 |

### 예상 난이도

* 낮음 ~ 중간

단순 드라이버 교체만 보면 난이도는 낮다. 그러나 DB 접속은 전체 업무 기능의 기반이므로, 운영 WAS 공통 lib와 애플리케이션 lib 중복, DB 버전, 커넥션 풀 설정, 배치 classpath까지 포함하면 실제 검증 난이도는 중간 수준으로 본다.

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
jdbc-14.jar
버전 확인 필요
Oracle JDBC Driver 또는 ojdbc14 계열 추정
```

### TO-BE 버전

```text
com.oracle.database.jdbc:ojdbc8
또는 프로젝트 표준 Oracle JDBC Driver ojdbc8 검증 버전
```

### 주요 변경사항

* 비표준 파일명 `jdbc-14.jar` 제거
* Oracle JDBC 드라이버를 `ojdbc8` 단일 버전으로 표준화
* JDK 8 기반 eGovFrame 4.3 환경에 맞는 JDBC 4.x 계열 드라이버 적용
* `ojdbc14`, `jdbc-14`, `ojdbc6`, `ojdbc8` 등 복수 드라이버 동시 존재 방지
* Maven 의존성 또는 WAS 공통 lib 중 하나의 관리 방식으로 통일
* `oracle.jdbc.driver.OracleDriver`보다 `oracle.jdbc.OracleDriver` 사용 여부 확인

### 소스 수정 여부

* 대부분 없음
* JDBC 표준 API만 사용하면 소스 수정 없음
* Oracle 전용 API를 직접 사용하는 경우 일부 수정 필요 가능

#### 수정 가능성이 있는 코드 예시

```java
// 구버전/전용 API 사용 여부 확인 대상
import oracle.sql.CLOB;
import oracle.sql.BLOB;
import oracle.jdbc.driver.OracleDriver;
```

```java
// 권장 확인 방향
import oracle.jdbc.OracleDriver;
```

## 7. 위험요소

### 보안

* `jdbc-14.jar`는 정확한 버전 및 공급사를 확인할 수 없어 보안 패치 수준 확인이 어렵다.
* Oracle JDBC 드라이버는 DB 인증정보, 네트워크 통신, SQL 실행과 직접 관련되므로 구버전 유지 시 보안 점검에서 지적될 가능성이 있다.
* 비표준 파일명으로 관리된 JAR는 SBOM, 오픈소스 점검, 취약점 점검 도구에서 식별이 누락될 수 있다.

### 운영

* `jdbc-14.jar`와 `ojdbc14-10.2.0.1.0.jar`가 함께 존재하면 클래스 로딩 순서에 따라 의도하지 않은 드라이버가 로딩될 수 있다.
* WAS 공통 lib와 애플리케이션 lib에 서로 다른 Oracle JDBC 드라이버가 동시에 존재하면 운영/검증 환경 간 동작 차이가 발생할 수 있다.
* JDBC 드라이버 교체 후 커넥션 풀 validation, 트랜잭션, CLOB/BLOB, 날짜/시간, 문자셋 처리 회귀 테스트가 필요하다.
* Oracle DB 버전과 JDBC 드라이버 버전의 조합을 DBA/WAS 담당자와 확인해야 한다.

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 8           | `ojdbc8` 계열 적용 권장 |
| Spring 5.3      | `DataSource`, 트랜잭션, MyBatis/iBATIS 연계 확인 필요 |
| eGovFrame 4.3   | JDK 8 기준 드라이버 표준화 필요 |
| Oracle DB       | DB 버전에 맞는 JDBC 드라이버 버전 선정 필요 |
| WAS             | 공통 lib 제공 여부 및 classloader 정책 확인 필요 |
| MyBatis/iBATIS  | SQL Mapper 자체보다 하위 JDBC 드라이버 교체 영향 확인 필요 |

## 9. 최종 판정

```text

조치방안 :
- jdbc-14.jar의 Manifest 및 포함 클래스를 확인한다.
- Oracle JDBC Driver로 확인될 경우 ojdbc8 단일 적용으로 대체한다.
- ojdbc14-10.2.0.1.0.jar와 중복 여부를 확인하고 Oracle JDBC 드라이버는 하나만 유지한다.
- WAS 공통 lib와 애플리케이션 WEB-INF/lib 중복 배치를 제거한다.
- DB 연결, 트랜잭션, CLOB/BLOB, 날짜/시간, 문자셋, 배치 처리 회귀 테스트를 수행한다.

자동변환 :
- 부분 가능
- JAR 검색, 중복 제거 후보 식별, 설정 파일 검색은 자동화 가능
- 실제 드라이버 버전 선정과 운영환경 classpath 정리는 수동 검토 필요

소스수정 :
- 대부분 없음
- Oracle 전용 API 직접 사용 시 일부 수정 필요

위험도 :
- 중간
```

### 비고

* `jdbc-14.jar`는 파일명만으로 공식 Oracle JDBC 드라이버인지 확정할 수 없으므로, 분석 단계에서 반드시 Manifest와 포함 클래스를 확인해야 한다.
* 50번 `ojdbc14-10.2.0.1.0.jar`와 함께 검토해야 하며, 최종적으로는 Oracle JDBC 드라이버를 `ojdbc8` 하나로 정리하는 것이 바람직하다.
* DB 접속 드라이버는 업무 기능 전체에 영향을 주므로 단위 테스트뿐 아니라 주요 업무 조회/등록/수정/삭제, 배치, 대량 조회, 트랜잭션 롤백 테스트까지 포함해야 한다.
* 폐쇄망 반입 시에는 Oracle JDBC 드라이버 라이선스, 반입 파일명, 체크섬, 배포 위치, 운영 WAS 적용 절차를 산출물로 남겨야 한다.
* Matrix 파일에는 병합 충돌 흔적이 있으므로 최종 산출물 반영 전 36번 항목의 기준 목록을 확정해야 한다.
