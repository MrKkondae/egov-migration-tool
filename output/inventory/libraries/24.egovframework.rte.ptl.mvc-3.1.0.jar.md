## 2. 라이브러리 개요

### 주요 기능

* Spring MVC 기반 전자정부 공통 MVC 기능 제공
* 파일 다운로드 View 제공
* Ajax 처리 지원
* 공통 Exception 처리
* Validation 연계 기능
* View Resolver 확장 기능
* Pagination Tag 지원
* Controller 공통 기능 제공

### 사용 목적

전자정부프레임워크 3.x에서 Controller 계층 및 화면 처리 기능을 제공하는 Runtime 라이브러리이다.

Spring MVC 위에서 동작하며 다음과 같은 기능을 제공한다.

* EgovBindAdvice
* EgovXssChecker
* EgovPaginationRenderer
* EgovFileDownloadView
* Exception Handler
* Ajax 지원 기능

### 대표 사용 예시

```java
@Controller
public class SampleController {

    @RequestMapping("/sample.do")
    public String sample() {
        return "sample/sampleList";
    }
}
```

파일 다운로드 View 사용 예시

```java
return new ModelAndView("downloadView", "downloadFile", file);
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

전자정부프레임워크 4.3에서는 패키지명이 변경되었다.

AS-IS

```java
import egovframework.rte.ptl.mvc.tags.ui.pagination.PaginationInfo;
```

TO-BE

```java
import org.egovframe.rte.ptl.mvc.tags.ui.pagination.PaginationInfo;
```

따라서 전체 import 변경이 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 높음 |
| 설정파일    | 중간 |
| pom.xml | 높음 |
| 운영환경    | 낮음 |

## 5. 전환 시 검토사항

### 변경 필요 사항

#### 1. 패키지명 변경

```text
egovframework.*
↓
org.egovframe.*
```

#### 2. Maven Dependency 변경

AS-IS

```xml
<dependency>
    <groupId>egovframework.rte</groupId>
    <artifactId>egovframework.rte.ptl.mvc</artifactId>
    <version>3.1.0</version>
</dependency>
```

TO-BE

```xml
<dependency>
    <groupId>org.egovframe.rte</groupId>
    <artifactId>org.egovframe.rte.ptl.mvc</artifactId>
    <version>4.3.0</version>
</dependency>
```

#### 3. Spring MVC 설정 검토

* View Resolver
* Exception Resolver
* Multipart Resolver
* Message Converter

설정 차이점 확인 필요

#### 4. JSP Taglib 확인

기존 JSP에서 사용하는 Pagination Tag 경로 점검 필요

## 7. 위험요소

### 보안

직접적인 보안 이슈는 없음

다만 3.1 기반 Spring MVC는 현재 유지보수 종료 버전 기반으로 운영 위험 존재

### 운영

파일 다운로드 기능을 사용하는 경우

```java
EgovFileDownloadView
```

설정 변경 여부 확인 필요

### 호환성

| 항목        | 영향              |
| --------- | --------------- |
| JDK       | Java 8 이상 권장    |
| Spring    | Spring 5.3.x 적용 |
| eGovFrame | 4.3 Runtime 사용  |

### 비고

* eGovFrame 4.3에서도 동일 기능이 계속 제공되므로 기능 자체의 재개발은 필요하지 않다.
* 대부분의 변경은 패키지명 변경(`egovframework → org.egovframe`) 수준이다.
* Pagination, FileDownloadView, Exception Resolver를 사용하는 경우 실제 사용 소스를 확인하여 설정 호환성을 검증하는 것이 바람직하다.
* 전환 프로젝트 기준으로는 **전형적인 자동변환 대상 라이브러리**로 분류할 수 있다.
