## 2. 라이브러리 개요

### 주요 기능

* JSP 화면에서 조건문, 반복문, 변수 처리 등 표준 태그 제공
* 국제화 메시지, 날짜/숫자 포맷 처리 지원
* URL 처리, XML 처리 등 JSP 기반 화면 개발 보조

### 사용 목적

JSTL은 JSP 화면에서 Java 코드를 직접 작성하지 않고 표준 태그를 이용하여 화면 로직을 처리하기 위한 라이브러리이다.
전자정부프레임워크 기반 JSP 화면에서는 `<c:if>`, `<c:forEach>`, `<fmt:message>`, `<fmt:formatDate>` 등으로 자주 사용된다.

본 항목은 eGovFrame 4.3 전환 시 `jakarta.*`로 일괄 전환하는 대상이 아니라, 기본적으로 `javax.*` 기반 JSTL 1.2 유지 또는 WAS 제공 라이브러리와의 중복 여부를 검토하는 대상이다.

### 대표 사용 예시

```jsp
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<%@ taglib prefix="fmt" uri="http://java.sun.com/jsp/jstl/fmt" %>

<c:forEach var="item" items="${list}">
    <c:out value="${item.name}" />
</c:forEach>

<fmt:formatDate value="${now}" pattern="yyyy-MM-dd" />
```

## 3. AS-IS 사용 현황

### 프로젝트 내 사용 여부

* 사용 가능성 높음
* JSP 기반 화면에서 직접 사용되는 경우가 많음
* 실제 사용 여부 및 WAS 제공 여부 확인 필요

### 확인 방법

JSP taglib 선언 확인

```bash
grep -R "java.sun.com/jsp/jstl" src webapp WebContent
```

JSTL prefix 사용 확인

```bash
grep -R "<c:" src webapp WebContent
grep -R "<fmt:" src webapp WebContent
grep -R "<fn:" src webapp WebContent
grep -R "<x:" src webapp WebContent
```

라이브러리 중복 확인

```bash
find . -name "*jstl*.jar" -o -name "*standard*.jar" -o -name "*taglibs*.jar"
```

Maven 의존성 확인

```bash
mvn dependency:tree -Dincludes=javax.servlet:jstl
mvn dependency:tree | grep -i "jstl\|taglibs\|standard"
```

### 예상 사용 위치

* JSP 화면
* 공통 JSP include 파일
* Tiles/JSP 레이아웃 파일
* 공통 태그 라이브러리 설정
* 메시지/다국어 처리 화면

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 유지 / 검토

### 전환 사유

`jstl-1.2.jar`는 JSP 화면에서 표준 태그를 제공하는 라이브러리로, eGovFrame 4.3 전환 시 반드시 제거하거나 `jakarta.*` 계열로 변경해야 하는 항목은 아니다.

Matrix 기준으로 eGovFrame 4.3 기본 방향은 `javax.*` 유지이며, Tomcat 10 또는 Servlet 5 이상 환경이 아니라면 `jakarta.*`로 일괄 전환하지 않는 것으로 정리한다.

다만 WAS가 JSTL을 기본 제공하거나, `standard.jar`, `taglibs-standard-*.jar`, `javax.servlet:jstl`이 중복 배치되어 있으면 JSP 태그 로딩 오류가 발생할 수 있으므로 배포 구조 검토가 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | JSP taglib URI 및 JSTL 태그 사용 영역 영향 가능 |
| 설정파일    | web.xml, JSP include, Tiles 설정에서 간접 영향 가능 |
| pom.xml | Maven 전환 시 `javax.servlet:jstl:1.2` 또는 WAS 제공 기준으로 정리 필요 |
| 운영환경    | WAS 제공 JSTL과 애플리케이션 lib 중복 여부 확인 필요 |

---

## 5. 전환 시 검토사항

### 변경 필요 사항

* JSP 화면에서 JSTL 사용 여부 확인
* `jstl-1.2.jar`, `standard.jar`, `taglibs-standard-*.jar` 중복 여부 확인
* WAS 공통 lib와 애플리케이션 `WEB-INF/lib` 중복 배치 여부 확인
* Tomcat 9 이하 또는 Servlet 3.1 기반이면 `javax.servlet:jstl:1.2` 유지 검토
* Tomcat 10 이상 또는 Servlet 5 이상 전환 시에는 `jakarta.servlet.jsp.jstl` 계열 별도 전환 검토
* JSP 화면 회귀 테스트 수행

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 불필요 / 제한적 |
| Python Rule | 가능 |
| LLM 보정      | 필요 |

### 예상 난이도

* 낮음 ~ 중간

JSTL 1.2를 유지하는 경우 난이도는 낮다. 다만 Servlet 5 이상 또는 Tomcat 10 이상으로 전환하면서 `jakarta.*` 전환이 함께 수행되는 경우 JSP, taglib, 의존성 전반의 확인이 필요하므로 난이도는 중간 이상으로 상승한다.

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
jstl-1.2.jar
```

### TO-BE 버전

```text
javax.servlet:jstl:1.2
또는 WAS 제공 JSTL 사용
```

Tomcat 10 / Servlet 5 이상 전환 시 별도 후보는 다음과 같다.

```text
jakarta.servlet.jsp.jstl:jakarta.servlet.jsp.jstl-api
org.glassfish.web:jakarta.servlet.jsp.jstl
```

단, eGovFrame 4.3 기본 전환 범위에서는 `jakarta.*` 전환을 기본값으로 두지 않는다.

### 주요 변경사항

* eGovFrame 4.3은 Spring 5.3.x, JDK 1.8, Servlet 3.1 기반이므로 기본적으로 `javax.*` 기반 유지가 적절하다.
* JSP taglib URI는 기존 `http://java.sun.com/jsp/jstl/*` 형태가 그대로 사용될 가능성이 높다.
* WAS 제공 라이브러리와 애플리케이션 내 JSTL JAR이 중복되면 태그 클래스 충돌 또는 JSP 컴파일 오류가 발생할 수 있다.
* Tomcat 10 이상은 Servlet/JSP API 패키지가 `javax.*`에서 `jakarta.*`로 변경되므로 별도 전환 과제로 분리해야 한다.

### 소스 수정 여부

* eGovFrame 4.3 / Servlet 3.1 기준 유지 시: 대부분 없음
* 중복 JAR 정리 시: 소스 수정 없음, 배포 구성 수정 필요
* Jakarta 전환 시: JSP 및 의존성 일부 수정 필요

## 7. 위험요소

### 보안

* JSTL 자체보다 JSP 화면 출력 시 XSS 방어 여부가 중요하다.
* 사용자 입력값 출력 시 `<c:out>` 사용 여부와 escape 처리 기준 확인 필요
* 오래된 JSP 화면에서 스크립틀릿, EL 직접 출력, 미검증 HTML 출력이 함께 사용되는지 확인 필요

### 운영

* WAS 공통 lib와 애플리케이션 lib의 JSTL 중복 배치 주의
* `jstl.jar`와 `standard.jar` 조합, `javax.servlet:jstl:1.2`, `taglibs-standard-*`가 혼재되면 JSP 태그 로딩 오류 가능
* JSP 컴파일 캐시 문제로 배포 후 기존 캐시 삭제가 필요할 수 있음

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 1.8         | 영향 낮음 |
| Spring 5.3      | 직접 영향 낮음 |
| eGovFrame 4.3   | `javax.*` 기반 유지 가능 |
| Servlet 3.1     | JSTL 1.2 유지 가능 |
| Tomcat 10 이상  | `jakarta.*` 전환 필요, 별도 과제 |
| JSP 화면        | taglib 선언 및 태그 사용 회귀 테스트 필요 |
