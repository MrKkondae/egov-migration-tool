## 2. 라이브러리 개요

### 주요 기능

* 템플릿 기반 문자열 생성
* HTML 생성
* XML 생성
* 코드 생성(Code Generation)
* SQL 생성
* 문서 생성

### 사용 목적

`stringtemplate-3.2.1.jar`는 ANTLR 프로젝트에서 제공하는 템플릿 엔진 라이브러리이다.

주로 다음과 같은 기능에 사용된다.

* 코드 자동 생성
* SQL 자동 생성
* HTML 생성
* XML 생성
* 보고서 생성
* 템플릿 기반 출력 처리

전자정부프레임워크 자체 기능과는 직접적인 관련이 없으며, 특정 솔루션 또는 개발도구에서 내부적으로 사용하는 경우가 많다.

### 대표 사용 예시

```java id="6r0s3q"
StringTemplate st =
    new StringTemplate(
        "Hello $name$"
    );

st.setAttribute(
    "name",
    "eGov"
);

String result =
    st.toString();
```

결과

```text id="qg4r8n"
Hello eGov
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 검토 대상

### 전환 사유

`stringtemplate-3.2.1.jar`는 Spring Framework 또는 eGovFrame Runtime 구성 라이브러리가 아니다.

따라서 eGovFrame 3.x → 4.3 전환 시 직접적인 변경 대상은 아니다.

다만 동일 프로젝트에 아래 라이브러리가 함께 존재하는 경우가 있다.

```text id="7t8ewm"
antlr-3.x.jar
stringtemplate-3.x.jar
ST4-4.x.jar
```

이 경우 ANTLR 또는 코드 생성 도구가 사용하는 내부 의존성일 가능성이 있으므로 제거 여부를 신중하게 검토해야 한다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 낮음 |
| 설정파일    | 없음 |
| pom.xml | 낮음 |
| 운영환경    | 낮음 |

### 비고

* eGovFrame 4.3 전환과 직접 관련이 없는 라이브러리이다.
* Spring Framework, eGovFrame Runtime, MyBatis, JSTL 등 핵심 전환 대상과는 무관하다.
* 실제 프로젝트에서는 대부분 직접 사용하는 경우보다 ANTLR 의존성으로 포함되는 경우가 많다.
* `stringtemplate-3.2.1.jar`와 `ST4-4.0.7.jar`가 동시에 존재하는 경우 각각 다른 버전 계열이므로 사용 여부를 확인해야 한다.
* 사용자의 eGovFrame 3.x → 4.3 전환 프로젝트 기준으로는 **"유지 검토 대상"** 으로 분류하는 것이 적절하다.
* 우선순위는 Spring, MyBatis, JSTL, Validation 계열보다 매우 낮다.
