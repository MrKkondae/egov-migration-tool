## 2. 라이브러리 개요

### 주요 기능

* SSH2 프로토콜 기반 원격 서버 접속
* SFTP 파일 업로드 / 다운로드
* 원격 명령 실행, 포트 포워딩, 키 기반 인증
* 배치 또는 연계 시스템의 파일 송수신 처리

### 사용 목적

JSch는 Java에서 SSH2 기반 통신을 수행하기 위한 라이브러리이다. 업무 시스템에서는 주로 외부 기관, 연계 서버, 배치 서버, 파일 서버와 SFTP 방식으로 파일을 송수신하거나 원격 명령을 실행할 때 사용된다.

`jsch-0.1.54.jar`는 구 JCraft 계열 JSch 라이브러리이며, 전자정부프레임워크 4.3 전환 시에는 보안 알고리즘 호환성, SFTP 서버 정책, JDK 8 호환성, 운영 연계 영향도를 함께 검토해야 한다.

### 대표 사용 예시

```java
import com.jcraft.jsch.ChannelSftp;
import com.jcraft.jsch.JSch;
import com.jcraft.jsch.Session;

public class SftpSample {
    public void upload() throws Exception {
        JSch jsch = new JSch();
        Session session = jsch.getSession("user", "sftp.example.com", 22);
        session.setPassword("password");
        session.setConfig("StrictHostKeyChecking", "no");
        session.connect();

        ChannelSftp channel = (ChannelSftp) session.openChannel("sftp");
        channel.connect();
        channel.put("/local/sample.txt", "/remote/sample.txt");

        channel.disconnect();
        session.disconnect();
    }
}
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`jsch-0.1.54.jar`는 SSH/SFTP 통신을 담당하는 네트워크 보안 라이브러리이다. 원본 JCraft JSch 계열은 유지보수가 제한적이므로, eGovFrame 4.3 전환 시 보안 알고리즘 호환성과 운영 연계 안정성을 위해 유지 여부를 재검토해야 한다.

Matrix 기준 TO-BE 후보는 `com.github.mwiede:jsch:0.2.x` 또는 대체 라이브러리이다. mwiede JSch fork는 기존 `com.jcraft.jsch` 패키지 구조를 유지하므로 단순 SFTP 사용 영역은 소스 변경 부담이 상대적으로 낮다. 다만 SSH 서버가 허용하는 KEX, Cipher, MAC, HostKey 알고리즘에 따라 접속 성공 여부가 달라질 수 있으므로 운영 연계 테스트가 필수이다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 대부분 없음 또는 일부 수정 필요. `com.jcraft.jsch.*` API 직접 사용 영역 확인 필요 |
| 설정파일    | SFTP 서버, 포트, 계정, 개인키, known_hosts, 알고리즘 설정 확인 필요 |
| pom.xml | `com.jcraft:jsch:0.1.54` 제거 후 `com.github.mwiede:jsch:0.2.x` 또는 대체 라이브러리 추가 |
| 운영환경    | 외부 SFTP 서버 보안정책, 방화벽, 키 파일 권한, 암호화 알고리즘 호환성 영향 가능 |

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
com.jcraft:jsch:0.1.54
```

### TO-BE 버전

```text
com.github.mwiede:jsch:0.2.x
```

또는 프로젝트 표준 보안정책에 따라 Apache MINA SSHD 등 대체 라이브러리 검토 가능

### 주요 변경사항

* JCraft 원본 JSch 계열에서 유지보수 fork 계열로 전환
* 기존 `com.jcraft.jsch` 패키지명은 대체로 유지 가능
* 최신 SSH 알고리즘 지원 범위 개선 가능
* Java 버전에 따라 일부 최신 알고리즘 사용 조건이 달라질 수 있음
* 서버 보안정책에 따라 KEX, Cipher, HostKey 알고리즘 설정 검토 필요

### 소스 수정 여부

* 대부분 없음
* 일부 수정 필요 가능

단순히 Maven 좌표만 변경하고 기존 `com.jcraft.jsch` API가 유지되는 경우 소스 수정은 거의 없을 수 있다. 다만 보안 설정, 키 포맷, 알고리즘 협상, 예외 처리, 파일 송수신 정책 변경이 필요한 경우 일부 수정이 필요하다.

## 7. 위험요소

### 보안

* 구버전 SSH/SFTP 라이브러리 사용 시 최신 서버 보안정책과 맞지 않을 수 있음
* 오래된 KEX, Cipher, MAC, HostKey 알고리즘 사용 가능성 확인 필요
* `StrictHostKeyChecking=no` 사용 시 중간자 공격 위험 증가
* 패스워드 하드코딩, 개인키 파일 권한, known_hosts 관리 방식 점검 필요

### 운영

* SFTP 연계는 외부기관, 배치, 정산, 수집 업무와 연결될 수 있어 장애 영향이 큼
* 버전 변경 후 특정 SFTP 서버와 알고리즘 협상 실패 가능
* 운영망 방화벽, 서버 접근 제어, 계정 잠금 정책 영향 가능
* 파일 전송 실패 시 재처리 및 중복 전송 방지 로직 확인 필요

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 1.8         | TO-BE 후보 버전의 JDK 8 호환성 확인 필요 |
| Spring 5.3      | 직접 영향 낮음 |
| eGovFrame 4.3   | 직접 영향 낮음. 단, 전체 라이브러리 업그레이드에 따른 충돌 가능성 확인 필요 |
| WAS             | WAS 공통 lib 중복 배치 여부 확인 필요 |
| SFTP 서버       | 알고리즘, 키 포맷, 호스트키 정책 영향 큼 |

### 비고

* `jsch-0.1.54.jar`는 단순 유틸리티가 아니라 외부 서버 연계와 연결되는 경우가 많으므로, 라이브러리 교체 자체보다 운영 연계 테스트가 중요하다.
* `com.github.mwiede:jsch`는 기존 JSch API 호환성을 유지하는 fork로 볼 수 있으나, 기관 보안정책 및 폐쇄망 반입 기준에 따라 오픈소스 검증이 필요하다.
* 운영 SFTP 서버가 허용하는 알고리즘 목록을 사전에 확보해야 한다.
* `StrictHostKeyChecking=no` 설정은 운영 보안 기준에 부적합할 수 있으므로 known_hosts 기반 검증 방식으로 개선하는 것을 권장한다.
* SFTP 업로드/다운로드, 실패 재처리, 중복 파일 처리, 대용량 파일 처리, 파일명 인코딩에 대한 회귀 테스트가 필요하다.
