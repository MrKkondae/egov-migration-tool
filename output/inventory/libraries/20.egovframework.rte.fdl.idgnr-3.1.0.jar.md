# 2. 라이브러리 개요

## 주요 기능

* 데이터베이스 기반 고유 ID 생성
* 테이블 기반 Sequence Generator 제공
* Block Allocation 방식 ID 생성
* 서비스 단위 ID 생성 정책 관리
* 전자정부프레임워크 표준 ID Generator 지원

## 사용 목적

전자정부프레임워크 3.x에서 업무 테이블의 PK(Primary Key) 생성 시 사용되는 라이브러리이다.

Oracle Sequence를 직접 사용하는 대신 별도 관리 테이블(COMTECOPSEQ 등)을 이용하여 ID를 생성한다.

주로 다음과 같은 업무에서 사용된다.

* 게시판 번호 생성
* 파일 번호 생성
* 사용자 번호 생성
* 공통코드 관리번호 생성

## 대표 사용 예시

```java
@Resource(name = "egovFileIdGnrService")
private EgovIdGnrService egovFileIdGnrService;

Long fileSn = egovFileIdGnrService.getNextLongId();
```

또는

```xml
<bean id="egovFileIdGnrService"
      class="egovframework.rte.fdl.idgnr.impl.EgovTableIdGnrServiceImpl">
</bean>
```

# 4. eGovFrame 4.3 영향도 분석

## 전환 대상 여부

* 대상

## 전환 사유

eGovFrame 4.3에서도 ID Generation 기능은 계속 제공된다.

다만 패키지명이 변경된다.

### AS-IS

```java
import egovframework.rte.fdl.idgnr.EgovIdGnrService;
```

### TO-BE

```java
import org.egovframe.rte.fdl.idgnr.EgovIdGnrService;
```

즉 기능은 유지되지만 eGovFrame 4.x 패키지 체계로 변경해야 한다.

## 영향 범위

| 구분      | 영향    |
| ------- | ----- |
| 소스      | 있음    |
| 설정파일    | 있음    |
| pom.xml | 있음    |
| 운영환경    | 거의 없음 |

# 5. 전환 시 검토사항

## 변경 필요 사항

### 1. Package 변경

```java
egovframework.rte.fdl.idgnr
↓
org.egovframe.rte.fdl.idgnr
```

### 2. Bean 정의 확인

기존 XML Bean 설정이 존재하는 경우 패키지 변경 필요

예)

```xml
egovframework.rte.fdl.idgnr.impl.EgovTableIdGnrServiceImpl
```

↓

```xml
org.egovframe.rte.fdl.idgnr.impl.EgovTableIdGnrServiceImpl
```

### 3. Sequence 관리 테이블 확인

다음과 같은 테이블 사용 여부 확인

```sql
COMTECOPSEQ
```

또는

```sql
IDS
```

프로젝트별 커스터마이징 여부 확인 필요

## 비고

* eGovFrame 공통컴포넌트에서는 매우 빈번하게 사용되는 라이브러리이다.
* DAO 전환(EgovAbstractDAO → EgovAbstractMapper)과 달리 기능 변경은 거의 없다.
* 대부분 패키지명 변경 수준으로 전환 가능하다.
* Migration Matrix 기준 분류:

| 구분        | 판정            |
| --------- | ------------- |
| 전환 유형     | eGov 전용 라이브러리 |
| 자동변환 난이도  | 매우 낮음         |
| 수동 검토 필요성 | 낮음            |
| 우선순위      | 중             |

**결론:** `egovframework.rte.fdl.idgnr-3.1.0.jar`는 eGovFrame 4.3에서도 계속 사용되는 라이브러리이며, 주된 변경사항은 `egovframework → org.egovframe` 패키지 변경이다. 소스 로직 변경은 거의 없으며 OpenRewrite/Python 규칙으로 자동 전환 가능한 대표적인 라이브러리이다.
