## 2. 라이브러리 개요

### 주요 기능

* Spring Expression Language(SpEL) 제공
* 런타임 객체 접근
* Bean 속성 조회 및 변경
* 조건식 평가
* 동적 메서드 호출 지원

### 사용 목적

`spring-expression`은 Spring Framework의 표현식 처리 엔진(SpEL)을 제공하는 라이브러리이다.

Spring XML 설정, Annotation 설정, Security 설정, Cache 설정 등에서 동적으로 값을 계산하거나 Bean 속성에 접근하기 위해 사용된다.

전자정부 프로젝트에서는 다음과 같은 용도로 사용된다.

* Spring XML 설정
* Property 값 처리
* Cache 설정
* Security 표현식
* Bean 참조 처리

### 대표 사용 예시

```java id="o4p8v7"
ExpressionParser parser =
    new SpelExpressionParser();

Expression exp =
    parser.parseExpression("'Hello'.concat(' World')");

String value =
    exp.getValue(String.class);
```

또는

```java id="yewd3z"
@Value("#{systemProperties['user.home']}")
private String userHome;
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-expression-3.2.9.RELEASE.jar`는 Spring Framework 3.2 계열 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 Spring Core 계열 라이브러리 전체 업그레이드가 필요하다.

`spring-expression`은 `spring-core`, `spring-beans`, `spring-context`와 함께 동작하는 핵심 라이브러리이므로 버전 정합성을 맞추어야 한다.

SpEL(Expression Language)을 사용하는 경우 표현식 처리 결과에 대한 검증이 필요하다.

##### (16) 최종 정리

```text
spring-expression 3.2.9.RELEASE → 5.3.37 전환은 SpEL 엔진 버전 정합성 확보가 핵심이다.

일반적인 @Value, ${...}, #{...}, SpelExpressionParser 사용 코드는 대부분 그대로 유지 가능하다.

OpenRewrite를 통해 Maven 의존성 변경, 구버전 Jar 탐지,
Spring Framework 계열 라이브러리 버전 통일, XML Schema Location 정리는 자동화할 수 있다.

다만 Bean 참조 SpEL, Cache SpEL, Security SpEL,
커스텀 EvaluationContext 사용 코드는 실제 표현식 평가 결과가 동일한지 수동 검증이 필요하다.

전환 후에는 Property 주입, Bean 참조, Cache Key 생성,
Security 권한 표현식 평가 결과를 반드시 검증해야 한다.
```
