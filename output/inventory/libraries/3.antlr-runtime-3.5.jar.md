## 2. 라이브러리 개요

### 주요 기능

* ANTLR 3.x 기반 Parser 실행 Runtime 제공
* Lexer 및 Parser 실행 지원
* Token Stream 관리
* 문법(Grammar) 기반 언어 해석 지원

### 사용 목적

ANTLR(ANother Tool for Language Recognition)은 문법(Grammar)을 기반으로 Parser를 생성하는 도구이다.

ANTLR Tool을 이용하여 생성된 Parser/Lexer는 실행 시 Runtime 라이브러리가 필요하며, ANTLR 3.x 계열에서는 `antlr-runtime-3.5.jar`가 해당 역할을 수행한다.

주요 활용 분야는 다음과 같다.

* DSL(Domain Specific Language)
* SQL Parser
* Query Parser
* Expression Parser
* 코드 생성기
* 사용자 정의 언어 처리

### 대표 사용 예시

```java
import org.antlr.runtime.*;

ANTLRStringStream input =
    new ANTLRStringStream(source);

MyLexer lexer =
    new MyLexer(input);

CommonTokenStream tokens =
    new CommonTokenStream(lexer);

MyParser parser =
    new MyParser(tokens);
```

## 3. AS-IS 사용 현황

### 프로젝트 내 사용 여부

* 확인 필요

AS-IS 라이브러리 목록에서 다음 Jar가 확인되었다.

```text
antlr-runtime-3.5.jar
```

확인된 Maven 좌표

```xml
<dependency>
    <groupId>org.antlr</groupId>
    <artifactId>antlr-runtime</artifactId>
    <version>3.5</version>
</dependency>
```

### 확인 방법

소스 코드 검색

```bash
grep -R "org.antlr.runtime" src
grep -R "ANTLRStringStream" src
grep -R "CommonTokenStream" src
grep -R "RecognitionException" src
```

Maven 의존성 확인

```bash
mvn dependency:tree | grep antlr
```

* DSL 처리 모듈

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 조건부 제외

### 전환 사유

전자정부프레임워크 3.x 및 4.3 기본 구성에서는 ANTLR Runtime 3.x를 사용하지 않는다.

또한 Spring Framework 5.3, MyBatis, eGovFrame 4.3 기본 라이브러리에서도 직접 의존하지 않는다.

따라서 전자정부프레임워크 전환 자체와는 무관한 라이브러리로 판단된다.

다만 프로젝트에서 직접 사용하는 경우에는 전환 이후에도 유지가 필요하다.

### 영향 범위

| 구분      | 영향         |
| ------- | ---------- |
| 소스      | 직접 사용 시 영향 |
| 설정파일    | 없음         |
| pom.xml | 없음         |
| 운영환경    | 없음         |

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
org.antlr:antlr-runtime:3.5
```

### TO-BE 버전

```text
org.antlr:antlr-runtime:3.5 (유지 권고)
```

### 주요 변경사항

ANTLR 계열은 다음과 같이 구분된다.

```text
ANTLR 2.x
  └─ antlr:antlr:2.7.7

ANTLR 3.x
  └─ org.antlr:antlr-runtime:3.5

ANTLR 4.x
  └─ org.antlr:antlr4-runtime
```

ANTLR 3.x와 ANTLR 4.x는 구조 및 API가 완전히 다르다.

따라서 다음과 같은 단순 업그레이드는 불가능하다.

```text
antlr-runtime:3.5
→
antlr4-runtime
```

ANTLR 4 전환 시에는 다음 작업이 필요할 수 있다.

* Grammar 재작성
* Parser 재생성
* 애플리케이션 수정
* 기능 테스트 재수행

### 소스 수정 여부

* eGovFrame 4.3 전환 시 없음
* ANTLR 4 전환 시 대규모 수정 필요

## 9. 최종 판정

### 판정

```text
전환대상여부 : 조건부 제외

조치방안 :
- org.antlr.runtime 사용 여부 확인
- 직접 사용 시 현행 유지
- 미사용 시 제거 검토

자동변환 :
- 불필요

소스수정 :
- 없음

위험도 :
- 낮음
```

### 비고

* ANTLR 2.7.7과는 별개의 라이브러리이다.
* ANTLR 2.7.7은 Hibernate 전이 의존성으로 유입된다.
* ANTLR Runtime 3.5는 프로젝트 또는 외부 솔루션에서 직접 사용할 가능성이 있다.
* 전자정부프레임워크 4.3 전환과 직접적인 관련은 없다.
* 최종 판단을 위해 다음 항목 확인이 필요하다.

  * `org.antlr.runtime` import 사용 여부
  * ANTLR Grammar(`*.g`) 파일 존재 여부
  * Maven 의존성 유입 경로 확인
