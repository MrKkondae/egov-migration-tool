## 2. 라이브러리 개요

### 주요 기능

* Spring Bean 생성 및 관리
* Dependency Injection(DI) 지원
* BeanFactory 제공
* ApplicationContext 기반 Bean 관리
* Bean 생명주기(Lifecycle) 관리

### 사용 목적

`spring-beans`는 Spring IoC Container의 핵심 라이브러리로 Bean 생성, 초기화, 의존성 주입 및 생명주기 관리를 담당한다.

Spring Framework의 가장 핵심적인 구성요소 중 하나이며, 대부분의 Spring 기반 애플리케이션이 직접 또는 간접적으로 사용한다.

### 대표 사용 예시

```java id="qj6u1k"
@Service
public class UserService {

    @Autowired
    private UserDAO userDAO;

}
```

또는

```xml id="wmbmzm"
<bean id="userService"
      class="egovframework.sample.service.UserService">
    <property name="userDAO" ref="userDAO"/>
</bean>
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-beans-3.2.9.RELEASE.jar`는 Spring Framework 3.2 계열의 핵심 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 Spring Core 계열 라이브러리 전체를 업그레이드해야 한다.

특히 `spring-beans`는 Spring IoC Container의 핵심 모듈로 `spring-core`, `spring-context`, `spring-expression`, `spring-aop`, `spring-tx`와 강하게 연관되어 있으므로 단독 업그레이드는 불가능하다.

| 설정파일    | 높음 |

OpenRewrite를 통해 Maven 의존성 변경, 구버전 Jar 탐지,
Spring Framework 계열 라이브러리 버전 통일, XML Schema Location 정리는 자동화할 수 있다.

다만 XmlBeanFactory와 같은 구형 BeanFactory 직접 사용,
커스텀 FactoryBean, BeanPostProcessor, 순환참조 Bean 구조는 프로젝트별로 수동 검토가 필요하다.

전환 후에는 ApplicationContext 기동 여부, Bean 생성 여부,
Dependency Injection 정상 동작 여부를 반드시 검증해야 한다.
