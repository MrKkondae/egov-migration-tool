## 2. 라이브러리 개요

### 주요 기능

* JSON 파싱(JsonParser)
* JSON 생성(JsonGenerator)
* JSON 토큰 처리(JsonToken)
* Streaming API 기반 저수준 JSON 입출력 처리
* `jackson-mapper-asl` 등 Jackson 1.x 데이터 바인딩 계층의 기반 기능 제공

### 사용 목적

`jackson-core-asl-1.9.13.jar`는 Jackson 1.x 계열의 JSON Core 라이브러리이다.  
주로 JSON 문자열, 파일, HTTP 요청/응답 데이터를 파싱하거나 생성하는 데 사용된다.

Jackson 1.x 계열은 `org.codehaus.jackson` 패키지를 사용하며, Jackson 2.x 계열은 `com.fasterxml.jackson` 패키지를 사용한다.  
따라서 단순 JAR 교체만으로는 전환이 완료되지 않으며, import 구문과 일부 API 사용 방식 변경이 필요하다.

전자정부프레임워크 4.3 전환 관점에서는 Jackson 1.x 계열을 신규 표준으로 유지하기보다 Jackson 2.x 계열로 통합하는 것이 바람직하다.

### 대표 사용 예시

#### AS-IS: Jackson 1.x

```java
import org.codehaus.jackson.JsonFactory;
import org.codehaus.jackson.JsonParser;
import org.codehaus.jackson.JsonToken;

public class LegacyJacksonSample {
    public void parse(String json) throws Exception {
        JsonFactory factory = new JsonFactory();
        JsonParser parser = factory.createJsonParser(json);

        while (parser.nextToken() != JsonToken.END_OBJECT) {
            String fieldName = parser.getCurrentName();
            // JSON 필드 처리
        }

        parser.close();
    }
}
```

#### TO-BE: Jackson 2.x

```java
import com.fasterxml.jackson.core.JsonFactory;
import com.fasterxml.jackson.core.JsonParser;
import com.fasterxml.jackson.core.JsonToken;

public class Jackson2Sample {
    public void parse(String json) throws Exception {
        JsonFactory factory = new JsonFactory();
        JsonParser parser = factory.createParser(json);

        while (parser.nextToken() != JsonToken.END_OBJECT) {
            String fieldName = parser.currentName();
            // JSON 필드 처리
        }

        parser.close();
    }
}
```

## 3. AS-IS 사용 현황

### 프로젝트 내 사용 여부

* 확인 필요
* 직접 사용 시 전환 대상
* 외부 라이브러리 전이 의존성으로만 존재하는 경우 별도 영향도 검토 필요

### 확인 방법

#### 1) 소스 import 사용 여부 확인

```bash
grep -R "org.codehaus.jackson" src
```

#### 2) Jackson Core API 사용 여부 확인

```bash
grep -R "JsonFactory\|JsonParser\|JsonGenerator\|JsonToken" src
```

#### 3) Spring MVC 설정 내 Jackson 1.x 사용 여부 확인

```bash
grep -R "MappingJacksonHttpMessageConverter\|org.codehaus.jackson" src main webapp WEB-INF
```

#### 4) Maven 의존성 확인

```bash
mvn dependency:tree -Dincludes=org.codehaus.jackson
mvn dependency:tree -Dincludes=com.fasterxml.jackson.core
```

#### 5) WEB-INF/lib 직접 배포 여부 확인

```bash
find . -name "jackson-core-asl-*.jar" -o -name "jackson-mapper-asl-*.jar"
find . -name "jackson-core-*.jar" -o -name "jackson-databind-*.jar"
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`jackson-core-asl-1.9.13.jar`는 Jackson 1.x 계열 라이브러리이며, eGovFrame 4.3 전환 시 Jackson 2.x 계열인 `com.fasterxml.jackson.core:jackson-core`로 대체하는 것이 권장된다.

Matrix 기준으로도 30번 항목은 `JSON / JSON Core / 대체` 대상으로 정의되어 있으며, `org.codehaus.jackson` 패키지를 `com.fasterxml.jackson` 패키지로 변경해야 한다.

다만 eGovFrame Lite 4.3 대규모 라이브러리 목록에는 Jackson 2.x 계열(`jackson-core 2.17.1`, `jackson-databind 2.17.1`)과 Jackson 1.x 계열(`jackson-core-asl 1.9.13`, `jackson-mapper-asl 1.9.13`)이 함께 포함되어 있다.  
따라서 실제 사업에서는 다음 기준으로 구분해야 한다.

* 업무 소스가 `org.codehaus.jackson`을 직접 import하는 경우: Jackson 2.x로 전환
* 외부 라이브러리 또는 eGovFrame 제공 묶음에 포함된 경우: 직접 제거하지 말고 의존성 사용 주체 확인
* Jackson 1.x와 2.x가 동시에 존재하는 경우: 신규/수정 소스는 Jackson 2.x 기준으로 통합

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | `org.codehaus.jackson.*` import를 `com.fasterxml.jackson.*`로 변경 필요 |
| 설정파일    | Spring MVC message-converter, ObjectMapper Bean 설정 확인 필요 |
| pom.xml | `org.codehaus.jackson:jackson-core-asl` 제거 후 `com.fasterxml.jackson.core:jackson-core` 추가 |
| 운영환경    | WEB-INF/lib 또는 WAS 공통 lib 내 Jackson 1.x/2.x 중복 구성 확인 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `org.codehaus.jackson.*` import 제거
* `com.fasterxml.jackson.core.*` import 적용
* `jackson-core-asl-1.9.13.jar` 제거 여부 판단
* `jackson-mapper-asl-1.9.13.jar`와 함께 사용하는 경우 31번 항목과 통합 검토
* Spring MVC 설정의 구형 `MappingJacksonHttpMessageConverter` 사용 여부 확인
* `ObjectMapper`, `JsonFactory`, `JsonParser`, `JsonGenerator` 사용 방식 변경 확인
* JSON 직렬화/역직렬화 결과 회귀 테스트
* Jackson 1.x와 2.x JAR 동시 존재 여부 확인
* WAS 공통 라이브러리와 애플리케이션 `WEB-INF/lib` 중복 여부 확인

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 가능 |
| Python Rule | 부분 가능 |
| LLM 보정      | 필요 |

#### 자동 변환 가능 항목

* Maven 의존성 좌표 변경
* `org.codehaus.jackson` → `com.fasterxml.jackson` import 변경
* 단순 클래스명 매핑
* 사용처 목록화
* 변환 대상 파일 목록 생성

#### 수동 검토 필요 항목

* `ObjectMapper` 세부 설정
* 커스텀 Serializer / Deserializer
* Annotation 사용 방식
* 날짜/시간 포맷 처리
* null 처리 정책
* 예외 처리 클래스 변경
* Spring MVC message-converter 설정
* 기존 JSON 응답 포맷과 TO-BE 응답 포맷 비교

### 예상 난이도

* 중간

단순 Streaming API 사용만 있다면 난이도는 낮음~중간 수준이다.  
다만 `jackson-mapper-asl`, Spring MVC message-converter, 커스텀 직렬화 설정과 함께 사용 중이면 31번 항목과 연계되어 난이도는 중간~높음으로 상승한다.

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
org.codehaus.jackson:jackson-core-asl:1.9.13
```

### TO-BE 버전

```text
com.fasterxml.jackson.core:jackson-core:2.x
```

eGovFrame 4.3 Lite 대규모 라이브러리 목록 기준으로는 다음 버전이 확인된다.

```text
com.fasterxml.jackson.core:jackson-core:2.17.1
com.fasterxml.jackson.core:jackson-databind:2.17.1
com.fasterxml.jackson.core:jackson-annotations:2.17.1
```

사업 적용 시에는 프로젝트 표준 라이브러리 목록, 보안점검 기준, WAS/JDK 호환성 기준에 따라 Jackson 2.x 세부 버전을 확정해야 한다.

### 주요 변경사항

* Maven 좌표 변경
  * AS-IS: `org.codehaus.jackson:jackson-core-asl`
  * TO-BE: `com.fasterxml.jackson.core:jackson-core`
* Java 패키지 변경
  * AS-IS: `org.codehaus.jackson`
  * TO-BE: `com.fasterxml.jackson.core`
* Jackson 1.x와 2.x API 일부 차이 발생
* Jackson 2.x는 core, annotations, databind 모듈을 분리하여 관리
* `jackson-mapper-asl` 사용 영역은 `jackson-databind` 및 `jackson-annotations`와 함께 검토 필요

### 소스 수정 여부

* 일부 수정 필요

#### 단순 Core API 사용 시

* import 변경
* 일부 메서드명 변경
* 예외 처리 확인

#### ObjectMapper / Annotation / MessageConverter 사용 시

* `jackson-mapper-asl-1.9.13.jar` 분석보고서와 연계 필요
* Spring MVC 설정 변경 필요
* JSON 응답 결과 회귀 테스트 필요

## 7. 위험요소

### 보안

* Jackson 1.x 계열은 유지보수 관점에서 장기 사용에 부적합하다.
* 보안점검 시 `org.codehaus.jackson` 계열 구버전 라이브러리로 식별될 가능성이 있다.
* `jackson-core` 단독보다 `jackson-mapper-asl`, `jackson-databind`와 함께 사용하는 영역의 취약점 검토가 중요하다.
* 최종 적용 전 SCA 도구 또는 보안취약점 DB 기준으로 CVE 점검이 필요하다.

### 운영

* Jackson 1.x와 2.x가 동시에 존재하면 신규 소스와 레거시 소스가 서로 다른 ObjectMapper 체계를 사용할 수 있다.
* JSON 응답 필드 순서, 날짜 포맷, null 처리, escape 처리 방식이 달라질 수 있다.
* REST API, 외부 연계, Ajax 응답 화면은 회귀 테스트가 필요하다.
* WAS 공통 lib와 애플리케이션 lib에 서로 다른 Jackson 버전이 중복 배치되면 클래스 로딩 이슈가 발생할 수 있다.

### 호환성

| 항목            | 영향 |
| ------------- | ---- |
| JDK 1.8       | Jackson 2.x 적용 가능. 단, 적용 버전은 프로젝트 표준 기준으로 확정 필요 |
| Spring 5.3.x  | Jackson 2.x 기반 MessageConverter 사용 권장 |
| eGovFrame 4.3 | Jackson 2.x 계열과 호환. eGovFrame Lite 대규모 목록에도 Jackson 2.17.1 포함 |
| WAS           | WEB-INF/lib와 WAS 공통 lib 중복 여부 확인 필요 |
| 소스           | `org.codehaus.jackson` 직접 import 시 수정 필요 |

## 9. 최종 판정

### 판정

```text
전환대상여부 : 대상

조치방안 :
- jackson-core-asl-1.9.13.jar 직접 사용 여부 확인
- 업무 소스의 org.codehaus.jackson import를 com.fasterxml.jackson.core로 전환
- TO-BE는 com.fasterxml.jackson.core:jackson-core:2.x 적용
- eGovFrame 4.3 기준 Jackson 2.17.1 계열을 1차 검토 후보로 사용
- jackson-mapper-asl 사용 영역은 31번 항목과 함께 통합 전환
- WEB-INF/lib 및 WAS 공통 lib의 Jackson 1.x/2.x 중복 구성 확인

자동변환 :
- 부분 가능
- OpenRewrite의 Codehaus → FasterXML 전환 Recipe 활용 가능
- 단순 import 및 의존성 변경은 자동화 가능
- JSON 포맷, ObjectMapper 설정, MessageConverter 설정은 수동 검토 필요

소스수정 :
- 일부 수정 필요

위험도 :
- 중간
```

### 비고

* Matrix 기준으로 30번 항목은 Jackson 1.x 계열에서 Jackson 2.x 계열로 대체하는 항목이다.
* eGovFrame Lite 4.3 대규모 라이브러리 목록에는 `jackson-core-asl 1.9.13`도 포함되어 있으므로, 단순히 파일명만 보고 제거하면 안 된다.
* 업무 소스에서 직접 사용하는 `org.codehaus.jackson` 계열은 전환 대상으로 보고, 외부 라이브러리 전이 의존성은 사용 주체를 확인한 뒤 제거 여부를 판단해야 한다.
* `jackson-core-asl`은 `jackson-mapper-asl`과 함께 사용되는 경우가 많으므로 31번 `jackson-mapper-asl-1.9.13.jar` 분석과 연계해야 한다.
* 최종 반영 전 REST API, Ajax 응답, 외부 연계 JSON 전문, 배치 JSON 파일 처리 결과에 대한 회귀 테스트가 필요하다.
* `Library Migration Matrix.md` 파일에는 Git 병합 충돌 마커가 남아 있으므로 최종 산출물 반영 전 Matrix 파일 정리가 필요하다.
