## 2. 라이브러리 개요

### 주요 기능

* JSON 문자열과 Java 객체 간 변환
* `ObjectMapper` 기반 직렬화 / 역직렬화
* Java Bean, Map, List, 배열 등 데이터 구조 매핑
* Spring MVC JSON 응답/요청 처리 연계
* `@JsonProperty`, `@JsonIgnore`, `@JsonFormat` 등 Jackson Annotation 기반 매핑 지원

### 사용 목적

`jackson-mapper-asl-1.9.13.jar`는 Jackson 1.x 계열의 JSON 데이터 바인딩 라이브러리이다.  
JSON 문자열을 Java 객체로 변환하거나 Java 객체를 JSON 문자열로 변환할 때 사용된다.

Jackson 1.x 계열은 `org.codehaus.jackson` 패키지를 사용하고, Jackson 2.x 계열은 `com.fasterxml.jackson` 패키지를 사용한다.  
따라서 `jackson-mapper-asl-1.9.13.jar`를 `jackson-databind`로 전환할 때는 단순 JAR 교체만으로는 부족하며, import 구문, Spring MVC 메시지 컨버터 설정, ObjectMapper 설정, Annotation 사용 여부를 함께 확인해야 한다.

전자정부프레임워크 4.3 전환 관점에서는 Jackson 1.x 계열을 신규 표준으로 계속 사용하기보다 Jackson 2.x 계열인 `jackson-databind`, `jackson-core`, `jackson-annotations`로 통합 전환하는 것이 바람직하다.

### 대표 사용 예시

#### AS-IS: Jackson 1.x

```java
import org.codehaus.jackson.map.ObjectMapper;

public class JsonSample {
    public String toJson(User user) throws Exception {
        ObjectMapper mapper = new ObjectMapper();
        return mapper.writeValueAsString(user);
    }

    public User fromJson(String json) throws Exception {
        ObjectMapper mapper = new ObjectMapper();
        return mapper.readValue(json, User.class);
    }
}
```

#### TO-BE: Jackson 2.x

```java
import com.fasterxml.jackson.databind.ObjectMapper;

public class JsonSample {
    public String toJson(User user) throws Exception {
        ObjectMapper mapper = new ObjectMapper();
        return mapper.writeValueAsString(user);
    }

    public User fromJson(String json) throws Exception {
        ObjectMapper mapper = new ObjectMapper();
        return mapper.readValue(json, User.class);
    }
}
```

단순한 `ObjectMapper` 사용은 import 변경만으로 전환되는 경우가 많지만, 날짜 포맷, Null 처리, Enum 처리, 커스텀 Serializer/Deserializer, Spring MVC 메시지 컨버터 설정은 별도 검증이 필요하다.

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`jackson-mapper-asl-1.9.13.jar`는 Jackson 1.x 계열의 구버전 JSON 매핑 라이브러리이다.  
전자정부프레임워크 4.3 환경은 Spring 5.3 기반이며, 일반적으로 Jackson 2.x 계열의 `MappingJackson2HttpMessageConverter` 및 `com.fasterxml.jackson` 패키지 기반 구성이 표준적이다.

기존 업무 소스에서 `org.codehaus.jackson.map.ObjectMapper` 또는 `MappingJacksonHttpMessageConverter`를 직접 사용하고 있다면, eGovFrame 4.3 전환 시 Jackson 2.x 계열로 변경하는 것이 필요하다.

단, eGovFrame 4.3 Lite 대규모 라이브러리 목록에는 Jackson 2.x 계열과 함께 legacy Jackson 1.x 라이브러리도 포함되어 있는 것으로 확인된다. 따라서 업무 소스 직접 사용분은 Jackson 2.x로 전환하되, 외부 라이브러리 또는 프레임워크 잔존 의존성으로 포함되는 경우에는 사용 주체를 분리해서 판단해야 한다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | `org.codehaus.jackson` import, `ObjectMapper`, Annotation, Serializer/Deserializer 사용 소스 수정 필요 |
| 설정파일    | Spring MVC `MappingJacksonHttpMessageConverter` 설정 사용 시 `MappingJackson2HttpMessageConverter`로 변경 필요 |
| pom.xml | `org.codehaus.jackson:jackson-mapper-asl` 제거, `com.fasterxml.jackson.core:jackson-databind`, `jackson-core`, `jackson-annotations` 추가 필요 |
| 운영환경    | JSON 응답 포맷, 날짜/시간 포맷, Null 처리, 문자 인코딩, 외부 연계 인터페이스 회귀 테스트 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `org.codehaus.jackson.map.ObjectMapper` → `com.fasterxml.jackson.databind.ObjectMapper`
* `org.codehaus.jackson.JsonNode` → `com.fasterxml.jackson.databind.JsonNode`
* `org.codehaus.jackson.annotate.*` → `com.fasterxml.jackson.annotation.*`
* `org.codehaus.jackson.map.annotate.*` → `com.fasterxml.jackson.databind.annotation.*`
* `MappingJacksonHttpMessageConverter` → `MappingJackson2HttpMessageConverter`
* `jackson-mapper-asl` → `jackson-databind`로 Maven 의존성 변경
* `jackson-core-asl`은 30번 항목과 함께 `jackson-core`로 통합 변경
* JSON 직렬화 결과가 변경되는지 업무별 회귀 테스트 수행

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 부분 가능 |
| Python Rule | 부분 가능 |
| LLM 보정      | 필요 |

### 자동 변환 가능 범위

* Maven 의존성 좌표 변경
* `org.codehaus.jackson` → `com.fasterxml.jackson` import 변경
* `ObjectMapper`, `JsonNode`, `JsonParser`, `JsonGenerator` 등 대표 타입 변경
* Spring MVC 메시지 컨버터 클래스명 변경 후보 탐지
* Jackson 1.x 사용처 목록화

### 수동 검토 필요 범위

* 커스텀 Serializer / Deserializer
* 날짜/시간 포맷 처리
* `SerializationConfig.Feature`, `DeserializationConfig.Feature` 등 설정값 변경
* Null 필드 포함/제외 정책
* Enum 직렬화 방식
* 외부 연계 API JSON 포맷 호환성
* `ObjectMapper` 전역 Bean 설정
* 레거시 JSON Util 공통 모듈

### 예상 난이도

* 중간

단순 JSON 변환 코드만 있으면 난이도는 낮을 수 있으나, Spring MVC 전역 설정, 공통 JSON Util, 외부 연계 포맷, 날짜/시간 포맷이 포함되어 있으면 난이도는 중간 이상으로 상승한다.

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
org.codehaus.jackson:jackson-mapper-asl:1.9.13
```

### TO-BE 버전

```text
com.fasterxml.jackson.core:jackson-databind:2.x
com.fasterxml.jackson.core:jackson-core:2.x
com.fasterxml.jackson.core:jackson-annotations:2.x
```

### eGovFrame 4.3 기준 검토 후보

```text
com.fasterxml.jackson.core:jackson-databind:2.17.1
com.fasterxml.jackson.core:jackson-core:2.17.1
com.fasterxml.jackson.core:jackson-annotations:2.17.1
```

실제 적용 버전은 eGovFrame 4.3 표준 라이브러리 목록, 내부 보안 검증 기준, JDK 버전, WAS 호환성 기준에 맞춰 확정해야 한다.

### 주요 변경사항

* Maven Group ID 변경
  * AS-IS: `org.codehaus.jackson`
  * TO-BE: `com.fasterxml.jackson.core`
* 패키지 변경
  * AS-IS: `org.codehaus.jackson.*`
  * TO-BE: `com.fasterxml.jackson.*`
* 데이터 바인딩 Artifact 변경
  * AS-IS: `jackson-mapper-asl`
  * TO-BE: `jackson-databind`
* Annotation Artifact 분리
  * TO-BE에서는 `jackson-annotations` 의존성 확인 필요
* Spring MVC 메시지 컨버터 변경
  * AS-IS: `MappingJacksonHttpMessageConverter`
  * TO-BE: `MappingJackson2HttpMessageConverter`

### 소스 수정 여부

* 일부 수정 필요
* 공통 JSON Util과 Spring MVC 설정이 복잡한 경우 대규모 수정 가능

## 7. 위험요소

### 보안

* Jackson 1.x 계열은 구버전이므로 신규 표준으로 유지하는 것은 바람직하지 않음
* Jackson Databind는 과거 다수의 역직렬화 관련 취약점 이력이 있으므로, TO-BE 버전은 내부 보안 검증 기준에 맞춰 확정 필요
* 외부 입력 JSON을 다형성 역직렬화하는 경우 `DefaultTyping` 사용 여부 확인 필요

### 운영

* JSON 응답 구조 변경 가능성
* 날짜/시간 직렬화 결과 변경 가능성
* Null 필드 처리 결과 변경 가능성
* 외부 시스템 연계 API와 JSON 필드명, 대소문자, 포맷 불일치 가능성
* 전역 `ObjectMapper` 설정 변경 시 전체 API 응답에 영향 가능

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 8           | Jackson 2.x 계열 적용 가능. 단, 최신 3.x 계열은 별도 검토 필요 |
| Spring 5.3      | Jackson 2.x 메시지 컨버터 사용 권장 |
| eGovFrame 4.3   | Jackson 2.x 계열 통합 전환 검토 필요 |
| Jackson 1.x     | `org.codehaus.jackson` 패키지 기반으로 Jackson 2.x와 API 호환 불가 |
| 30번 항목       | `jackson-core-asl`과 함께 통합 전환 필요 |

### 비고

* 31번 항목은 30번 `jackson-core-asl-1.9.13.jar`와 분리해서 판단하지 말고 세트로 전환해야 한다.
* `jackson-mapper-asl`은 Jackson 1.x의 데이터 바인딩 계층이고, TO-BE의 직접 대응 라이브러리는 `jackson-databind`이다.

* eGovFrame 4.3 Lite 대규모 라이브러리 목록에는 Jackson 2.x와 Jackson 1.x가 함께 보일 수 있으므로, 단순 목록 존재만으로 유지 판단하지 말고 실제 사용 주체를 확인해야 한다.
* 외부 시스템과 JSON 전문을 주고받는 경우 전환 전후 JSON 샘플을 비교하여 필드명, 날짜 포맷, Null 처리, 문자 인코딩 변경 여부를 검증해야 한다.
* `Library Migration Matrix.md`에 Git 병합 충돌 마커가 남아 있으므로 최종 산출물 반영 전 Matrix 파일 정리가 필요하다.
