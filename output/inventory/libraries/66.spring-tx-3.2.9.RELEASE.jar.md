## 2. 라이브러리 개요

### 주요 기능

* 선언적 트랜잭션 관리
* 프로그래밍 방식 트랜잭션 관리
* Transaction Manager 제공
* AOP 기반 트랜잭션 처리
* JDBC, iBATIS, MyBatis, JPA, Hibernate 연계 지원

### 사용 목적

`spring-tx`는 Spring Framework의 트랜잭션 처리 기능을 제공하는 핵심 라이브러리이다.

전자정부프레임워크 기반 시스템에서는 데이터베이스 처리 시 트랜잭션의 시작, 커밋, 롤백을 관리하기 위해 사용된다.

주로 다음과 같은 환경에서 사용된다.

* JDBC
* iBATIS
* MyBatis
* Hibernate
* JPA

### 대표 사용 예시

#### Annotation 기반

```java id="xz9m4r"
@Service
public class UserService {

    @Transactional
    public void saveUser(UserVO user) {

    }

}
```

#### XML 기반

```xml id="5q9gca"
<tx:advice id="txAdvice">
    <tx:attributes>
        <tx:method name="*" propagation="REQUIRED"/>
    </tx:attributes>
</tx:advice>
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-tx-3.2.9.RELEASE.jar`는 Spring Framework 3.2 기반 트랜잭션 관리 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 Spring Transaction 라이브러리 역시 함께 업그레이드해야 한다.

특히 `spring-tx`는 다음 라이브러리와 밀접하게 연관된다.

* spring-jdbc
* spring-orm
* spring-aop
* spring-context
* spring-beans

트랜잭션 설정 오류는 데이터 정합성 문제로 이어질 수 있으므로 전환 시 반드시 검증이 필요하다.

| 설정파일    | 높음 |

### 비고

* eGovFrame 4.3 전환 시 반드시 업그레이드해야 하는 핵심 라이브러리이다.
* `spring-tx`는 데이터 정합성과 직접 관련되므로 전환 후 가장 우선적으로 검증해야 하는 영역 중 하나이다.
* `@Transactional`, `<tx:advice>`, `<tx:annotation-driven>` 설정을 사용하는 경우 반드시 동작 검증이 필요하다.
* 프로젝트에서 iBATIS → MyBatis 전환이 포함되는 경우 Transaction 설정도 함께 점검해야 한다.
* CRUD 기능뿐 아니라 예외 발생 시 Rollback 처리 여부까지 포함한 통합 테스트 수행이 필요하다.
