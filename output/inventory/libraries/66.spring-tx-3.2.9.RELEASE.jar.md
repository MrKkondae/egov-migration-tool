## 2. 라이브러리 개요

### 주요 기능

* 선언적 트랜잭션 관리
* 프로그래밍 방식 트랜잭션 관리
* Transaction Manager 제공
* AOP 기반 트랜잭션 처리
* JDBC, iBATIS, MyBatis, JPA, Hibernate 연계 지원

### 사용 목적

`spring-tx`는 Spring Framework의 트랜잭션 처리 기능을 제공하는 핵심 라이브러리이다.

전자정부프레임워크 기반 시스템에서는 데이터베이스 처리 시 트랜잭션의 시작, 커밋, 롤백을 관리하기 위해 사용된다.

주로 다음과 같은 환경에서 사용된다.

* JDBC
* iBATIS
* MyBatis
* Hibernate
* JPA

### 대표 사용 예시

#### Annotation 기반

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-tx-3.2.9.RELEASE.jar`는 Spring Framework 3.2 기반 트랜잭션 관리 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 Spring Transaction 라이브러리 역시 함께 업그레이드해야 한다.

특히 `spring-tx`는 다음 라이브러리와 밀접하게 연관된다.

* spring-jdbc
* spring-orm
* spring-aop
* spring-context
* spring-beans

트랜잭션 설정 오류는 데이터 정합성 문제로 이어질 수 있으므로 전환 시 반드시 검증이 필요하다.

| 설정파일    | 높음 |

```java
@Service

    @Transactional
```

##### (16) 최종 정리

```text
spring-tx 3.2.9.RELEASE → 5.3.37 전환은 트랜잭션 API 변경보다 Spring Framework 계열 버전 정합성과 실제 commit/rollback 검증이 핵심이다.

일반적인 @Transactional, TransactionTemplate, DataSourceTransactionManager,
<tx:annotation-driven>, <tx:advice> 설정은 대부분 그대로 유지 가능하다.

OpenRewrite를 통해 Maven 의존성 변경, 구버전 Jar 탐지,
Spring Framework 계열 라이브러리 버전 통일, XML Schema Location 정리는 자동화할 수 있다.

다만 다중 TransactionManager, ORM 연계 TransactionManager,
AOP Advisor 연계, Checked Exception rollback 정책은 수동 검토가 필요하다.

전환 후에는 주요 CRUD, 배치 처리, 예외 발생 시 rollback,
read-only Transaction, 다중 DB Transaction 적용 여부를 반드시 검증해야 한다.
```
