## 2. 라이브러리 개요

### 주요 기능

* DispatcherServlet 제공
* Controller 처리
* URL Mapping 처리
* View Resolver 처리
* REST Controller 처리
* Data Binding 처리
* Validation 처리

### 사용 목적

`spring-webmvc`는 Spring MVC 프레임워크의 핵심 라이브러리이다.

전자정부프레임워크 기반 웹 시스템의 Controller, URL Mapping, 화면 처리, REST 서비스 대부분이 이 라이브러리를 기반으로 동작한다.

전자정부 프로젝트에서는 다음과 같은 기능에 사용된다.

* Controller 처리
* 화면 이동
* REST API 처리
* 파일 다운로드
* Validation 처리
* Exception 처리

### 대표 사용 예시

#### Controller

```java id="d2v8km"
@Controller
@RequestMapping("/user")
public class UserController {

    @RequestMapping("/list.do")
    public String selectUserList() {

        return "user/list";
    }
}
```

#### REST Controller

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-webmvc-3.2.9.RELEASE.jar`는 Spring Framework 3.2 기반 MVC 프레임워크이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 반드시 업그레이드해야 한다.

특히 `spring-webmvc`는 전자정부프레임워크 웹 애플리케이션의 핵심 실행 구조를 담당하므로 영향도가 가장 높은 라이브러리 중 하나이다.

다음 영역에 직접적인 영향을 준다.

* Controller
* URL Mapping
* View Resolver
* Validation
* REST 서비스
* File Upload/Download
* Exception 처리

* 영향도 기준으로는 현재까지 분석한 Spring 라이브러리 중 `spring-core`, `spring-context`, `spring-webmvc`, `spring-tx`가 최상위 우선순위 전환 대상으로 분류된다.

##### (1) Java 소스 수정 여부 요약

| 구분                                  |  수정 여부 | 내용                                      |
| ----------------------------------- | -----: | --------------------------------------- |
| `@Controller`                       | 대부분 없음 | 일반적인 Controller 클래스는 유지 가능              |
| `@RequestMapping`                   | 대부분 없음 | 기존 URL 매핑 방식 유지 가능                      |
| `@GetMapping`, `@PostMapping`       |     선택 | 기존 `@RequestMapping`을 유지해도 됨            |
| `@ResponseBody`                     | 대부분 없음 | JSON 응답 처리 구조 유지 가능                     |
| `@RestController`                   |     선택 | 기존 `@Controller + @ResponseBody` 유지 가능  |
| `Model`, `ModelMap`, `ModelAndView` | 대부분 없음 | 화면 전달 방식 유지 가능                          |
| `@RequestParam`                     | 대부분 없음 | 파라미터 바인딩 방식 유지 가능                       |
| `@ModelAttribute`                   | 대부분 없음 | Command/Form 객체 바인딩 유지 가능               |
| `@Valid`, `BindingResult`           |  검토 필요 | Validation 라이브러리 전환 여부와 함께 검증           |
| `@ExceptionHandler`                 | 대부분 없음 | 일반적인 예외 처리 방식 유지 가능                     |
| ViewResolver 설정                     |  검토 필요 | JSP 경로, Tiles, JsonView 등 확인 필요         |
| MessageConverter 설정                 |  검토 필요 | Jackson 2.x, 문자셋, JSON 응답 검증 필요         |
| Multipart 설정                        |  검토 필요 | `spring-web`, commons-fileupload와 함께 검증 |

    @ExceptionHandler(Exception.class)
    public String handleException(Exception e) {
        return "cmmn/error";
    }
}
```

```

###### 비고

기존 `SimpleMappingExceptionResolver`는 유지 가능하다. `@ControllerAdvice` 전환은 선택사항이다.

##### (16) OpenRewrite 적용 범위 정리

| 구분                                                   | 자동변환        |
| ---------------------------------------------------- | ----------- |
| `pom.xml` spring-webmvc 버전 변경                        | 가능          |
| `WEB-INF/lib/spring-webmvc-3.2.9.RELEASE.jar` 제거 목록화 | 가능          |
| Spring Framework 계열 버전 통일                            | 가능          |
| `@Controller` 코드                                     | 수정 불필요      |
| `@RequestMapping` 코드                                 | 수정 불필요      |
| `@RequestMapping` → `@GetMapping/@PostMapping`       | 일부 가능, 선택사항 |
| `ModelAndView` 사용 코드                                 | 수정 불필요      |
| `@ResponseBody` 사용 코드                                | 수정 불필요      |
| `@RestController` 전환                                 | 선택사항        |
| ViewResolver 설정                                      | 대부분 수정 불필요  |
| `mvc:annotation-driven`                              | 수정 불필요      |
| MessageConverter 설정                                  | 수동 검토       |
| Validation 설정                                        | 수동 검토       |
| ExceptionResolver 설정                                 | 대부분 수정 불필요  |
| XML Schema Location 정리                               | 가능          |
| 전체 URL 호출 및 화면 이동 검증                                 | 수동          |

##### (17) 최종 정리

```text
spring-webmvc 3.2.9.RELEASE → 5.3.37 전환은 Spring MVC 웹 계층의 버전 정합성과 Controller, URL Mapping, ViewResolver, MessageConverter 검증이 핵심이다.

일반적인 @Controller, @RequestMapping, ModelAndView, @ResponseBody,
mvc:annotation-driven, ViewResolver, ExceptionResolver 설정은 대부분 그대로 유지 가능하다.

OpenRewrite를 통해 Maven 의존성 변경, 구버전 Jar 탐지,
Spring Framework 계열 라이브러리 버전 통일, XML Schema Location 정리는 자동화할 수 있다.

다만 @RequestMapping을 @GetMapping/@PostMapping으로 바꾸는 것은 필수가 아니며,
안정성 중심 전환에서는 기존 @RequestMapping 유지가 더 적절하다.

MessageConverter, Validation, JSON 응답, JSP 화면 이동,
파일 업로드/다운로드, 예외 화면은 수동 검증 대상으로 관리해야 한다.
```
