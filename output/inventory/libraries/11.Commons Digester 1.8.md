## 2. 라이브러리 개요

### 주요 기능

* XML → Java Object 매핑
* SAX 기반 XML 파싱
* XML Rule 기반 객체 생성 및 속성 매핑
* Apache Struts, Tomcat 설정 파싱 지원
* XML 설정파일 자동 바인딩

### 사용 목적

Commons Digester는 XML 문서를 읽어 Java 객체로 자동 변환하는 라이브러리이다.

JAXB 등장 이전에 많이 사용되었으며 다음과 같은 용도로 활용되었다.

* XML 설정파일 로딩
* 프레임워크 설정 파싱
* 배치 설정파일 처리
* Struts Configuration Parsing
* 사용자 정의 XML Import

### 대표 사용 예시

```java
Digester digester = new Digester();

digester.addObjectCreate(
    "employees/employee",
    Employee.class);

digester.addBeanPropertySetter(
    "employees/employee/name",
    "name");

Employee employee =
    (Employee) digester.parse(xmlFile);
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 제외

### 전환 사유

Commons Digester는 전자정부프레임워크와 직접적인 연관이 없는 범용 Apache 라이브러리이다.

eGovFrame 4.3으로 전환하더라도 Digester 자체를 변경해야 하는 요구사항은 없다.

다만 다음과 같은 경우에는 검토가 필요하다.

* XML 기반 설정을 사용하는 경우
* JDK 17 환경에서 동작 검증 필요
* Commons Digester 1.8의 유지보수 종료
* 보안 취약점 점검 필요

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 낮음 |
| 설정파일    | 없음 |
| pom.xml | 일부 |

## 7. 위험요소

### 보안

Commons Digester 1.8은 매우 오래된 버전으로 알려진 XML 관련 취약점 영향 가능성이 있다.

특히 아래 항목 검토 필요

* XXE(XML External Entity) 공격
* XML Injection
* 오래된 Commons 계열 라이브러리 취약점

### 운영

* 유지보수 종료 버전
* 최신 JDK 환경 검증 부족

### 호환성

| 항목            | 영향    |
| ------------- | ----- |
| JDK 17        | 검증 필요 |
| Spring 5.3    | 영향 없음 |
| eGovFrame 4.3 | 영향 없음 |

### 비고

* 전자정부프레임워크 4.3 전환과 직접 관련된 라이브러리는 아니다.
* 실제 소스에서 Digester를 사용하지 않는다면 제거 후보로 검토 가능하다.
* Digester 1.8은 2007년경 릴리즈된 매우 오래된 버전이므로 JDK 17 기반 운영환경에서는 최신 버전(2.1) 업그레이드를 권장한다.
* Struts, Tomcat 커스텀 설정 파싱 등에서 간접적으로 사용될 수 있으므로 의존성 추적(`mvn dependency:tree`)을 통한 확인이 필요하다.
