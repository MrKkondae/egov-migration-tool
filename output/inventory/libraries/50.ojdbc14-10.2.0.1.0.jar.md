## 2. 라이브러리 개요

### 주요 기능

* Java 애플리케이션과 Oracle Database 간 JDBC 연결 제공
* SQL 실행, ResultSet 처리, PreparedStatement/CallableStatement 지원
* Oracle 전용 데이터 타입, CLOB/BLOB, 날짜/시간 타입 처리
* Spring JDBC, MyBatis, iBATIS 등 Persistence 계층의 DB 연결 기반 제공

### 사용 목적

`ojdbc14-10.2.0.1.0.jar`는 Oracle Database 10g 계열에서 사용되던 구형 Oracle JDBC 드라이버이다.  
파일명상 JDK 1.4 시대의 `ojdbc14` 계열로 판단되며, eGovFrame 4.3 전환 시점의 JDK 1.8+ 환경에는 적합하지 않다.

전자정부프레임워크 4.3 전환 시에는 JDK 8 기준의 Oracle JDBC 드라이버인 `ojdbc8` 계열로 단일화하는 것이 권장된다.

### 대표 사용 예시

```java
Class.forName("oracle.jdbc.OracleDriver");

Connection conn = DriverManager.getConnection(
    "jdbc:oracle:thin:@//dbhost:1521/ORCL",
    "username",
    "password"
);
```

Spring 또는 MyBatis 환경에서는 직접 `DriverManager`를 호출하기보다 DataSource 설정에서 Oracle JDBC 드라이버를 지정하는 경우가 많다.

```xml
<bean id="dataSource" class="org.apache.commons.dbcp2.BasicDataSource">
    <property name="driverClassName" value="oracle.jdbc.OracleDriver"/>
    <property name="url" value="jdbc:oracle:thin:@//dbhost:1521/ORCL"/>
    <property name="username" value="${db.username}"/>
    <property name="password" value="${db.password}"/>
</bean>
```

## 3. AS-IS 사용 현황

### 프로젝트 내 사용 여부

* 확인 필요
* Oracle DB를 사용하는 경우 실제 운영 필수 가능성이 높음
* `jdbc-14.jar`와 중복 배치 가능성 있음

### 확인 방법

#### JAR 파일 중복 확인

```bash
find . -name "*ojdbc*.jar" -o -name "jdbc-14.jar"
find . -name "*.jar" | grep -Ei "ojdbc|oracle|jdbc"
```

#### Manifest 확인

```bash
jar xf ojdbc14-10.2.0.1.0.jar META-INF/MANIFEST.MF
cat META-INF/MANIFEST.MF
```

#### 포함 클래스 확인

```bash
jar tf ojdbc14-10.2.0.1.0.jar | grep "oracle/jdbc/OracleDriver"
jar tf jdbc-14.jar | grep "oracle/jdbc/OracleDriver"
```

#### 설정파일 확인

```bash
grep -R "oracle.jdbc.driver.OracleDriver" .
grep -R "oracle.jdbc.OracleDriver" .
grep -R "jdbc:oracle:thin" .
grep -R "ojdbc" .
```

#### Maven 의존성 확인

```bash
mvn dependency:tree | grep -i ojdbc
mvn dependency:tree | grep -i oracle
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`ojdbc14-10.2.0.1.0.jar`는 구형 Oracle JDBC 드라이버이며, eGovFrame 4.3의 JDK 1.8+ 기반 환경에서는 `ojdbc8` 계열로 정리하는 것이 적절하다.

또한 Matrix의 36번 `jdbc-14.jar` 항목과 중복 가능성이 크므로 Oracle JDBC 드라이버는 하나의 표준 버전으로 통일해야 한다.  
동일 애플리케이션 또는 WAS 공통 lib에 여러 Oracle JDBC 드라이버가 동시에 존재하면 클래스 로딩 순서에 따라 예기치 않은 DB 연결 오류, 날짜/시간 타입 처리 차이, CLOB/BLOB 처리 오류가 발생할 수 있다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 대부분 없음. 단, Oracle 전용 클래스 직접 사용 시 일부 수정 가능 |
| 설정파일    | DataSource driverClassName, JDBC URL, WAS datasource 설정 확인 필요 |
| pom.xml | `ojdbc14` 또는 수동 lib 의존성을 `ojdbc8`로 대체 필요 |
| 운영환경    | WAS 공통 lib, JDBC Provider, DB 접속 테스트, 폐쇄망 반입 라이브러리 확인 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `ojdbc14-10.2.0.1.0.jar` 제거
* `jdbc-14.jar`와 중복 여부 확인
* Oracle JDBC 드라이버를 `ojdbc8` 단일 버전으로 표준화
* WAS 공통 lib와 애플리케이션 `WEB-INF/lib` 중복 배치 제거
* Maven/Gradle 또는 폐쇄망 반입 라이브러리 기준 정리
* DB 서버 버전과 Oracle JDBC 드라이버 호환성 확인
* `oracle.jdbc.driver.OracleDriver` 사용 시 `oracle.jdbc.OracleDriver`로 정리 검토
* CLOB/BLOB, 날짜/시간, NUMBER, CHAR/VARCHAR2, NLS 문자셋 처리 회귀 테스트
* Spring Transaction, MyBatis/iBATIS, 배치 처리 DB 연결 테스트
* 운영 DB 계정, TNS, 서비스명/SID, RAC/SCAN URL 구성 확인

### 자동 변환 가능 여부

| ----------- | -------- |

* `jdbc:oracle:thin` 설정 위치 목록화

### 수동 검토 필요 범위

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
ojdbc14-10.2.0.1.0.jar
```

### TO-BE 버전

```text
com.oracle.database.jdbc:ojdbc8
또는
com.oracle.ojdbc:ojdbc8
```

> 실제 Maven 좌표는 프로젝트 표준 저장소, 폐쇄망 반입 기준, DBMS 버전, 보안검증 결과에 맞춰 확정한다.

### 주요 변경사항

* JDK 1.4 시대 드라이버에서 JDK 8 호환 드라이버로 전환
* JDBC 4.x 기반 기능 및 최신 Oracle DB 호환성 확보
* DB 드라이버 중복 제거
* `jdbc-14.jar`와 `ojdbc14` 중복 가능성 해소
* WAS 공통 lib와 애플리케이션 lib 배치 기준 정리

### 소스 수정 여부

* 대부분 없음
* 다음 경우 일부 수정 필요 가능
  * `oracle.sql.*` 등 Oracle 내부/전용 API 직접 사용
  * 구형 `oracle.jdbc.driver.OracleDriver` 사용
  * CLOB/BLOB을 구형 Oracle API로 직접 처리
  * 날짜/시간 타입 변환을 드라이버 동작에 의존
  * Stored Procedure, REF CURSOR, ARRAY, STRUCT 등 Oracle 확장 타입 사용

## 7. 위험요소

### 보안

* 구형 DB 드라이버는 보안 패치 및 최신 JDK 호환성 측면에서 위험이 있다.
* Oracle JDBC 드라이버는 일반 오픈소스 라이선스가 아니므로 배포 라이선스와 폐쇄망 반입 기준 확인이 필요하다.
* DB 접속 계정, 접속 문자열, Wallet, TLS 사용 여부가 설정파일에 평문으로 남아 있는지 확인해야 한다.

### 운영

* WAS 공통 lib와 애플리케이션 lib에 서로 다른 Oracle JDBC 드라이버가 중복 배치되면 클래스 로딩 충돌 가능성이 있다.
* 운영 DB 버전이 낮거나 특정 패치셋에 종속된 경우 최신 드라이버 적용 후 SQL 실행 결과 또는 타입 매핑 차이가 발생할 수 있다.
* CLOB/BLOB, 날짜/시간, NLS 문자셋, 프로시저 호출, 대량 배치 처리에서 회귀 테스트가 필요하다.
* RAC, SCAN, 서비스명/SID 방식에 따라 JDBC URL 검증이 필요하다.

### 호환성

| 항목 | 영향 |
| ---- | ---- |
| JDK 8 | `ojdbc8` 계열 적용 권장 |
| Spring 5.3 | DataSource 설정 및 트랜잭션 동작 검증 필요 |
| eGovFrame 4.3 | JDK 1.8+ 기준에 맞춰 구형 드라이버 대체 필요 |
| MyBatis/iBATIS | SQL 실행 결과, 타입 핸들러, 프로시저 호출 검증 필요 |
| WAS | 공통 lib/JDBC Provider 중복 확인 필요 |
| Oracle DB | DB 버전과 JDBC 드라이버 호환성 확인 필요 |

### 비고

* 36번 `jdbc-14.jar`와 함께 반드시 통합 검토해야 한다.
* Oracle JDBC 드라이버는 애플리케이션에 여러 버전을 동시에 배치하지 않고 하나의 표준 버전으로 통일하는 것이 바람직하다.
* 프로젝트가 Maven을 사용하지 않고 `WEB-INF/lib` 또는 WAS 공통 lib 방식으로 운영되는 경우, 실제 반입 JAR 파일명과 Manifest 정보를 기준으로 최종 판단해야 한다.
* DB 서버 버전, WAS 종류, JDK 버전, 운영환경 JDBC Provider 정책에 따라 최종 `ojdbc8` 세부 버전을 확정해야 한다.
* 라이선스는 Oracle 배포 조건을 따르므로 공공기관 오픈소스/상용SW 반입 절차에 따라 별도 확인이 필요하다.
