## 2. 라이브러리 개요

### 주요 기능

* MyBatis와 Spring Framework 연계 지원
* `SqlSessionFactoryBean`을 통한 MyBatis `SqlSessionFactory` 생성
* `SqlSessionTemplate` 기반 Thread-safe SQL 실행 지원
* Mapper Interface 자동 스캔 및 Bean 등록 지원
* Spring 트랜잭션 관리와 MyBatis SQL 실행 연계

### 사용 목적

MyBatis-Spring은 MyBatis SQL Mapper를 Spring 애플리케이션에서 사용하기 위한 연계 라이브러리이다.

기존 eGovFrame 3.1 기반 시스템에서는 MyBatis 3.2.x와 MyBatis-Spring 1.2.x 조합으로 DAO 또는 Mapper 계층을 구성했을 가능성이 있다. eGovFrame 4.3은 Spring Framework 5.3.x 기반이므로 MyBatis-Spring도 Spring 5와 호환되는 2.1.x 계열로 버전업하는 것이 적절하다.

본 항목은 48번 `mybatis-3.2.7.jar`와 세트로 검토해야 하며, 26번 `ibatis-sqlmap-2.3.4.726.jar` 잔존 영역을 MyBatis로 통합 전환할 때에도 함께 영향을 받는다.

### 대표 사용 예시

```xml
<!-- AS-IS 예시: Spring XML 기반 MyBatis 설정 -->
<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <property name="dataSource" ref="dataSource" />
    <property name="configLocation" value="classpath:/sqlmap/mybatis-config.xml" />
    <property name="mapperLocations" value="classpath:/sqlmap/mappers/**/*.xml" />
</bean>

<bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
    <property name="basePackage" value="egovframework.example.mapper" />
</bean>
```

```java
import org.mybatis.spring.SqlSessionTemplate;
import org.springframework.stereotype.Repository;

@Repository
public class SampleDAO {

    private final SqlSessionTemplate sqlSession;

    public SampleDAO(SqlSessionTemplate sqlSession) {
        this.sqlSession = sqlSession;
    }

    public Object selectSample(Object param) {
        return sqlSession.selectOne("sample.selectSample", param);
    }
}
```

## 3. AS-IS 사용 현황

### 확인 방법

#### JAR 및 Maven 의존성 확인

```bash
find . -name "mybatis-spring-*.jar"
grep -R "mybatis-spring" .
mvn dependency:tree -Dincludes=org.mybatis:mybatis-spring
```

#### Spring 설정 확인

```bash
grep -R "SqlSessionFactoryBean" src/main/resources
grep -R "SqlSessionTemplate" src/main/java src/main/resources
grep -R "MapperScannerConfigurer" src/main/resources
grep -R "org.mybatis.spring" src/main/java src/main/resources
```

#### Mapper Interface 확인

```bash
grep -R "@Mapper" src/main/java
grep -R "namespace=" src/main/resources | grep -i mapper
find src/main/resources -name "*Mapper.xml" -o -name "*Mapper*.xml"
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상
* 버전업 대상

### 전환 사유

`mybatis-spring-1.2.2.jar`는 Spring 3.x/4.x 시절에 사용되던 구버전 MyBatis-Spring 연계 라이브러리이다.

eGovFrame 4.3 실행환경은 Spring Framework 5.3.x 기반이며, MyBatis도 3.5.x 계열로 상향된다. 따라서 MyBatis-Spring 역시 Spring 5와 MyBatis 3.5 이상을 지원하는 2.1.x 계열로 버전업하는 것이 필요하다.

Matrix 기준으로도 49번 항목은 `mybatis-spring-2.1.x`로 버전업하고, `SqlSessionFactoryBean`, `MapperScannerConfigurer` 설정을 확인하는 것으로 정리되어 있다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 대부분 영향 없음. 단, `SqlSessionDaoSupport`, `SqlSessionTemplate`, Mapper Interface 사용 방식에 따라 일부 수정 가능 |
| 설정파일    | 영향 있음. `SqlSessionFactoryBean`, `MapperScannerConfigurer`, `mapperLocations`, `typeAliasesPackage` 등 확인 필요 |
| pom.xml | 영향 있음. `org.mybatis:mybatis-spring` 버전 변경 및 MyBatis 본체 버전 정합성 필요 |
| 운영환경    | 영향 있음. DB 연결, 트랜잭션, Mapper 스캔, SQL 실행 결과 회귀 테스트 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `mybatis-spring-1.2.2.jar`를 `mybatis-spring-2.1.x` 계열로 버전업
* 48번 `mybatis-3.2.7.jar`를 MyBatis 3.5.x 계열로 함께 버전업
* eGovFrame 4.3의 `org.egovframe.rte.psl.dataaccess` 구성과 충돌 여부 확인
* `SqlSessionFactoryBean` 설정 확인
* `MapperScannerConfigurer` 또는 Mapper scan 설정 확인
* `SqlSessionTemplate` Bean 중복 정의 여부 확인
* Spring 트랜잭션 설정과 MyBatis SQL 실행 연계 확인
* iBATIS 잔존 영역이 있으면 26번 항목과 연계하여 통합 전환 기준 수립

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 부분 가능 |
| Python Rule | 가능 |
| LLM 보정      | 필요 |

### 예상 난이도

* 중간

단순 버전업만 수행하는 경우 난이도는 중간 이하이다. 그러나 iBATIS 잔존 영역을 MyBatis로 통합하거나, DAO 구조를 `EgovAbstractMapper`/Mapper Interface 중심으로 재정리하는 경우 난이도는 중간 이상으로 상승할 수 있다.

## 7. 위험요소

### 보안

* MyBatis-Spring 자체보다는 MyBatis 본체, JDBC Driver, DB SQL 사용 방식의 보안 영향이 더 큼
* SQL Injection 방지를 위해 `${}` 사용 SQL 확인 필요
* 동적 SQL, 문자열 결합 SQL, LIKE 조건 처리 방식 검토 필요
* 구버전 MyBatis/MyBatis-Spring 조합 유지 시 보안점검 및 취약점 진단에서 지적될 가능성 있음

### 운영

* Mapper XML 경로 누락 시 애플리케이션 기동 오류 발생 가능
* Mapper Interface와 XML namespace 불일치 시 런타임 오류 발생 가능
* 트랜잭션 설정 오류 시 commit/rollback 동작 이상 가능
* MyBatis 본체와 MyBatis-Spring 버전 불일치 시 Bean 생성 오류 가능
* WAS 공통 lib와 애플리케이션 lib 중복 배치 시 클래스 충돌 가능

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 8           | MyBatis-Spring 2.1.x 적용 가능 |
| Spring 5.3.x    | MyBatis-Spring 2.1.x 필요 |
| eGovFrame 4.3   | `org.egovframe.rte.psl.dataaccess`와 설정 정합성 확인 필요 |
| MyBatis 3.5.x   | MyBatis-Spring 2.1.x와 세트 구성 필요 |
| Spring Batch    | Batch DAO 사용 시 Spring Batch 4.x 조합 확인 필요 |

### 비고

* 49번 항목은 48번 `mybatis-3.2.7.jar`와 세트로 전환해야 한다.
* 26번 `ibatis-sqlmap-2.3.4.726.jar`가 잔존하는 경우 MyBatis 통합 전환 기준을 함께 수립해야 한다.
* eGovFrame 4.3에서는 Spring Framework 5.3.x 기반이므로 MyBatis-Spring 1.2.2 유지보다는 2.1.x 계열 전환이 적절하다.
* XML 기반 설정을 유지하더라도 `SqlSessionFactoryBean`, `MapperScannerConfigurer`, `mapperLocations`, `typeAliasesPackage`, TransactionManager 설정을 모두 확인해야 한다.
* Mapper XML namespace와 Mapper Interface FQCN이 일치하는지 확인해야 한다.
* `${}` 기반 SQL 문자열 치환은 SQL Injection 위험이 있으므로 전환 작업과 별개로 보안 점검 대상에 포함한다.
