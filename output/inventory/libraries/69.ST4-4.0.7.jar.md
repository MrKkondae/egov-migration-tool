## 2. 라이브러리 개요

### 주요 기능

* 문자열 템플릿 생성
* HTML 템플릿 생성
* 코드 생성(Code Generation)
* 문서 생성
* 동적 문자열 치환

### 사용 목적

`ST4-4.0.7.jar`는 ANTLR 프로젝트에서 제공하는 StringTemplate 엔진이다.

주로 다음과 같은 목적으로 사용된다.

* 코드 자동 생성
* SQL 자동 생성
* HTML 생성
* XML 생성
* 보고서 생성
* 템플릿 기반 문자열 처리

전자정부프레임워크 자체에서 사용하는 핵심 라이브러리는 아니며, 특정 솔루션이나 개발도구가 내부적으로 사용하는 경우가 많다.

### 대표 사용 예시

```java id="4d6ztp"
ST hello =
    new ST("Hello <name>");

hello.add("name", "eGov");

String result =
    hello.render();
```

결과

```text id="4ik4qk"
Hello eGov
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 검토 대상

### 전환 사유

`ST4-4.0.7.jar`는 Spring Framework 또는 eGovFrame Runtime 라이브러리가 아니다.

따라서 eGovFrame 3.x → 4.3 전환 시 직접적인 전환 대상은 아니다.

다만 다음과 같은 경우 검토가 필요하다.

* 실제 업무 기능에서 사용 중인 경우
* ANTLR 기반 기능과 연계된 경우
* 코드 생성 기능에서 사용하는 경우
* StringTemplate API 의존성이 존재하는 경우

일반적인 전자정부 업무 시스템에서는 사용 빈도가 높지 않다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 낮음 |
| 설정파일    | 없음 |
| pom.xml | 낮음 |
| 운영환경    | 낮음 |

### 비고

* eGovFrame 4.3 전환과 직접적인 관련이 없는 라이브러리이다.
* Spring Framework 계열 라이브러리처럼 반드시 전환해야 하는 대상은 아니다.
* 실제 프로젝트에서 `org.stringtemplate.*` 패키지 사용 여부를 확인하는 것이 우선이다.
* 사용 흔적이 없는 경우 제거 후보로 분류 가능하다.
* 전자정부프레임워크 전환 관점에서는 **"유지 검토 대상"** 으로 분류하는 것이 적절하다.
* 사용자의 eGovFrame 3.x → 4.3 전환 프로젝트 기준으로는 영향도가 매우 낮은 라이브러리에 해당한다.
