## 2. 라이브러리 개요

### 주요 기능

* 인증서 기반 사용자 인증 또는 전자서명 연계 추정
* GPKI, 공인/사설 인증서, 인증서 로그인 연계 가능성
* 클라이언트 인증서 검증, 인증서 선택, 인증서 정보 추출 기능 가능성
* ActiveX 제거 또는 Non-ActiveX 인증 모듈과의 연계 영향 가능성

### 사용 목적

`jcaos-client-arccert-1.4.7.1.jar`는 공개 Maven Repository에서 일반적으로 확인되는 범용 오픈소스 라이브러리가 아니라, 인증서 또는 GPKI 연계용 상용/기관 특화 클라이언트 모듈로 추정된다.

파일명에 `client`, `arccert`, `cert`가 포함되어 있으므로 인증서 로그인, 인증서 검증, 전자서명, GPKI 또는 사설 인증체계 연계에 사용되었을 가능성이 있다. 다만 제조사, 공식 문서, Maven 좌표, 라이선스가 명확히 확인되지 않으므로 임의 버전업이나 단순 제거는 금지하고 공급사 확인 및 업무 사용처 분석을 선행해야 한다.

### 대표 사용 예시

실제 API는 공급사 문서 확인 전까지 확정할 수 없다. 아래는 사용처 탐색을 위한 예시 수준이다.

```java
// 실제 클래스명은 JAR 내부 패키지 확인 필요
// 예: 인증서 로그인, 인증서 검증, 인증서 정보 추출 등
// import com.vendor.cert.*;

public class CertificateLoginService {
    public boolean verifyCertificate(byte[] certData) {
        // 공급사 API 기반 인증서 검증 처리 추정
        return false;
    }
}
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 검토 대상

### 전환 사유

전자정부프레임워크 4.3 전환 시 JDK, Spring, WAS, 브라우저, ActiveX 제거 방식, 인증서 모듈 구성이 함께 변경될 수 있다. 인증서 모듈은 업무 로그인, 전자서명, 기관 연계, 보안 정책과 직결되므로 단순 컴파일 호환성만으로 판단할 수 없다.

특히 `jcaos-client-arccert-1.4.7.1.jar`는 공개 오픈소스 라이브러리로 식별되지 않으며, 공급사 제공 파일일 가능성이 높다. 따라서 eGovFrame 4.3 환경에서 사용할 수 있는 공급사 최신 JDK8/WAS 호환 버전이 있는지 확인해야 한다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 인증서 API 직접 호출 시 영향 높음 |
| 설정파일    | 인증서 경로, 라이선스, 서버 URL, Provider, 알고리즘 설정 영향 가능 |
| pom.xml | Maven Central 의존성 전환 불가 가능성 높음. 로컬/사내 Nexus 반입 관리 필요 |
| 운영환경    | 인증서 저장소, GPKI 모듈, 폐쇄망 반입, WAS 공통 lib, 라이선스 파일 영향 가능 |

### 주요 변경사항

* 공개 표준 라이브러리로 단순 대체하기 어려움
* 공급사 API, 라이선스, 설치 방식, Native 모듈 포함 여부 확인 필요
* 인증서 저장소 접근 방식 변경 가능성 있음
* ActiveX 기반 클라이언트 모듈과 연동되어 있었다면 웹표준/Non-ActiveX 방식으로 재설계 필요 가능

### 소스 수정 여부

* 사용처 없음: 제거 가능
* 단순 유틸성 참조: 일부 수정 필요
* 로그인/전자서명 핵심 기능 사용: 대규모 수정 가능
* ActiveX 연계 사용: 대체 솔루션 연계 및 화면/스크립트 수정 가능

## 7. 위험요소

### 보안

* 인증서/전자서명 모듈은 사용자 인증과 직접 연결되므로 보안 영향도 높음
* 공급사 미확인 JAR 또는 오래된 버전 사용 시 보안 취약점 확인이 어려움
* 인증서 검증 로직 오류는 우회 인증, 위변조, 폐지 인증서 허용 등으로 이어질 수 있음
* CRL/OCSP, 인증서 체인 검증, 알고리즘 정책, 키 저장소 접근 권한 확인 필요

### 운영

* 폐쇄망 반입 승인 필요 가능
* 운영 WAS 공통 lib 또는 애플리케이션 lib 중복 배치 시 충돌 가능
* 라이선스 파일, Native DLL/SO, 브라우저 플러그인, 보안 프로그램 설치 여부 확인 필요
* 운영 인증기관, GPKI 서버, 검증 서버, 방화벽 정책과 연계 가능
* 인증서 만료/폐지/갱신 시 운영 장애 가능성 있음

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 1.8         | 공급사 호환 버전 확인 필요 |
| Spring 5.3      | 직접 영향은 낮을 수 있으나 Bean 등록/Service 연계 시 확인 필요 |
| eGovFrame 4.3   | 인증/보안 공통모듈 연계 여부 확인 필요 |
| WAS             | JEUS, WebLogic, Tomcat 등 대상 WAS 호환성 확인 필요 |
| 브라우저        | ActiveX 제거 및 Non-ActiveX 인증 방식 영향 가능 |
| 폐쇄망 운영     | 반입 파일, 라이선스, 인증서 검증 서버 통신 확인 필요 |

### 비고

* 본 JAR는 공개 Maven Repository 기준으로 일반 오픈소스 라이브러리로 식별하기 어렵다.
