## 2. 라이브러리 개요

### 주요 기능

* AspectJ AOP Weaving 수행
* Spring AOP 지원
* Load-Time Weaving(LTW) 지원
* Cross-Cutting Concern 분리 구현

### 사용 목적

AspectJ는 AOP(Aspect Oriented Programming)를 구현하기 위한 프레임워크이다.

공통 기능을 비즈니스 로직과 분리하여 적용할 수 있도록 지원하며 주로 다음 기능에 활용된다.

* 트랜잭션 처리
* 로깅
* 보안
* 성능 측정
* 예외 처리

AspectJ 관련 주요 구성요소는 다음과 같다.

| 라이브러리         | 역할             |
| ------------- | -------------- |
| aspectjrt     | Runtime        |
| aspectjtools  | Compiler       |
| aspectjweaver | Weaving Engine |

### 대표 사용 예시

Spring AOP 설정

```xml id="6z2n0h"
<aop:aspectj-autoproxy />
```

Load-Time Weaving 설정

```xml id="jp3fws"
<context:load-time-weaver />
```

Java Aspect 예시

```java id="n9w4y2"
@Aspect
@Component
public class LoggingAspect {

    @Before(
      "execution(* com.sample..*(..))")
    public void before() {
        System.out.println("Before");
    }
}
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상 (버전 업그레이드)

### 전환 사유

전자정부프레임워크 4.3 기본 구성에서는 AspectJ Weaver 버전이 상향되었다.

| 구분            | 버전    |
| ------------- | ----- |
| AS-IS         | 1.8.0 |
| eGovFrame 4.3 | 1.9.7 |

따라서 전환 시 eGovFrame 4.3 표준 버전에 맞추어 업그레이드하는 것을 권장한다.

### 영향 범위

| 구분      | 영향          |
| ------- | ----------- |
| 소스      | 거의 없음       |
| 설정파일    | LTW 사용 시 검토 |
| pom.xml | 버전 변경       |
| 운영환경    | 없음          |

### 비고

* 현재 확인된 내용으로는 LTW(Load-Time Weaving) 사용 가능성이 낮다.
* 일반적인 Spring AOP 환경에서는 단순 버전 업그레이드만으로 전환 가능하다.
* `@Aspect`, `load-time-weaver`, `-javaagent` 사용 여부를 최종 확인하는 것이 바람직하다.
* Migration Matrix 기준 분류는 **Version Upgrade** 이다.
* 권장 TO-BE 버전은 전자정부프레임워크 4.3 기본 구성인 **1.9.7** 이다.
