# 2. 라이브러리 개요

## 주요 기능

* SLF4J 기반 로깅 추상화 지원
* Log4j 연동 기능 제공
* eGovFrame 공통 로깅 설정 지원
* AOP 기반 Logging 기능 지원
* 시스템 로그 표준화

## 사용 목적

전자정부프레임워크 3.x 환경에서 애플리케이션 로그를 표준화하고 Log4j, SLF4J 등의 로깅 프레임워크를 연계하기 위해 사용된다.

주로 다음과 같은 환경에서 사용된다.

* Service 로그 출력
* DAO SQL 로그 출력
* Batch 로그 출력
* AOP 기반 실행 이력 기록

## 대표 사용 예시

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class UserService {

    private static final Logger LOGGER =
        LoggerFactory.getLogger(UserService.class);

    public void insertUser() {
        LOGGER.info("사용자 등록");
    }
}
```

# 4. eGovFrame 4.3 영향도 분석

## 전환 대상 여부

**제외**

## 전환 사유

본 라이브러리는 업무 로직 라이브러리가 아니라 전자정부프레임워크 내부 공통 로깅 모듈이다.

eGovFrame 4.3에서도 동일한 기능이 제공되므로 별도 소스 변환 대상은 아니다.

다만 3.1 버전 JAR은 제거하고 4.3 의존성으로 교체해야 한다.

## 영향 범위

| 구분      | 영향    |

| 소스      | 거의 없음 |
| 설정파일    | 일부    |
| pom.xml | 있음    |
| 운영환경    | 있음    |

# 5. 전환 시 검토사항

## 변경 필요 사항

### 1) eGovFrame 3.1 Logging 제거

AS-IS

```xml
<dependency>
    <groupId>egovframework.rte.fdl</groupId>
    <artifactId>egovframework.rte.fdl.logging</artifactId>
    <version>3.1.0</version>
</dependency>
```

TO-BE

```xml
<dependency>
    <groupId>org.egovframe.rte</groupId>
    <artifactId>org.egovframe.rte.fdl.logging</artifactId>
    <version>4.3.0</version>
</dependency>
```

---

### 2) Log4j 1.x 사용 여부 확인

eGovFrame 3.1 시절에는 다음 구성이 많이 사용되었다.

```text
log4j-1.2.x
slf4j-log4j12
```

전자정부프레임워크 4.3에서는 Log4j 1.x 사용을 권장하지 않는다.

확인 필요

```bash
find . -name "log4j.xml"
find . -name "log4j.properties"
```

---

### 3) Logback 사용 여부 검토

Spring 5.x 기반에서는 일반적으로

```text
SLF4J + Logback
```

구성이 권장된다.

# 7. 위험요소

## 보안

Log4j 1.x 사용 시 보안 위험 존재

확인 필요

```text
log4j-1.2.x
slf4j-log4j12
```

특히 Log4j 1.x는 공식 지원 종료(EOL) 상태이다.

## 운영

운영 로그 정책 변경 시 로그 포맷 차이 발생 가능

## 비고

* 본 라이브러리는 전자정부프레임워크 공통 로깅 모듈이다.
* 업무 소스에서 직접 사용하는 경우는 거의 없다.
* 실제 전환 시 핵심 검토 대상은 **Log4j 1.x → Logback 전환 여부**이다.
* 프로젝트에서 `log4j.xml`, `log4j.properties`, `slf4j-log4j12` 의존성이 존재하는 경우 별도 분석이 필요하다.
* 라이브러리 자체는 전환 대상이 아니라 **4.3 버전으로 교체하는 업그레이드 대상**으로 분류하는 것이 적절하다.
