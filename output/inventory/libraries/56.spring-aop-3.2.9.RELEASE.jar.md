## 2. 라이브러리 개요

### 주요 기능

* AOP 기반 공통 관심사 분리
* Method Interceptor 처리
* Pointcut 및 Advice 처리
* Transaction, Logging, Security 등 공통 기능 적용
* Spring Bean 기반 프록시 생성 지원

### 사용 목적

`spring-aop`는 Spring Framework에서 AOP 기능을 제공하는 라이브러리이다.
업무 로직과 공통 기능을 분리하여 트랜잭션, 로깅, 권한, 예외처리 등의 횡단 관심사를 적용하는 데 사용된다.

Spring 공식 문서에서도 Spring Framework는 Java 엔터프라이즈 프로그래밍에서 AOP 요구사항을 처리하기 위한 자체 AOP 프레임워크를 제공한다고 설명한다. ([Home][1])

### 대표 사용 예시

```java
@Aspect
@Component
public class LoggingAspect {

    @Before("execution(* egovframework..service.*Service.*(..))")
    public void beforeLog() {
        System.out.println("service method start");
    }
}
```

또는 XML 설정에서 사용된다.

```xml
<aop:config>
    <aop:pointcut id="serviceMethod"
        expression="execution(* egovframework..service.*Service.*(..))" />

    <aop:advisor advice-ref="txAdvice"
        pointcut-ref="serviceMethod" />
</aop:config>
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-aop-3.2.9.RELEASE.jar`는 Spring Framework 3.2 계열 라이브러리로, eGovFrame 4.3의 Spring Framework 기준 버전과 맞지 않는다.

eGovFrame 4.3.0 안내에서는 Spring Framework가 5.3.27에서 5.3.37로 업데이트되었고, 기존 eGovFrame 3.x/2.x/1.x 실행환경과 호환되지 않으며 JDK 1.8 이상이 필요하다고 안내한다. ([eGovFrame][2])

따라서 `spring-aop`만 단독으로 유지할 수 없으며 `spring-core`, `spring-beans`, `spring-context`, `spring-expression`, `spring-tx` 등 Spring 계열 라이브러리와 함께 5.3.37 기준으로 정합성을 맞춰야 한다.

### 영향 범위

| 구분      | 영향 |

| 소스      | 중간 |
| 설정파일    | 중간 |
| pom.xml | 높음 |

## 7. 위험요소

### 보안

* Spring 3.2 계열은 노후 버전으로 최신 보안 패치 적용이 어렵다.
* Spring 계열 전체 라이브러리의 공개 취약점 및 기술지원 종료 여부 확인이 필요하다.

### 운영

* AOP 설정 오류 시 트랜잭션, 로깅, 권한, 예외처리 기능이 정상 동작하지 않을 수 있다.
* Spring 라이브러리 버전 혼용 시 `NoSuchMethodError`, `ClassNotFoundException`, `BeanCreationException`이 발생할 수 있다.
* ApplicationContext 초기화 실패 가능성이 있다.

### 호환성

* JDK 영향 있음
* Spring 영향 높음
* eGovFrame 영향 높음

##### (15) 최종 정리

```text
spring-aop 3.2.9.RELEASE → 5.3.37 전환은 Spring Framework 계열 버전 정합성 확보가 핵심이다.

일반적인 @Aspect, @Before, @After, @Around, ProceedingJoinPoint 사용 코드는 대부분 그대로 유지 가능하다.

OpenRewrite를 통해 Maven 의존성 변경, 구버전 Jar 탐지,
Spring Framework 계열 라이브러리 버전 통일, XML Schema Location 정리는 자동화할 수 있다.

다만 ProxyFactoryBean, BeanNameAutoProxyCreator, Load-Time Weaving,
AspectJ Weaver 설정은 프로젝트별 사용 방식에 따라 수동 검토가 필요하다.

전환 후에는 주요 Service Bean에 AOP Advice가 정상 적용되는지,
Transaction Advice와 충돌이 없는지 반드시 검증해야 한다.
```
