## 2. 라이브러리 개요

### 주요 기능

* Spring ApplicationContext 제공
* Bean 생성 및 관리 (IoC/DI)
* Component Scan 지원
* Event 처리 기능 제공
* MessageSource 제공
* Resource 로딩 지원

### 사용 목적

Spring Framework의 핵심 컨테이너 기능을 제공하는 라이브러리로서 애플리케이션 Bean 관리 및 의존성 주입을 수행한다.

### 대표 사용 예시

```java
ApplicationContext context =
    new ClassPathXmlApplicationContext(
        "egovframework/spring/context-common.xml");
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

eGovFrame 4.3은 Spring Framework 5.3.x 기반으로 구성되어 있으며, 기존 Spring 4.0.x 라이브러리는 버전 호환성 및 보안 측면에서 업그레이드가 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 낮음 |
| 설정파일    | 중간 |
| pom.xml | 높음 |
| 운영환경    | 낮음 |

---

## 5. 전환 시 검토사항

### 변경 필요 사항

* Spring 4.0.x → Spring 5.3.x 버전 업그레이드
* WEB-INF/lib 직접 참조 제거 검토
* Maven Dependency 정비
* Spring XML Schema 검증
* Bean 초기화 오류 검증

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | ----- |
| OpenRewrite | 가능    |
| Python Rule | 가능    |
| LLM 보정      | 필요    |

### 예상 난이도

* 중간

## 7. 위험요소

### 보안

* Spring 4.0.x 계열 기술지원 종료
* 공개 취약점(CVE) 다수 존재

### 운영

* Spring 라이브러리 버전 혼용 시 기동 오류 가능
* Bean 생성 오류 가능

### 호환성

* JDK 8 이상 필요
* Spring Core 계열 버전 일치 필요
* eGovFrame 4.3 표준 적용 필요
