## 2. 라이브러리 개요

### 주요 기능

* Spring Framework 핵심 기능 제공
* Resource 로딩 기능 제공
* Reflection 및 Utility 기능 제공
* 타입 변환(Type Conversion) 지원
* Spring Container 기반 클래스 제공

### 사용 목적

`spring-core`는 Spring Framework의 가장 핵심적인 라이브러리로 다른 Spring 모듈들이 공통적으로 사용하는 기반 기능을 제공한다.

`spring-beans`, `spring-context`, `spring-aop`, `spring-tx`, `spring-web` 등 대부분의 Spring 라이브러리가 `spring-core`에 의존한다.

전자정부프레임워크 기반 시스템에서는 Spring Runtime 전체의 기반 라이브러리 역할을 수행한다.

### 대표 사용 예시

```java
import org.springframework.core.io.ClassPathResource;

ClassPathResource resource =
    new ClassPathResource("config.properties");
```

또는

```java
import org.springframework.util.StringUtils;

String value =
    StringUtils.trimWhitespace(text);
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-core-3.2.9.RELEASE.jar`는 Spring Framework 3.2 계열의 핵심 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 Spring Core 계열 라이브러리 전체 업그레이드가 필요하다.

특히 `spring-core`는 Spring Framework 전체의 기반 모듈이므로 버전 불일치가 발생할 경우 ApplicationContext 초기화 실패, Bean 생성 오류, Class Loading 오류 등이 발생할 수 있다.

따라서 eGovFrame 4.3 전환 시 반드시 업그레이드해야 하는 핵심 라이브러리이다.

### 영향 범위

| 설정파일    | 중간 |

### 비고

* eGovFrame 4.3 전환 시 반드시 업그레이드해야 하는 최우선 핵심 라이브러리이다.
* `spring-core`는 Spring Framework 전체의 기반 모듈이므로 영향 범위가 매우 넓다.
* 단독 전환이 아닌 Spring Core 계열(`spring-beans`, `spring-context`, `spring-expression`, `spring-aop`, `spring-tx`)과 함께 일괄 전환해야 한다.
* 일반적인 `org.springframework.util.*`, `org.springframework.core.*` 사용 코드는 대부분 유지 가능하나 컴파일 및 기동 테스트는 반드시 수행해야 한다.
* 전환 후 ApplicationContext 기동 테스트, 주요 업무 기능 테스트, 트랜잭션 및 DI 동작 검증을 수행해야 한다.
