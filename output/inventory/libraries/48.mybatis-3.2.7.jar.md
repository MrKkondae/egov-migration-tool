## 2. 라이브러리 개요

### 주요 기능

* SQL Mapper 기반 데이터 접근 처리
* XML Mapper 또는 Annotation 기반 SQL 매핑
* Java 객체와 SQL 결과 간 매핑
* 동적 SQL 처리
* Stored Procedure 및 복잡 SQL 호출 지원

### 사용 목적

MyBatis는 Java 애플리케이션에서 JDBC 반복 코드를 줄이고 SQL과 Java 객체 간 매핑을 지원하는 Persistence Framework이다.

전자정부프레임워크 기반 업무시스템에서는 DAO 계층에서 SQL Mapper 역할로 사용되며, 기존 iBATIS를 대체하는 대표적인 라이브러리이다.

AS-IS의 `mybatis-3.2.7.jar`는 이미 MyBatis 3.x 계열이므로 iBATIS처럼 구조 전환 대상은 아니지만, eGovFrame 4.3 및 Spring 5 환경과의 호환성을 위해 MyBatis 3.5.x 계열로 버전업이 필요하다.

특히 26번 `ibatis-sqlmap-2.3.4.726.jar` 잔존 영역과 함께 통합 기준을 수립해야 한다.

### 대표 사용 예시

```java
@Repository("sampleMapper")
public class SampleMapper extends EgovAbstractMapper {

    public List<?> selectSampleList(SampleVO searchVO) {
        return selectList("sampleMapper.selectSampleList", searchVO);
    }
}
```

또는 Mapper Interface 방식으로 사용할 수 있다.

```java
@Mapper("sampleMapper")
public interface SampleMapper {
    List<SampleVO> selectSampleList(SampleVO searchVO);
}
```

## 3. AS-IS 사용 현황

### 프로젝트 내 사용 여부

### 확인 방법

#### JAR 및 Maven 의존성 확인

```bash
find . -name "mybatis-*.jar"
find . -name "mybatis-spring-*.jar"
mvn dependency:tree -Dincludes=org.mybatis
```

#### MyBatis 설정 확인

```bash
grep -R "SqlSessionFactoryBean" .
grep -R "MapperScannerConfigurer" .
grep -R "MapperConfigurer" .
grep -R "mapperLocations" .
grep -R "typeAliasesPackage" .
```

#### Mapper XML 확인

```bash
find . -name "*.xml" | xargs grep -n "<!DOCTYPE mapper"
find . -name "*.xml" | xargs grep -n "<mapper namespace"
find . -name "*.xml" | xargs grep -n "<select\|<insert\|<update\|<delete"
```

#### Java 소스 사용처 확인

```bash
grep -R "org.apache.ibatis" src
grep -R "SqlSession" src
grep -R "EgovAbstractMapper" src
grep -R "@Mapper" src
grep -R "selectList\|selectOne\|insert\|update\|delete" src
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상
* 버전업 대상

### 전환 사유

AS-IS의 `mybatis-3.2.7.jar`는 2014년 전후의 구버전 MyBatis 3.x 계열로, eGovFrame 4.3의 Spring 5 기반 환경에서는 `mybatis-spring-2.1.x`와 함께 MyBatis 3.5.x 계열로 맞추는 것이 적절하다.

MyBatis 자체 API는 비교적 하위 호환성이 유지되는 편이지만, 다음 항목은 영향 가능성이 있으므로 확인이 필요하다.

* `mybatis-spring-1.2.2.jar`와의 조합 변경
* `SqlSessionFactoryBean` 설정 변경
* Mapper XML DTD 및 namespace 구성
* TypeAlias, TypeHandler, Plugin 설정
* Executor, Cache, Lazy Loading, ResultMap 동작 차이
* iBATIS 잔존 영역과의 통합 전환 기준

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 대부분 없음 또는 일부 수정 필요. `SqlSession`, `EgovAbstractMapper`, Mapper Interface 사용 방식 확인 필요 |
| 설정파일    | `SqlSessionFactoryBean`, `mapperLocations`, `typeAliasesPackage`, `MapperConfigurer` 설정 확인 필요 |
| pom.xml | `org.mybatis:mybatis:3.5.x`, `org.mybatis:mybatis-spring:2.1.x` 계열로 버전 정합성 필요 |
| 운영환경    | DB 연결, 트랜잭션, SQL 실행 결과, 로그 출력, 배치 처리 회귀 테스트 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `mybatis-3.2.7.jar`를 MyBatis 3.5.x 계열로 버전업
* `mybatis-spring-1.2.2.jar`를 49번 항목 기준 `mybatis-spring-2.1.x` 계열로 세트 업그레이드
* eGovFrame 4.3의 `org.egovframe.rte.psl.dataaccess` 설정과 충돌 여부 확인
* iBATIS 잔존 SQL Map과 MyBatis Mapper XML의 통합 기준 수립
* Mapper XML, namespace, query id 중복 여부 확인
* `resultMap`, `parameterType`, `resultType`, `typeAlias`, `typeHandler` 동작 확인
* SQL 실행 결과 및 페이징/대량 조회 성능 회귀 테스트

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 부분 가능 |
| Python Rule | 가능 |
| LLM 보정      | 필요 |

### 자동화 가능 항목

* Maven/Gradle 의존성 버전 변경
* JAR 파일명 변경 목록 작성
* `org.apache.ibatis` import 사용처 목록화
* Mapper XML 목록화
* `SqlSessionFactoryBean`, `MapperConfigurer`, `mapperLocations` 설정 사용처 추출
* iBATIS와 MyBatis 혼재 여부 탐지

### 자동화 어려운 항목

* SQL 실행 결과 검증
* 동적 SQL 조건 누락 여부 검증
* ResultMap 매핑 차이 검증
* DBMS별 SQL 호환성 검증
* 트랜잭션 및 배치 처리 결과 검증

### 예상 난이도

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
org.mybatis:mybatis:3.2.7
```

### TO-BE 버전

```text
org.mybatis:mybatis:3.5.x
org.mybatis:mybatis-spring:2.1.x
```

### 권장 검토 후보

```text
org.mybatis:mybatis:3.5.16 이상 또는 eGovFrame 4.3 검증 버전
org.mybatis:mybatis-spring:2.1.2 또는 eGovFrame 4.3 검증 버전
```

단, 폐쇄망 반입 및 프로젝트 표준 라이브러리 목록이 별도로 확정되어 있다면 해당 기준을 우선한다.

### 주요 변경사항

* Spring 5 환경 호환을 위해 `mybatis-spring-2.1.x`와 함께 적용 필요
* Mapper Interface, SqlSession, EgovAbstractMapper 방식의 동작 확인 필요
* 기존 iBATIS 잔존 영역은 MyBatis 3.5.x 기준으로 통합 전환 필요
* MyBatis 3.5.x 계열에서 일부 Deprecated API 또는 내부 동작 변경 가능성 확인 필요
* TypeHandler, Plugin, Cache, Lazy Loading 등 확장 설정 사용 시 별도 검증 필요

### 소스 수정 여부

* MyBatis 3.x를 이미 사용 중인 영역: 대부분 없음 또는 일부 수정 필요
* iBATIS 잔존 영역과 통합하는 경우: 일부~대규모 수정 가능
* Mapper Interface로 표준화하는 경우: 일부 수정 필요

## 7. 위험요소

### 보안

* MyBatis 3.2.7은 구버전이므로 보안점검 도구에서 취약 라이브러리 또는 구버전 라이브러리로 식별될 가능성이 있다.
* SQL Injection 자체는 MyBatis 버전만으로 해결되지 않으며, `${}` 문자열 치환 사용 여부 확인이 필요하다.
* 사용자 입력값이 포함되는 SQL은 `#{}` 바인딩을 원칙으로 점검해야 한다.

### 운영

* Mapper XML의 SQL 실행 결과가 업무 기능과 직접 연결된다.
* 버전업 후 페이징, 동적 SQL, ResultMap, TypeHandler 처리 결과 차이가 발생할 수 있다.
* iBATIS와 MyBatis가 혼재된 경우 DAO 표준이 이원화되어 유지보수 위험이 증가한다.
* 운영 DBMS의 SQL 문법, Stored Procedure, CLOB/BLOB 처리, 날짜/숫자 타입 매핑 테스트가 필요하다.

### 호환성

| 항목              | 영향 |
| --------------- | ---- |
| JDK 8           | MyBatis 3.5.x 및 mybatis-spring 2.1.x 계열 검토 가능 |
| Spring 5.3      | mybatis-spring 2.1.x 계열 필요 |
| eGovFrame 4.3   | `org.egovframe.rte.psl.dataaccess`와 설정 정합성 확인 필요 |
| DBMS            | SQL 실행 결과 및 TypeHandler 검증 필요 |
| iBATIS 잔존 영역 | MyBatis 통합 전환 기준 필요 |

## 9. 최종 판정

### 판정

```text
전환대상여부 : 대상 / 버전업

조치방안 :
- mybatis-3.2.7.jar를 MyBatis 3.5.x 계열로 버전업
- mybatis-spring-2.1.x와 세트로 업그레이드
- eGovFrame 4.3의 org.egovframe.rte.psl.dataaccess 설정과 정합성 확인
- iBATIS 잔존 영역은 MyBatis 기준으로 통합 전환
- Mapper XML, DAO, 트랜잭션, SQL 실행 결과 회귀 테스트 수행

자동변환 :
- 부분 가능
- 의존성 변경, 사용처 목록화, 설정 파일 탐지는 자동화 가능
- SQL 결과 검증과 DAO 구조 정리는 수동 검토 필요

소스수정 :
- 대부분 없음 또는 일부 수정 필요
- iBATIS 통합 전환 범위가 포함되면 일부~대규모 수정 가능

위험도 :
- 중간
```

### 비고

* 48번 `mybatis-3.2.7.jar`는 단독으로 판단하지 않고 49번 `mybatis-spring-1.2.2.jar`와 세트로 검토해야 한다.
* 26번 `ibatis-sqlmap-2.3.4.726.jar`가 잔존하는 경우 MyBatis 3.5.x 기준으로 통합 전환하는 것이 바람직하다.
* MyBatis Mapper XML은 업무 SQL을 직접 포함하므로 단순 컴파일 성공만으로 전환 완료로 판단하면 안 된다.
* SQL 실행 결과, 건수, 정렬, 페이징, NULL 처리, 날짜/숫자 타입 처리에 대한 회귀 테스트가 필수이다.
* `${}` 사용 SQL은 SQL Injection 위험이 있으므로 전환 시 보안 점검 대상에 포함해야 한다.
* `Library Migration Matrix.md`에는 Git 병합 충돌 마커가 남아 있으므로 최종 산출물 반영 전 Matrix 정리가 필요하다.
