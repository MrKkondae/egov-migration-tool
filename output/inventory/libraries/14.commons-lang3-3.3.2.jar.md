# 2. 라이브러리 개요

## 주요 기능

* 문자열(String) 처리 유틸리티
* Object, Number, Boolean 유틸리티
* Reflection 지원
* Date/Time 보조 기능
* Random String 생성
* Equals, HashCode, ToString Builder 제공
* Enum 처리 지원

## 사용 목적

Java 표준 API에서 부족한 공통 기능을 제공하는 범용 유틸리티 라이브러리이다.

특히 eGovFrame 3.x 프로젝트에서는 다음과 같은 코드가 매우 자주 사용된다.

* StringUtils
* ArrayUtils
* ObjectUtils
* Validate
* RandomStringUtils

## 대표 사용 예시

```java
String name = StringUtils.defaultString(userNm);

if(StringUtils.isBlank(userNm)) {
    throw new IllegalArgumentException();
}

String tempPwd =
    RandomStringUtils.randomAlphanumeric(8);
```

# 4. eGovFrame 4.3 영향도 분석

## 전환 대상 여부

* 제외

## 전환 사유

Commons Lang는 eGovFrame 프레임워크 전용 라이브러리가 아니다.

전자정부프레임워크 4.3으로 전환하더라도 Commons Lang 사용 방식은 동일하다.

따라서 프레임워크 전환 대상이 아니라 일반 오픈소스 라이브러리 업그레이드 검토 대상으로 분류하는 것이 적절하다.

## 영향 범위

| 구분      | 영향          |
| ------- | ----------- |
| 소스      | 거의 없음       |
| 설정파일    | 없음          |
| pom.xml | 버전 업그레이드 검토 |
| 운영환경    | 없음          |

# 6. 버전 업그레이드 검토

## AS-IS 버전

```text
commons-lang3-3.3.2
(2014년 출시)
```

## TO-BE 버전

```text
commons-lang3-17.0.0
또는
commons-lang3-18.x 최신 안정버전
```

※ 실제 적용 시 프로젝트 표준 버전에 맞춰 결정

## 주요 변경사항

### 성능 개선

* String 처리 성능 향상
* Reflection 처리 개선

### JDK 지원 확대

* JDK 8 ~ JDK 21+ 지원
* JDK 17 환경 최적화

### 신규 API 추가

* StringUtils 기능 확장
* ObjectUtils 개선
* Streams 지원 강화

### Deprecated API 증가

일부 구버전 API는 Deprecated 되었으나 대부분 하위 호환성을 유지한다.

## 소스 수정 여부

* 대부분 없음
* 일부 Deprecated API 사용 시 경고 발생 가능

# 7. 위험요소

## 보안

Commons Lang 3.3.2 자체에 심각한 보안 이슈는 알려져 있지 않다.

다만 2014년 버전으로 상당히 오래되었으므로 최신 버전 사용이 권장된다.

| eGovFrame 4.3 | 문제 없음 |

## 비고

* Commons Lang는 eGovFrame 3.x → 4.3 전환 과정에서 직접적인 변경 대상이 아니다.
* 실제 프로젝트에서 `StringUtils`, `ObjectUtils`, `RandomStringUtils` 사용 여부를 확인하는 정도면 충분하다.
* 라이브러리 매트릭스 기준 분류는 **"전환 제외 / 업그레이드 권장"** 으로 관리하는 것을 권장한다.
* 우선순위는 매우 낮으며, DAO/iBatis/MyBatis/Spring/eGovFrame 관련 라이브러리보다 후순위 검토 대상이다.
