## 2. 라이브러리 개요

### 주요 기능

* JavaMail 연계 지원
* Quartz Scheduler 연계 지원
* JMX 지원
* Cache 추상화 지원
* Task Scheduling 지원

### 사용 목적

`spring-context-support`는 Spring Context의 확장 기능을 제공하는 라이브러리이다.

일반적인 Spring Bean 관리 기능은 `spring-context`가 담당하며, `spring-context-support`는 메일 발송, 스케줄링, 캐시, JMX 등의 부가 기능을 지원한다.

전자정부 프로젝트에서는 다음과 같은 경우에 사용된다.

* 메일 발송 기능
* Quartz 스케줄러 연동
* 배치 프로그램
* 캐시 기능
* 시스템 관리 기능

### 대표 사용 예시

```java id="xt8t4q"
JavaMailSenderImpl mailSender =
    new JavaMailSenderImpl();

mailSender.send(message);
```

또는

```xml id="jy4tva"
<bean id="scheduler"
      class="org.springframework.scheduling.quartz.SchedulerFactoryBean">
</bean>
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-context-support-3.2.9.RELEASE.jar`는 Spring Framework 3.2 계열 라이브러리로 eGovFrame 4.3의 Spring Framework 5.3.37 환경과 버전이 맞지 않는다.

또한 `spring-context-support`는 `spring-context`, `spring-core`, `spring-beans`, `spring-expression` 등과 동일한 Spring Framework 버전으로 유지되어야 하므로 함께 업그레이드가 필요하다.

특히 Quartz 연계 기능이나 JavaMail 기능을 사용하는 경우 버전 변경에 따른 영향도 검토가 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 중간 |
| 설정파일    | 중간 |
| pom.xml | 높음 |
| 운영환경    | 중간 |

### 비고

* eGovFrame 4.3 전환 시 Spring Framework 버전 정합성을 위해 반드시 함께 업그레이드해야 하는 라이브러리이다.
* 실제 프로젝트에서 `JavaMailSender`, `SchedulerFactoryBean`, `CacheManager` 등을 사용하지 않는 경우 영향도는 낮아질 수 있다.

* 메일 발송 기능, 배치 프로그램, 스케줄러 기능에 대한 회귀 테스트를 반드시 수행해야 한다.
* `spring-context-support`는 핵심 컨테이너 라이브러리는 아니지만 부가 기능(메일, 스케줄링, 캐시) 제공을 위해 운영 영향도가 존재한다.

##### (16) 최종 정리

```text
spring-context-support 3.2.9.RELEASE → 5.3.37 전환은 Spring Context 확장 기능의 버전 정합성 확보가 핵심이다.
```
