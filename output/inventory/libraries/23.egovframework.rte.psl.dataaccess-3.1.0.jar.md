# 2. 라이브러리 개요

## 주요 기능

* DAO 추상 클래스 제공
* iBatis / MyBatis 데이터 접근 지원
* SQL Mapper 연동
* CRUD 공통 API 제공
* Mapper 기반 DAO 구현 지원

## 사용 목적

전자정부프레임워크 3.x 환경에서 데이터베이스 접근 계층(DAO)을 구현하기 위한 핵심 라이브러리이다.

주요 제공 클래스는 다음과 같다.

```java
egovframework.rte.psl.dataaccess.EgovAbstractDAO
egovframework.rte.psl.dataaccess.EgovAbstractMapper
```

프로젝트 대부분의 DAO가 해당 라이브러리를 기반으로 구현되어 있다.

## 대표 사용 예시

### EgovAbstractDAO 방식

```java
@Repository
public class UserDAO extends EgovAbstractDAO {

    public List<UserVO> selectUserList(UserVO vo) {
        return (List<UserVO>) list("UserDAO.selectUserList", vo);
    }
}
```

### EgovAbstractMapper 방식

```java
@Repository
public class UserDAO extends EgovAbstractMapper {

    public List<UserVO> selectUserList(UserVO vo) {
        return selectList("UserDAO.selectUserList", vo);
    }
}
```

# 4. eGovFrame 4.3 영향도 분석

## 전환 대상 여부

* 대상

## 전환 사유

eGovFrame 4.3에서는 패키지명이 변경되었다.

### AS-IS

```java
import egovframework.rte.psl.dataaccess.EgovAbstractMapper;
import egovframework.rte.psl.dataaccess.EgovAbstractDAO;
```

### TO-BE

```java
import org.egovframe.rte.psl.dataaccess.EgovAbstractMapper;
import org.egovframe.rte.psl.dataaccess.EgovAbstractDAO;
```

또한 프로젝트에 따라 EgovAbstractDAO 기반 구조를 유지할지, EgovAbstractMapper 기반으로 전환할지 검토가 필요하다.

현재 사용자의 전환 정책상

* EgovComAbstractDAO 유지
* DAO 상속 구조 유지
* 자동 제거 금지

원칙을 적용해야 한다.

## 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 높음 |
| 설정파일    | 낮음 |
| pom.xml | 높음 |
| 운영환경    | 중간 |

# 5. 전환 시 검토사항

## 변경 필요 사항

### 1. Import 변경

```java
egovframework.rte.psl.dataaccess.*
```

↓

```java
org.egovframe.rte.psl.dataaccess.*
```

### 2. Maven Dependency 변경

AS-IS

```xml
<dependency>
    <groupId>egovframework.rte</groupId>
    <artifactId>egovframework.rte.psl.dataaccess</artifactId>
</dependency>
```

TO-BE

```xml
<dependency>
    <groupId>org.egovframe.rte</groupId>
    <artifactId>org.egovframe.rte.psl.dataaccess</artifactId>
    <version>${org.egovframe.rte.version}</version>
</dependency>
```

### 3. DAO 부모 클래스 검증

다음 항목 확인 필요

```java
extends EgovAbstractDAO
extends EgovComAbstractDAO
extends EgovAbstractMapper
```

특히 공통 DAO(EgovComAbstractDAO)가 존재하는 경우 영향도 분석 필요

### 4. SQL Mapping 검증

* iBatis 사용 여부
* MyBatis 사용 여부
* SqlMapClient 사용 여부

확인 필요

# 7. 위험요소

## 보안

직접적인 보안 취약점은 없으나 구버전 Spring 및 데이터 접근 라이브러리와 함께 사용되는 경우 취약점 전파 가능성이 존재한다.

## 운영

DAO 공통 상속 구조가 변경되면 전체 CRUD 동작에 영향이 발생할 수 있다.

특히 다음 메서드 사용 여부 확인 필요

```java
list()
selectByPk()
insert()
update()
delete()
```

## 호환성

### JDK 영향

* Java 8 이상 필요

### Spring 영향

* Spring 5.3.x 적용

### eGovFrame 영향

* 패키지 변경 필수
* pom.xml 변경 필수

## 비고

본 라이브러리는 전자정부프레임워크의 핵심 DAO 계층 라이브러리로, 프로젝트 내 DAO 대부분이 직접 또는 간접적으로 의존할 가능성이 높다.

특히 현재 프로젝트에서 확인된 다음 전환 규칙을 반드시 적용해야 한다.

```text
- EgovComAbstractDAO 자동 제거 금지
- DAO extends 구조 유지
- list()/insert()/update()/delete() 호출 유지
- EgovAbstractMapper 전환 시 수동 검증 수행
```

따라서 **전환 대상 라이브러리 중 최우선 검토 대상(Core Library)** 으로 분류한다.
