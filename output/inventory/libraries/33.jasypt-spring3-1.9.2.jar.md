## 2. 라이브러리 개요

### 주요 기능

* Jasypt 암호화 기능과 Spring Framework 설정 연계
* Spring XML 설정에서 암호화된 Property 값 복호화
* `EncryptablePropertyPlaceholderConfigurer` 기반 Property Placeholder 처리
* DB 접속정보, 외부 연계 비밀번호, API Key 등 민감 설정값 보호
* `jasypt-1.9.2.jar`의 암호화/복호화 기능을 Spring Bean으로 구성

### 사용 목적

`jasypt-spring3-1.9.2.jar`는 Jasypt의 암호화 기능을 Spring Framework 기반 애플리케이션에서 사용할 수 있도록 지원하는 연계 모듈이다.

eGovFrame 3.1 기반 시스템에서는 주로 `context-*.xml`, `egov-com-servlet.xml`, `globals.properties`, `jdbc.properties` 등에서 암호화된 설정값을 복호화하기 위해 사용되었을 가능성이 있다.

대표적으로 DB 접속 비밀번호를 다음과 같은 형태로 저장하고, Spring 기동 시 Jasypt Bean을 통해 복호화하는 방식이다.

```properties
Globals.DriverClassName=oracle.jdbc.OracleDriver
Globals.Url=jdbc:oracle:thin:@127.0.0.1:1521:ORCL
Globals.UserName=APPUSER
Globals.Password=ENC(encrypted-text)
```

본 라이브러리는 단독으로 동작하기보다 `jasypt-1.9.2.jar`와 함께 사용되므로 32번 항목과 세트로 검토해야 한다.

### 대표 사용 예시

```xml
<bean id="environmentVariablesConfiguration"
      class="org.jasypt.encryption.pbe.config.EnvironmentStringPBEConfig">
    <property name="algorithm" value="PBEWithMD5AndDES" />
    <property name="passwordEnvName" value="JASYPT_ENCRYPTION_PASSWORD" />
</bean>

<bean id="configurationEncryptor"
      class="org.jasypt.encryption.pbe.StandardPBEStringEncryptor">
    <property name="config" ref="environmentVariablesConfiguration" />
</bean>

<bean id="propertyConfigurer"
      class="org.jasypt.spring3.properties.EncryptablePropertyPlaceholderConfigurer">
    <constructor-arg ref="configurationEncryptor" />
    <property name="locations">
        <list>
            <value>classpath:/egovframework/egovProps/globals.properties</value>
        </list>
    </property>
</bean>
```

위 예시는 Spring XML 기반 설정에서 암호화된 Property 값을 복호화하는 구성이다. 실제 프로젝트에서는 클래스명, 설정파일 경로, 암호화 알고리즘, 키 관리 방식이 다를 수 있으므로 소스 및 설정 확인이 필요하다.

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`jasypt-spring3-1.9.2.jar`는 Spring 3 연계용으로 사용된 Jasypt 모듈이다.

eGovFrame 4.3은 Spring Framework 5.3.x 기반으로 전환되므로, Spring 3 전용 연계 모듈을 그대로 유지하는 것은 표준 전환 관점에서 적절하지 않다.

다만 Jasypt 1.9.x 라인에는 Spring 5 전용 모듈이 명확하게 제공되지 않으므로 단순 버전업보다는 다음 중 하나로 전환 방안을 결정해야 한다.

1. XML 기반 Spring MVC 환경 유지 시
   * `jasypt-spring31` 또는 `jasypt-spring4` 계열 호환성 검토
   * 기존 XML Bean 구성을 수동 Bean 설정 방식으로 정리
   * 암호화/복호화 결과가 기존과 동일한지 회귀 테스트 수행

2. Spring Boot 기반으로 전환하는 경우
   * `jasypt-spring-boot` 적용 검토
   * 단, Spring Boot 버전, Java 버전 요구사항 확인 필요

3. Jasypt 사용처가 없는 경우
   * `jasypt-spring3-1.9.2.jar` 제거
   * `jasypt-1.9.2.jar`도 함께 제거 가능 여부 검토

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 낮음 ~ 중간. 직접 Java 코드 사용은 적을 수 있으나 암호화 유틸 직접 호출 시 수정 필요 |
| 설정파일    | 높음. Spring XML의 Property Placeholder, Encryptor Bean, Property 파일 경로 변경 가능 |
| pom.xml | 중간. `jasypt-spring3` 제거 또는 `jasypt-spring31`/`jasypt-spring4`/대체 모듈 적용 필요 |
| 운영환경    | 높음. 암호화 키 환경변수, JVM 옵션, WAS 기동 스크립트, 배포환경별 키 관리 방식 확인 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `jasypt-spring3-1.9.2.jar` 직접 사용 여부 확인
* `jasypt-1.9.2.jar`와 세트 구성 여부 확인
* Spring XML 내 `EncryptablePropertyPlaceholderConfigurer` 사용 여부 확인
* 암호화된 Property 표기 방식 확인
  * `ENC(...)`
  * 커스텀 Prefix/Suffix
  * 자체 복호화 유틸 사용 여부
* 기존 암호화 알고리즘 확인
  * `PBEWithMD5AndDES`
  * `PBEWithMD5AndTripleDES`
  * 기타 커스텀 알고리즘
* 암호화 키 관리 방식 확인
  * 소스 하드코딩 여부
  * Property 파일 저장 여부
  * WAS JVM 옵션 사용 여부
  * OS 환경변수 사용 여부
  * 별도 보안 저장소 사용 여부
* 운영/개발/검증 환경별 키 분리 여부 확인
* 기존 암호문 복호화 가능 여부 테스트
* 신규 암호화 값 생성 방식 및 운영 반영 절차 정의

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 제한적 가능. Maven 의존성 변경과 일부 클래스 변경은 가능하나 XML 설정 의미 변환은 제한적 |
| Python Rule | 부분 가능. `grep`, XML 패턴 검색, `ENC(...)` 사용처 목록화, 의존성 변경 후보 산출 가능 |
| LLM 보정      | 필요. Spring XML Bean 구조, 키 관리 정책, 암호화 알고리즘 유지 여부 판단 필요 |

### 예상 난이도

* 중간

단순히 라이브러리만 존재하고 실제 사용처가 없다면 난이도는 낮다. 그러나 DB 접속정보 또는 외부 연계 비밀번호 복호화에 사용 중이면 운영 기동과 직접 연결되므로 난이도와 위험도가 상승한다.

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
org.jasypt:jasypt-spring3:1.9.2
org.jasypt:jasypt:1.9.2
```

### TO-BE 버전

```text
1차 검토 후보:
org.jasypt:jasypt:1.9.3
org.jasypt:jasypt-spring31:1.9.3 또는 org.jasypt:jasypt-spring4:1.9.3

Spring Boot 전환 시 별도 후보:
com.github.ulisesbocchio:jasypt-spring-boot
```

### 주요 변경사항

* `jasypt-spring3`는 Spring 3 연계 모듈이므로 Spring 5 기반 eGovFrame 4.3 환경에서 호환성 확인 필요
* `jasypt-spring31` 또는 `jasypt-spring4` 적용 시 XML Bean 클래스명 변경 가능
* 기존 암호화 알고리즘과 키를 그대로 유지하지 않으면 기존 암호문 복호화 실패 가능
* Spring Boot용 `jasypt-spring-boot`는 기존 XML 기반 Spring MVC 구조와 적용 방식이 다르므로 별도 전환 과제로 분리 필요
* 최신 Spring Boot용 Jasypt 모듈은 Java/Spring Boot 요구사항이 높을 수 있으므로 eGovFrame 4.3의 JDK 1.8 기반 환경에서는 무조건 최신 버전 적용 금지

### 소스 수정 여부

* 사용처 없음: 없음
* Spring XML Property 복호화만 사용: 설정파일 일부 수정 필요
* 업무 소스에서 Jasypt API 직접 호출: 일부 수정 필요
* Spring Boot 방식으로 전환: 설정 구조 변경 필요

## 7. 위험요소

### 보안

* 암호화 키가 소스, XML, Properties 파일에 평문으로 저장되어 있으면 보안 취약점이 될 수 있음
* 구버전 Jasypt 예제에서 자주 사용되는 `PBEWithMD5AndDES`는 현재 기준으로 강한 암호화 방식으로 보기 어려우므로 개선 검토 필요
* 알고리즘 변경 시 기존 암호문 복호화 불가 가능성 있음
* 운영환경별 키 분리 및 접근권한 관리 필요
* 암호화 키를 Git, 배포 패키지, 산출물에 포함하지 않도록 관리 필요

### 운영

* 설정값 복호화 실패 시 애플리케이션 기동 실패 가능
* DB 비밀번호 복호화 실패 시 DataSource 생성 실패 가능
* 운영/검증/개발 환경의 암호화 키가 다르면 동일 암호문 사용 불가 가능성 있음
* 기동 스크립트, WAS JVM 옵션, 환경변수 반영 누락 시 장애 발생 가능
* 암호화 값 재생성 절차가 없으면 운영 비밀번호 변경 시 대응 지연 가능

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 1.8         | eGovFrame 4.3 기준 충족 필요. 최신 Spring Boot용 Jasypt는 Java 요구사항 확인 필요 |
| Spring 5.3.x    | Spring 3 전용 모듈 유지 시 호환성 확인 필요 |
| eGovFrame 4.3   | XML 기반 설정 유지 여부에 따라 수동 Bean 설정 검토 필요 |
| WAS             | 환경변수/JVM 옵션 전달 방식 확인 필요 |
| 운영 배포       | 키 관리, 암호문 생성, 환경별 설정 분리 절차 필요 |

### 비고

* 본 항목은 32번 `jasypt-1.9.2.jar`와 반드시 함께 검토해야 한다.
* 단순히 `jasypt-spring3` JAR만 교체하면 되는 항목이 아니라, Spring 설정 구조와 운영 키 관리 방식까지 포함해 검토해야 한다.
* 암호화 알고리즘을 변경하면 기존 운영 암호문을 복호화하지 못할 수 있으므로 변경 전/후 비교 테스트가 필요하다.
* 운영 반영 전 개발/검증/운영 환경별로 동일한 암호문 복호화 테스트를 수행해야 한다.
* 암호화 키는 소스 저장소, 산출물, 배포 패키지에 포함하지 않는 것을 원칙으로 한다.
* Matrix 기준 TO-BE 후보에 `jasypt-spring31`이 명시되어 있으나, 실제 적용 시 Spring 5.3.x와의 호환성 및 XML 설정 동작 여부를 사업 환경에서 검증해야 한다.
