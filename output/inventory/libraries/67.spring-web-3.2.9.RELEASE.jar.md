## 2. 라이브러리 개요

### 주요 기능

* 웹 애플리케이션 공통 기능 제공
* Servlet 연계 지원
* Multipart File Upload 지원
* WebApplicationContext 제공
* REST 연계 지원
* HTTP 요청/응답 처리 지원

### 사용 목적

`spring-web`은 Spring Framework의 웹 계층 공통 기능을 제공하는 라이브러리이다.

Spring MVC의 핵심 기능은 `spring-webmvc`에서 제공하지만, 웹 환경 구동을 위한 기반 기능은 `spring-web`에서 제공한다.

전자정부 프로젝트에서는 다음과 같은 기능에서 사용된다.

* Controller 처리
* File Upload 처리
* Multipart Request 처리
* REST API 처리
* WebApplicationContext 초기화
* HTTP 요청 및 응답 처리

### 대표 사용 예시

#### MultipartFile 사용

```java id="5k3yxa"
public String upload(
    MultipartFile file
) {

    String fileName =
        file.getOriginalFilename();

    return fileName;
}
```

#### RestTemplate 사용

```java id="q6n7pd"
RestTemplate restTemplate =
    new RestTemplate();

String result =
    restTemplate.getForObject(
        url,
        String.class
    );
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-web-3.2.9.RELEASE.jar`는 Spring Framework 3.2 기반 웹 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 `spring-web` 역시 반드시 업그레이드해야 한다.

특히 `spring-web`은 다음 라이브러리와 밀접하게 연관된다.

* spring-webmvc
* spring-core
* spring-context
* spring-beans
* spring-expression

전자정부프레임워크 4.3 전환 시 웹 계층 전반에 영향을 주는 핵심 라이브러리이므로 영향도가 높다.

### 비고

* eGovFrame 4.3 전환 시 반드시 업그레이드해야 하는 Spring Framework 핵심 라이브러리이다.
* `spring-web`은 `spring-webmvc`의 기반 라이브러리 역할을 수행하므로 두 라이브러리는 함께 전환해야 한다.
* `MultipartFile`, `MultipartResolver`, `RestTemplate`, `HttpMessageConverter` 사용 여부를 우선 확인해야 한다.
* 파일 업로드, 다운로드, REST API 연계, 외부 시스템 인터페이스 기능은 집중 검증 대상이다.
* 전환 후 파일 업로드 기능, 인터페이스 연계 기능, REST 서비스 기능에 대한 회귀 테스트를 반드시 수행해야 한다.
