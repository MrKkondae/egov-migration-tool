## 2. 라이브러리 개요

### 주요 기능

* JDBC Template 제공
* DataSource 관리
* SQL 실행 지원
* ResultSet 매핑 지원
* SQLException 예외 변환 지원

### 사용 목적

`spring-jdbc`는 JDBC 프로그래밍을 단순화하기 위한 Spring Framework 라이브러리이다.

반복적인 JDBC 코드(Connection, Statement, ResultSet 처리)를 제거하고 예외 처리 및 자원 해제를 자동화한다.

전자정부 프로젝트에서는 다음과 같은 경우에 사용된다.

* JdbcTemplate 사용
* NamedParameterJdbcTemplate 사용
* 직접 SQL 처리
* Batch 처리
* Stored Procedure 호출

### 대표 사용 예시

```java id="r9w4kt"
@Autowired
private JdbcTemplate jdbcTemplate;

public List<UserVO> selectUserList() {

    return jdbcTemplate.query(
        "SELECT * FROM TB_USER",
        new UserRowMapper()
    );
}
```

또는

```java id="m7u4ha"
int count =
    jdbcTemplate.queryForObject(
        "SELECT COUNT(*) FROM TB_USER",
        Integer.class
    );
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-jdbc-3.2.9.RELEASE.jar`는 Spring Framework 3.2 계열 JDBC 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 Spring JDBC 라이브러리도 함께 업그레이드해야 한다.

특히 JDBC Template, Transaction, DataSource 설정은 Spring Core 계열과 밀접하게 연계되므로 버전 정합성 확보가 필요하다.

또한 프로젝트에서 iBATIS, MyBatis와 함께 사용 중인 경우 JDBC 관련 설정 검증이 필요하다.

### 영향 범위

| 소스      | 높음 |
| 설정파일    | 높음 |
| pom.xml | 높음 |
| 운영환경    | 중간 |

### 비고

* eGovFrame 4.3 전환 시 반드시 업그레이드해야 하는 Spring Framework 핵심 라이브러리이다.
* `JdbcTemplate`, `NamedParameterJdbcTemplate`, `DataSource` 관련 기능은 대부분 그대로 사용 가능하나 실제 SQL 실행 검증이 필요하다.
* 전환 후 CRUD 기능, 배치 기능, Stored Procedure 기능에 대한 회귀 테스트를 수행해야 한다.
* 프로젝트에서 iBATIS 또는 MyBatis를 사용하더라도 Spring JDBC 기반 DataSource 및 Transaction 관리 기능은 계속 사용될 가능성이 높다.
* `spring-jdbc`는 `spring-tx`와 밀접하게 연계되므로 두 라이브러리는 함께 전환해야 한다.
