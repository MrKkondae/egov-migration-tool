# 2. 라이브러리 개요

## 주요 기능

* Multipart/Form-Data 파일 업로드 처리
* 업로드 파일 정보 조회
* 업로드 파일 저장 및 스트림 처리
* Servlet 기반 파일 업로드 지원

## 사용 목적

웹 화면에서 사용자가 첨부파일을 업로드할 때 Multipart Request를 분석하고 파일 정보를 추출하기 위해 사용한다.

전자정부프레임워크 3.x 기반 시스템에서는 Spring MultipartResolver 또는 CommonsMultipartResolver 내부에서 사용되는 경우가 많다.

## 대표 사용 예시

```java
DiskFileItemFactory factory = new DiskFileItemFactory();

ServletFileUpload upload =
        new ServletFileUpload(factory);

List<FileItem> items =
        upload.parseRequest(request);

for (FileItem item : items) {
    if (!item.isFormField()) {
        String fileName = item.getName();
        item.write(new File("/upload/" + fileName));
    }
}
```

# 4. eGovFrame 4.3 영향도 분석

## 전환 대상 여부

**조건부 대상**

## 전환 사유

전자정부프레임워크 4.3은 Spring Framework 5.3 기반이며 Multipart 처리 기능을 자체 제공한다.

AS-IS에서 Commons FileUpload API를 직접 사용하는 경우에는 영향이 있다.

반면 Spring MultipartResolver를 통해 간접적으로만 사용하는 경우에는 별도 소스 수정이 거의 없을 수 있다.

## 영향 범위

| 구분      | 영향                 |
| ------- | ------------------ |
| 소스      | 직접 API 사용 시 영향     |
| 설정파일    | Multipart 설정 검토 필요 |
| pom.xml | 버전 업그레이드 검토        |
| 운영환경    | 업로드 제한 설정 검토       |

# 5. 전환 시 검토사항

## 변경 필요 사항

### 1. 직접 API 사용 여부 확인

아래 코드가 존재하면 전환 검토 필요

```java
ServletFileUpload
FileItem
DiskFileItemFactory
```

### 2. Spring MultipartResolver 사용 여부 확인

```xml
<bean id="multipartResolver"
      class="org.springframework.web.multipart.commons.CommonsMultipartResolver"/>
```

### 3. 업로드 제한값 검토

```xml
maxUploadSize
maxInMemorySize
```

설정이 4.3 환경에서도 동일하게 동작하는지 확인 필요

### 4. 보안 패치 적용 검토

1.3.1은 다수의 보안 취약점이 보고된 구버전이다.

# 7. 위험요소

## 보안

### 알려진 취약점 존재

Commons FileUpload 1.3.1은 다음 계열 취약점 영향을 받을 수 있다.

* Multipart Parsing DoS
* 대용량 업로드 처리 취약점
* Request Parsing 관련 취약점

따라서 운영환경에서는 상위 버전 사용 권장

## 운영

* 대용량 첨부파일 업로드 시 메모리 사용량 증가 가능
* 업로드 제한값 재검토 필요

## 호환성

| 항목            | 영향    |
| ------------- | ----- |
| JDK 17        | 영향 없음 |
| Spring 5.3    | 영향 없음 |
| eGovFrame 4.3 | 영향 없음 |
| Tomcat 9      | 영향 없음 |

## 비고

전자정부프레임워크 4.3 자체가 Commons FileUpload 제거를 요구하지는 않는다.

다만 현재 사용 중인 1.3.1 버전은 보안 취약점이 존재하는 구버전이므로 **AS-IS 프로젝트에서 실제 사용 여부를 먼저 확인한 후 업그레이드 또는 제거 여부를 결정하는 것이 바람직하다.**

특히 `ServletFileUpload`, `FileItem`, `DiskFileItemFactory` 사용 여부를 반드시 확인할 것을 권고한다.
