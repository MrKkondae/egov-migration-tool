## 2. 라이브러리 개요

### 주요 기능

* SMTP 기반 이메일 발송
* POP3 / IMAP 기반 이메일 수신
* MIME 메시지, 첨부파일, HTML 메일 처리
* 인증 기반 메일 서버 연계
* Spring `JavaMailSender` 또는 직접 JavaMail API 기반 메일 처리 지원

### 사용 목적

`mail-1.4.jar`는 Java 애플리케이션에서 메일 송수신 기능을 구현하기 위해 사용되는 JavaMail 계열 라이브러리이다.

업무 시스템에서는 주로 비밀번호 초기화, 알림 메일, 배치 결과 통보, 민원/업무 처리 안내, 첨부파일 발송 등에 사용될 수 있다.

전자정부프레임워크 4.3 기본 실행환경은 Spring Framework 5.3.x, JDK 1.8+, Servlet 3.1 기반이므로 `jakarta.mail`로 일괄 전환하기보다 `javax.mail` 계열의 JDK 8 호환 상위 버전 적용을 우선 검토하는 것이 적절하다.

### 대표 사용 예시

```java
import java.util.Properties;
import javax.mail.Message;
import javax.mail.MessagingException;
import javax.mail.Session;
import javax.mail.Transport;
import javax.mail.internet.InternetAddress;
import javax.mail.internet.MimeMessage;

public class MailSample {
    public void sendMail() throws MessagingException {
        Properties props = new Properties();
        props.put("mail.smtp.host", "smtp.example.com");
        props.put("mail.smtp.port", "25");

        Session session = Session.getInstance(props);
        Message message = new MimeMessage(session);
        message.setFrom(new InternetAddress("noreply@example.com"));
        message.setRecipients(Message.RecipientType.TO,
                InternetAddress.parse("user@example.com"));
        message.setSubject("메일 테스트");
        message.setText("메일 발송 테스트입니다.");

        Transport.send(message);
    }
}
```

Spring 연계 방식에서는 다음과 같이 사용될 수 있다.

```java
import org.springframework.mail.javamail.JavaMailSender;
import org.springframework.mail.SimpleMailMessage;

public class MailService {
    private JavaMailSender mailSender;

    public void send(String to, String subject, String text) {
        SimpleMailMessage message = new SimpleMailMessage();
        message.setTo(to);
        message.setSubject(subject);
        message.setText(text);
        mailSender.send(message);
    }
}
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상 / 버전업 / 검토

### 전환 사유

`mail-1.4.jar`는 오래된 JavaMail 계열 라이브러리이므로 eGovFrame 4.3의 JDK 1.8+ 및 Spring 5.3.x 기반 환경에서는 `javax.mail-1.6.x` 계열로 버전업하는 것이 적절하다.

다만 eGovFrame 4.3은 기본적으로 `javax.*` 기반 실행환경이므로 Tomcat 10, Servlet 5 이상, Spring 6 이상으로 전환하는 경우가 아니라면 `jakarta.mail` 전환은 별도 과제로 분리하는 것이 안전하다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 직접 `javax.mail.*` 사용 시 대부분 유지 가능. `jakarta.mail` 전환 시 import 변경 필요 |
| 설정파일    | SMTP Host, Port, 인증, TLS/SSL, Timeout 설정 확인 필요 |
| pom.xml | `mail-1.4.jar` 제거 후 `com.sun.mail:javax.mail:1.6.x` 또는 프로젝트 표준 의존성으로 변경 |
| 운영환경    | SMTP 서버, 방화벽, TLS 정책, 인증 계정, 운영 메일 릴레이 정책 영향 가능 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `mail-1.4.jar` 직접 배치 여부 확인
* `activation-1.1.jar` 등 Java Activation Framework 의존성 동반 여부 확인
* `javax.mail` 직접 사용 소스 확인
* Spring `JavaMailSenderImpl` 설정 확인
* SMTP 인증 방식 확인
* TLS/SSL 사용 여부 확인
* 메일 서버 인증서 및 암호화 프로토콜 확인
* 첨부파일, HTML 메일, 다국어 제목/본문 인코딩 테스트
* 운영/검증/개발 메일 서버 설정 분리 여부 확인
* 메일 발송 실패 시 재처리/로그 정책 확인

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 부분 가능 |
| Python Rule | 가능 |
| LLM 보정      | 필요 |

### 예상 난이도

* 중간

단순 `javax.mail` 1.4 → 1.6.x 버전업은 난이도가 낮은 편이나, 운영 SMTP 서버의 TLS 정책, 인증 방식, 첨부파일/HTML 메일 처리, 메일 릴레이 정책이 얽혀 있으면 테스트 난이도는 중간 수준으로 판단한다.

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
mail-1.4.jar
```

### TO-BE 버전

```text
com.sun.mail:javax.mail:1.6.2
또는 프로젝트 표준에서 검증한 javax.mail-1.6.x 계열
```

### 주요 변경사항

* JavaMail 1.4 계열에서 1.6.x 계열로 버전업
* SMTP/POP3/IMAP 구현체 최신화
* TLS/SSL 및 인증 방식 호환성 확인 필요
* Maven 의존성 기준으로 관리 가능
* eGovFrame 4.3 기본 환경에서는 `javax.mail` 패키지 유지 권장
* `jakarta.mail`은 Tomcat 10/Servlet 5 이상 또는 Spring 6 이상 전환 시 별도 검토

### 소스 수정 여부

* `javax.mail` 1.6.x 유지 시 대부분 없음
* Spring Mail 설정 사용 시 대부분 없음
* 오래된 API, 커스텀 인증, 직접 Provider 지정 사용 시 일부 수정 필요
* `jakarta.mail` 전환 시 import 및 의존성 변경 필요

## 7. 위험요소

### 보안

* SMTP 인증 정보가 설정파일에 평문 저장되어 있는지 확인 필요
* TLS/SSL 적용 여부 확인 필요
* 운영 SMTP 서버 인증서 신뢰 체인 확인 필요
* 메일 헤더 인젝션 가능성 확인 필요
* 첨부파일 전송 시 파일명/확장자/용량 검증 필요
* 외부 메일 릴레이 정책 및 스팸 발송 오탐 방지 필요

### 운영

* 메일 서버 Host/Port, 계정, 인증 방식 변경 시 운영 장애 가능
* 방화벽 및 폐쇄망 SMTP Relay 정책 확인 필요
* 대량 메일 발송 시 큐잉, Timeout, 재시도 정책 확인 필요
* 개발/검증 환경에서 실제 사용자에게 메일이 발송되지 않도록 수신자 제한 필요
* 메일 발송 실패 로그 및 업무 재처리 기준 확인 필요

### 호환성

| 항목              | 영향 |
| ----------------- | ---- |
| JDK 1.8           | `javax.mail 1.6.x` 계열 적용 가능 |
| Spring 5.3.x      | `JavaMailSender` 기반 구성 가능 |
| eGovFrame 4.3     | `javax.*` 기반 유지 검토 |
| Tomcat 9 이하     | `javax.mail` 유지 권장 |
| Tomcat 10 이상    | `jakarta.mail` 전환 별도 검토 |
| Servlet 3.1       | `javax.*` 기반 호환 |
