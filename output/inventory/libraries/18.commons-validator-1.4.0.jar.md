# 2. 라이브러리 개요

## 주요 기능

* 이메일 주소 형식 검증
* URL 형식 검증
* 날짜 형식 검증
* 숫자 형식 검증
* 정규식 기반 입력값 검증
* 주민번호, 우편번호 등 사용자 정의 검증 로직 지원

## 사용 목적

웹 애플리케이션에서 사용자 입력값의 형식 유효성을 검증하기 위해 사용된다.

전자정부프레임워크 3.x 기반 프로젝트에서는 다음과 같은 용도로 사용되는 경우가 많다.

* Spring Validator 보조
* JSP 입력값 검증
* 서버측 파라미터 검증
* 공통 유틸 클래스

## 대표 사용 예시

```java
import org.apache.commons.validator.routines.EmailValidator;

boolean valid =
    EmailValidator.getInstance().isValid("test@test.com");
```

```java
import org.apache.commons.validator.routines.UrlValidator;

UrlValidator validator = new UrlValidator();
boolean valid = validator.isValid("https://www.egovframe.go.kr");
```

# 4. eGovFrame 4.3 영향도 분석

## 전환 대상 여부

* 제외

## 전환 사유

Commons Validator는 전자정부프레임워크 전용 라이브러리가 아니다.

eGovFrame 4.3으로 전환하더라도 다음과 같은 이유로 그대로 사용 가능하다.

* Spring Framework 변경과 무관
* eGovFrame API 변경과 무관
* Java 17 환경에서도 동작
* pom.xml 의존성만 유지하면 사용 가능

따라서 프레임워크 전환 대상이 아니라 일반 오픈소스 라이브러리 업그레이드 검토 항목으로 분류하는 것이 적절하다.

## 영향 범위

| 구분      | 영향             |
| ------- | -------------- |
| 소스      | 없음             |
| 설정파일    | 없음             |
| pom.xml | 버전 업그레이드 검토 가능 |
| 운영환경    | 없음             |

# 7. 위험요소

## 보안

Commons Validator 1.4.0은 매우 오래된 버전이다.

직접적인 치명적 보안 이슈는 많지 않으나 다음 위험이 존재한다.

* 오래된 URL 검증 로직
* 최신 RFC 미반영
* 유지보수 종료 수준의 버전

## 운영

* 신규 브라우저 환경 검증 부족
* 최신 국제 도메인(IDN) 처리 제한 가능성

## 호환성

| 항목            | 영향    |
| ------------- | ----- |
| JDK 17        | 문제 없음 |
| Spring 5.3    | 문제 없음 |
| eGovFrame 4.3 | 문제 없음 |
