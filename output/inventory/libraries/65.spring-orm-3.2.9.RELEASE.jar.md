## 2. 라이브러리 개요

### 주요 기능

* ORM 프레임워크 연계 지원
* Hibernate 연동
* JPA 연동
* iBATIS 연동
* MyBatis-Spring 연동 지원
* ORM 트랜잭션 관리 지원

### 사용 목적

`spring-orm`은 Spring Framework와 ORM(Object Relational Mapping) 프레임워크를 연계하기 위한 라이브러리이다.

전자정부프레임워크 3.x 환경에서는 다음과 같은 구조에서 사용된다.

* Hibernate + Spring
* JPA + Spring
* iBATIS + Spring
* MyBatis + Spring
* LocalSessionFactoryBean 사용
* SqlMapClientFactoryBean 사용

특히 eGovFrame 3.x 계열의 iBATIS 기반 프로젝트에서는 `spring-orm`을 통해 SQL Mapper와 Spring을 연계하는 경우가 많다.

### 대표 사용 예시

#### iBATIS 연계

```xml id="b7y8sv"
<bean id="sqlMapClient"
      class="org.springframework.orm.ibatis.SqlMapClientFactoryBean">
</bean>
```

#### Hibernate 연계

```xml id="eh4mnz"

      class="org.springframework.orm.hibernate3.LocalSessionFactoryBean">
</bean>
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-orm-3.2.9.RELEASE.jar`는 Spring Framework 3.2 기반 ORM 연계 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반이므로 Spring ORM 라이브러리도 함께 업그레이드가 필요하다.

특히 전자정부프레임워크 전환 시 가장 중요한 검토 대상 중 하나는 다음과 같다.

* iBATIS 사용 여부
* MyBatis 사용 여부
* Hibernate 사용 여부
* JPA 사용 여부

만약 iBATIS를 사용 중인 경우에는 eGovFrame 4.3 전환 과정에서 MyBatis 전환 검토가 필요할 수 있으며, `spring-orm` 설정 역시 영향을 받을 수 있다.

### 영향 범위

| 구분      | 영향 |

| 설정파일    | 높음 |

* CRUD 기능, 트랜잭션 처리, SQL Mapper 동작에 대한 회귀 테스트를 반드시 수행해야 한다.

##### (16) 최종 정리

```text
spring-orm 3.2.9.RELEASE → 5.3.37 전환은 Spring ORM 라이브러리 버전 정합성뿐 아니라 실제 ORM 기술 사용 여부 확인이 핵심이다.

JPA 표준 API(EntityManager, @PersistenceContext)를 사용하는 경우 Java 소스 수정은 대부분 없으며, JpaTransactionManager 설정도 대부분 유지 가능하다.

반면 Hibernate 3.x 연계, iBATIS SqlMapClientFactoryBean, SqlMapClientTemplate을 사용하는 경우에는 단순 버전업이 아니라 ORM 구조 변경 또는 MyBatis 전환 검토가 필요하다.

OpenRewrite를 통해 Maven 의존성 변경, 구버전 Jar 탐지, Spring Framework 계열 라이브러리 버전 통일, XML Schema Location 정리는 자동화할 수 있다.

다만 Hibernate 설정, iBATIS → MyBatis 전환, SQL Map XML, DAO 호출 방식, Transaction 동작 검증은 수동 검토 대상으로 관리해야 한다.
```
