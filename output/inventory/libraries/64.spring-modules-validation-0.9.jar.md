## 2. 라이브러리 개요

### 주요 기능

* Validator 제공
* Commons Validator 연계
* Spring Form Validation 지원
* 입력값 검증
* 메시지 기반 Validation 처리

### 사용 목적

`spring-modules-validation`은 Spring Modules 프로젝트에서 제공하던 Validation 라이브러리이다.

과거 전자정부프레임워크 2.x ~ 3.x 초기 버전에서는 화면 입력값 검증을 위해 많이 사용되었으며, Apache Commons Validator와 연계하여 서버 측 Validation 기능을 제공하였다.

전자정부 프로젝트에서는 다음과 같은 용도로 사용된다.

* 화면 입력값 검증
* Form Validation
* 필수값 검증
* 길이 검증
* 패턴 검증
* 사용자 입력 데이터 검증

### 대표 사용 예시

```java id="8kz5ar"
public class UserValidator
    extends DefaultBeanValidator {

}
```

또는

```xml id="j98vuy"
<bean id="beanValidator"
      class="org.springmodules.validation.commons.DefaultBeanValidator">
</bean>
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-modules-validation-0.9.jar`는 현재 개발 및 유지보수가 사실상 종료된 라이브러리이다.

전자정부프레임워크 4.3에서는 해당 라이브러리를 표준으로 사용하지 않으며, 일반적으로 다음 방식으로 전환한다.

* Bean Validation(JSR-303)
* Hibernate Validator
* Spring Validator

특히 전자정부프레임워크 4.3 환경에서는 기존 `DefaultBeanValidator` 기반 구조를 유지하기 어려운 경우가 많아 검토가 필요하다.

```java
DefaultBeanValidator
```

```xml
<bean id="beanValidator"
```

```java
beanValidator.validate(...)
```

* 사용하지 않는 경우에는 제거 가능하지만, 사용 중인 경우에는 화면 입력 검증 전체에 영향을 줄 수 있으므로 영향도가 높은 라이브러리로 분류한다.

##### (16) 최종 정리

```text
spring-modules-validation 0.9 → Bean Validation / Hibernate Validator 전환은 단순 버전업이 아니라 검증 구조 변경에 해당한다.

기존 DefaultBeanValidator, validator-rules.xml, validation.xml 기반 검증 방식은 Hibernate Validator 기반의 Annotation 검증 방식으로 전환해야 한다.

OpenRewrite는 spring-modules-validation 의존성 제거, Hibernate Validator 의존성 추가,
DefaultBeanValidator 및 beanValidator.validate() 사용 위치 탐지에는 효과적이다.

다만 XML 검증 규칙을 VO Annotation으로 변환하는 작업,
검증 메시지 정비, Controller의 @Valid / BindingResult 적용,
화면 오류 표시 검증은 수동 보정 대상으로 관리해야 한다.

전환 후에는 등록/수정 화면의 필수값, 길이, 형식, 숫자 범위 검증과
오류 메시지 출력 여부를 반드시 검증해야 한다.
```
