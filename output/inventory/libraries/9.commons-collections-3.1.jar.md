## 2. 라이브러리 개요

Apache Commons Collections는 Java Collections Framework를 확장하는 유틸리티 라이브러리이다.

Java 표준 Collection API(List, Map, Set 등)에 부족한 기능을 제공하며 과거 전자정부프레임워크 2.x~3.x 프로젝트에서 광범위하게 사용되었다.

대표 기능은 다음과 같다.

* Collection Utility
* Predicate
* Transformer
* Factory
* BidiMap
* MultiMap
* OrderedMap
* Collection Decorator
* Lazy Collection

예시

```java
CollectionUtils.isEmpty(list);
```

```java
MapUtils.getString(map, "userId");
```

```java
Predicate predicate = EqualPredicate.getInstance("ADMIN");
```

## 3. 전자정부프레임워크 사용 현황

전자정부프레임워크 3.x 환경에서는 다음과 같은 라이브러리들이 Commons Collections에 의존하였다.

* Spring Framework 3.x
* Apache Commons BeanUtils
* Apache Commons Digester
* 일부 Hibernate 버전
* 전자정부프레임워크 공통 컴포넌트

따라서 직접 사용하지 않더라도 Transitive Dependency로 포함되는 경우가 많다.

## 5. 전환 대상 여부

### CASE 1 : 직접 사용하지 않는 경우

Maven Dependency Tree 또는 라이브러리 의존성으로만 포함된 경우

전환 대상 아님

조치

```text
제거 가능
```

또는

```text
상위 라이브러리가 관리하도록 유지
```

---

### CASE 2 : CollectionUtils / MapUtils 만 사용하는 경우

대부분 Java 표준 API 또는 Apache Commons Collections 4.x로 대체 가능

예시

AS-IS

```java
CollectionUtils.isEmpty(list);
```

TO-BE

```java
CollectionUtils.isEmpty(list);
```

패키지 변경

```java
org.apache.commons.collections
```

↓

```java
org.apache.commons.collections4
```

---

### CASE 3 : Predicate / Transformer 사용

Commons Collections 4.x에서는 Generic 기반으로 변경되었다.

AS-IS

```java
Predicate predicate;
Transformer transformer;
```

TO-BE

```java
Predicate<T>
Transformer<I,O>
```

소스 수정 필요 가능성 있음

---

### CASE 4 : LazyMap / MultiMap 사용

Collections 4.x에서 API 변경이 존재한다.

수동 검토 필요

## 6. 보안 취약점 검토

Commons Collections 3.x는 Java 생태계에서 가장 유명한 취약점 중 하나인

### CommonsCollections Gadget Chain

과 관련되어 있다.

대표 취약점

* CVE-2015-4852
* CVE-2015-7501

해당 취약점은 Java 역직렬화 공격(Remote Code Execution)에 활용되었다.

특히 다음 클래스들이 문제 대상이다.

```java
InvokerTransformer
```

```java
ChainedTransformer
```

```java
LazyMap
```

```java
TransformedMap
```

보안 정책상 신규 시스템에서는 Commons Collections 3.x 사용을 권장하지 않는다.

## 9. 최종 결론

**commons-collections-3.1.jar는 eGovFrame 4.3 전환 시 필수 전환 대상 라이브러리는 아니다.**

다만 다음 사항을 반드시 확인해야 한다.

* 직접 사용 여부
* Commons Collections 3.x 의존 라이브러리 존재 여부
* CollectionUtils, MapUtils 사용 여부
* LazyMap, Transformer 등 취약 API 사용 여부

직접 사용이 없다면 제거를 권장하며, 사용 중인 경우에는 Commons Collections 4.x 또는 Java 표준 Collection API로 전환하는 것이 바람직하다.
