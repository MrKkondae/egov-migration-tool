## 2. 라이브러리 개요

### 주요 기능

* ApplicationContext 제공
* Bean 관리 및 의존성 주입
* Component Scan 지원
* 이벤트(Event) 처리
* MessageSource 및 Resource 관리

### 사용 목적

`spring-context`는 Spring Framework의 핵심 컨테이너 라이브러리로 ApplicationContext를 제공하며 Bean 생성, 의존성 주입, 이벤트 처리, 리소스 관리 등의 기능을 담당한다.

전자정부프레임워크 기반 시스템에서는 대부분의 Spring Bean 관리가 `spring-context`를 중심으로 수행된다.

### 대표 사용 예시

```java id="tqz8g3"
ApplicationContext context =
    new ClassPathXmlApplicationContext(
        "egovframework/spring/context-common.xml"
    );

UserService service =
    context.getBean(UserService.class);
```

또는

```xml id="d6rq1z"
<context:component-scan
    base-package="egovframework" />

<context:annotation-config />
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`spring-context-3.2.9.RELEASE.jar`는 Spring Framework 3.2 계열의 핵심 라이브러리이다.

eGovFrame 4.3은 Spring Framework 5.3.37 기반으로 구성되므로 Spring Core 계열 라이브러리 전체 업그레이드가 필요하다.

특히 `spring-context`는 `spring-core`, `spring-beans`, `spring-expression`, `spring-aop`, `spring-tx`와 긴밀하게 연계되어 동작하므로 단독 교체는 불가능하다.

ApplicationContext 초기화, Bean 등록, Component Scan, Event 처리 기능에 직접적인 영향을 미치므로 전환 시 영향도가 매우 높다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 높음 |
| 설정파일    | 높음 |
| pom.xml | 높음 |
| 운영환경    | 중간 |

### 비고

* eGovFrame 4.3 전환 시 반드시 업그레이드해야 하는 핵심 라이브러리이다.
* `spring-context`는 Spring Container의 중심 모듈로 시스템 전체에 영향을 준다.
* `context:component-scan`, `context:annotation-config`, `ApplicationContext` 관련 설정은 대부분 유지 가능하나 실제 동작 검증이 필요하다.
* `spring-context` 단독 전환이 아닌 `spring-core`, `spring-beans`, `spring-expression`, `spring-aop`, `spring-tx`와 함께 일괄 전환해야 한다.
* 전환 후 ApplicationContext 기동 테스트, 주요 업무 기능 테스트, 트랜잭션 및 DI 동작 검증을 반드시 수행해야 한다.
