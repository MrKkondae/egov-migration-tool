## 2. 라이브러리 개요

### 주요 기능

* ANTLR v3 Parser 실행 Runtime 제공
* Lexer 및 Parser 실행 지원
* Token Stream 관리
* Grammar 기반 언어 해석 지원

### 사용 목적

ANTLR(ANother Tool for Language Recognition)은 문법(Grammar)을 기반으로 Parser를 생성하는 도구이다.

ANTLR Tool을 이용하여 생성된 Parser는 실행 시 해당 버전의 Runtime 라이브러리를 필요로 한다.

주요 활용 분야는 다음과 같다.

* 컴파일러
* DSL(Domain Specific Language)
* SQL Parser
* Expression Parser
* Query 변환기
* 코드 생성기

### 대표 사용 예시

```java
import org.antlr.runtime.*;

ANTLRStringStream input =
    new ANTLRStringStream(sqlText);

MyLexer lexer =
    new MyLexer(input);

CommonTokenStream tokens =
    new CommonTokenStream(lexer);

MyParser parser =
    new MyParser(tokens);
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 원칙적으로 제외
* 단, 직접 사용 여부 확인 필요

### 전환 사유

ANTLR Runtime 3.5는 전자정부프레임워크, Spring Framework, MyBatis와 직접적인 연관이 없는 독립 라이브러리이다.

전자정부프레임워크 3.x → 4.3 전환 시 자동 변경 대상이 아니며 별도 전환 규칙도 존재하지 않는다.

다만 프로젝트에서 직접 사용하는 경우에는 유지 여부를 검토해야 한다.

### 영향 범위

| 구분      | 영향         |
| ------- | ---------- |
| 소스      | 직접 사용 시 영향 |
| 설정파일    | 없음         |
| pom.xml | 없음         |
| 운영환경    | 없음         |

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
org.antlr:antlr-runtime:3.5
```

### TO-BE 버전

```text
org.antlr:antlr-runtime:3.5 (유지 권고)
```

### 주요 변경사항

ANTLR 3 계열은 사실상 유지보수가 종료된 상태이다.

최신 버전은 다음과 같다.

```text
ANTLR 4.x
```

그러나 다음과 같은 단순 업그레이드는 불가능하다.

```text
ANTLR Runtime 3.5
→
ANTLR Runtime 4.x
```

ANTLR 4는 구조와 API가 완전히 변경되었기 때문에 다음 작업이 필요할 수 있다.

* Grammar 재작성
* Parser 재생성
* 애플리케이션 수정
* 테스트 재수행

### 소스 수정 여부

* ANTLR 4 전환 시 대규모 수정 필요
* eGovFrame 4.3 전환 범위에서는 수정 불필요

## 7. 위험요소

### 보안

* 현재 알려진 주요 보안 취약점 영향은 낮음
* 유지보수 종료(EOL) 상태로 장기적 관점에서는 리스크 존재

### 운영

* 직접 사용하는 경우 제거 시 Parser 기능 장애 발생 가능
* 사용 여부 확인 없이 삭제 금지

### 호환성

| 항목            | 영향          |
| ------------- | ----------- |
| JDK 8         | 지원          |
| JDK 17        | 일반적으로 동작 가능 |
| Spring 5.3    | 영향 없음       |
| MyBatis       | 영향 없음       |
| eGovFrame 4.3 | 영향 없음       |

### 비고

* ANTLR 2.7.7과는 별개의 라이브러리이다.
* ANTLR 2.7.7은 Hibernate 내부 전이 의존성으로 사용된다.
* ANTLR Runtime 3.5는 프로젝트에서 직접 사용할 가능성이 있다.
* 전자정부프레임워크 4.3 전환과 직접적인 관련은 없다.
* 직접 사용 여부를 확인한 후 유지 또는 제거를 결정하는 것이 바람직하다.
* Migration 분류는 **Investigate Only**로 판단한다.
