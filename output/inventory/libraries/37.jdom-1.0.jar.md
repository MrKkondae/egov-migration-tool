## 2. 라이브러리 개요

### 주요 기능

* XML 문서 파싱
* XML Element, Attribute, Namespace 처리
* XML 문서 생성 및 출력
* SAXBuilder, DOMBuilder 기반 XML 로딩
* XPath 기반 XML 데이터 조회

### 사용 목적

JDOM은 Java 코드에서 XML 문서를 쉽게 읽고, 생성하고, 조작하기 위한 XML 처리 라이브러리이다.

기존 `jdom-1.0.jar`는 JDOM 1.x 계열로, 오래된 XML 처리 라이브러리이다. eGovFrame 4.3 전환 시에는 JDOM2 계열인 `org.jdom:jdom2:2.x`로 대체 또는 버전업하는 것을 검토한다.

단, JDOM 1.x와 JDOM2는 패키지 구조가 다르다.

```text
AS-IS : org.jdom.*
TO-BE : org.jdom2.*
```

따라서 단순 JAR 교체만으로는 컴파일이 되지 않을 수 있으며, 프로젝트 내 직접 사용 소스 확인이 필요하다.

### 대표 사용 예시

AS-IS JDOM 1.x 사용 예시

```java
import java.io.File;

import org.jdom.Document;
import org.jdom.Element;
import org.jdom.input.SAXBuilder;

public class JdomSample {
    public void readXml() throws Exception {
        SAXBuilder builder = new SAXBuilder();
        Document document = builder.build(new File("sample.xml"));

        Element root = document.getRootElement();
        String name = root.getChildText("name");

        System.out.println(name);
    }
}
```

TO-BE JDOM2 사용 예시

```java
import java.io.File;

import org.jdom2.Document;
import org.jdom2.Element;
import org.jdom2.input.SAXBuilder;

public class Jdom2Sample {
    public void readXml() throws Exception {
        SAXBuilder builder = new SAXBuilder();
        Document document = builder.build(new File("sample.xml"));

        Element root = document.getRootElement();
        String name = root.getChildText("name");

        System.out.println(name);
    }
}
```

## 3. AS-IS 사용 현황

### 프로젝트 내 사용 여부

* 확인 필요

### 확인 방법

소스 import 확인

```bash
grep -R "import org.jdom" src
grep -R "org.jdom" src
```

JDOM 주요 클래스 사용 여부 확인

```bash
grep -R "SAXBuilder" src
grep -R "DOMBuilder" src
grep -R "XMLOutputter" src
grep -R "XPath" src
grep -R "JDOMException" src
```

XML 처리 설정 및 라이브러리 확인

```bash
find . -name "*.jar" | grep -i jdom
mvn dependency:tree -Dincludes=jdom:jdom
mvn dependency:tree -Dincludes=org.jdom:jdom
mvn dependency:tree -Dincludes=org.jdom:jdom2
```

### 예상 사용 위치

* XML 전문 처리 유틸리티
* 외부 기관 연계 전문 생성/파싱
* 배치 프로그램
* 설정 XML 파싱 로직
* 코드성 데이터 XML 처리
* 리포트/문서 생성 전처리
* 기타 XML 기반 인터페이스 처리

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

`jdom-1.0.jar`는 오래된 JDOM 1.x 계열 라이브러리이며, eGovFrame 4.3 전환 시 JDK 8 이상 환경과 최신 XML 처리 라이브러리 구성 기준에 맞춰 JDOM2 계열로 대체하는 것이 적절하다.

JDOM 1.x에서 JDOM2로 전환하면 Maven 좌표와 패키지가 변경된다.

```text
AS-IS : jdom:jdom:1.0
TO-BE : org.jdom:jdom2:2.x
```

특히 패키지가 `org.jdom`에서 `org.jdom2`로 변경되므로, 업무 소스에서 직접 import하고 있다면 소스 수정이 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 있음. `org.jdom.*` import 및 일부 API 변경 가능 |
| 설정파일    | 낮음. XML 파서 보안 설정, XPath 사용 설정 확인 필요 |
| pom.xml | 있음. `jdom:jdom` 제거 후 `org.jdom:jdom2` 추가 |
| 운영환경    | 낮음~중간. XML 파싱 결과, 문자셋, 외부 엔티티 처리 방식 검증 필요 |

---

## 5. 전환 시 검토사항

### 변경 필요 사항

* `jdom-1.0.jar` 직접 배치 여부 확인
* Maven 또는 WEB-INF/lib 내 중복 JDOM 확인
* `org.jdom.*` import를 `org.jdom2.*`로 변경
* `SAXBuilder`, `Document`, `Element`, `Attribute`, `Namespace`, `XMLOutputter` 사용 로직 확인
* XPath 사용 시 JDOM2 XPath API 변경 여부 확인
* XML 외부 엔티티 처리, DTD 처리, XXE 방어 설정 확인
* XML 생성 결과의 태그 순서, 공백, 인코딩, 네임스페이스 회귀 테스트 수행
* 외부 기관 연계 XML 전문이 있는 경우 전문 비교 테스트 수행

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 부분 가능 |
| Python Rule | 가능 |
| LLM 보정      | 필요 |

### 예상 난이도

* 중간

사용처가 단순 XML 읽기/쓰기라면 난이도는 낮음~중간이다. 다만 XPath, Namespace, 외부 연계 전문, XML 출력 포맷까지 엄격히 맞춰야 하는 업무라면 중간 이상으로 판단한다.

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
jdom:jdom:1.0
```

또는 수동 JAR 배치

```text
jdom-1.0.jar
```

### TO-BE 버전

```text
org.jdom:jdom2:2.0.6.1
```

또는 프로젝트 표준 라이브러리 검증 결과에 따른 JDOM2 2.x 계열 적용

### 주요 변경사항

* Maven 좌표 변경
* 패키지 변경

```text
org.jdom.*
→
org.jdom2.*
```

* XPath API 사용 방식 변경 가능
* XML 파서 보안 설정 확인 필요
* JDK 8 이상 환경에서 JDOM2 적용 가능
* 기존 JDOM 1.x와 JDOM2를 동시에 배치할 경우 혼동 가능성이 있으므로 단일화 필요

### 소스 수정 여부

* 일부 수정 필요

단순 사용처는 import 변경 중심으로 수정 가능하나, XPath·Namespace·출력 포맷·외부 연계 전문 처리 로직은 추가 수정 및 테스트가 필요할 수 있다.

## 7. 위험요소

### 보안

* XML 외부 엔티티 처리로 인한 XXE 취약점 가능성 확인 필요
* 외부 XML 파일 또는 외부 기관 전문을 파싱하는 경우 DTD, 외부 Entity, Schema 참조 차단 설정 확인 필요
* 라이브러리 자체보다 XML 파서 설정 방식에 따른 보안 영향이 큼

### 운영

* XML 파싱 결과가 기존과 달라질 가능성
* XML 출력 시 공백, 줄바꿈, 인코딩, 네임스페이스 선언 위치가 달라질 가능성
* 외부 기관 연계 XML 전문은 바이트 단위 비교가 필요한 경우가 있음
* 수동 JAR 배치와 Maven 의존성이 혼재되면 동일 클래스 중복 및 ClassNotFoundException 가능성 있음

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 8           | JDOM2 2.x 적용 가능 |
| Spring 5.3      | 직접 영향 낮음 |
| eGovFrame 4.3   | 직접 영향 낮음 |
| XML Parser      | SAX/DOM 파서 설정 확인 필요 |
| 기존 업무 소스  | `org.jdom.*` 직접 사용 시 수정 필요 |

### 비고

* Matrix 기준 37번 항목은 `jdom-1.0.jar`이며 XML 처리 라이브러리이다.
* TO-BE 후보는 `org.jdom:jdom2:2.x`이다.
