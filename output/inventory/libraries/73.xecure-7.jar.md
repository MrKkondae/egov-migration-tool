## 2. 라이브러리 개요

### 주요 기능

* 구간 암호화 처리
* 인증서 기반 암호화/복호화
* 전자서명 연계
* 보안 키 생성 및 검증
* 보안 솔루션 서버 연계

### 사용 목적

`xecure-7.jar`는 일반적으로 XecureWeb, XecureObject, XecureCrypto 등 Xecure 계열 보안 솔루션과 연계하기 위한 Java 라이브러리로 사용된다.

전자정부 및 공공기관 시스템에서는 다음과 같은 용도로 사용되는 경우가 많다.

* 로그인 구간 암호화
* 주민등록번호 등 개인정보 암호화
* 인증서 기반 전자서명
* 보안 모듈 연계
* 키보드보안 또는 구간암호화 솔루션 연동

### 대표 사용 예시

```java
Xecure xecure = new Xecure();

String encData = xecure.encrypt(plainText);

String decData = xecure.decrypt(encData);
```

또는

```java
String signedData =
    XecureSign.sign(data);
```

※ 실제 API 및 패키지명은 제품 버전과 설치 모듈에 따라 다를 수 있음

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 검토 대상

### 전환 사유

`xecure-7.jar`는 전자정부프레임워크 표준 라이브러리가 아니라 상용 보안 솔루션 연계 라이브러리이다.

따라서 eGovFrame 3.x → 4.3 전환 자체의 직접 전환 대상은 아니다.

다만 보안 라이브러리는 JDK 버전, WAS 환경, 브라우저 보안 정책, 인증서 정책, 암호화 알고리즘 지원 여부에 영향을 크게 받는다.

특히 기존 시스템에서 ActiveX, NPAPI, 구형 보안 플러그인, 인증서 모듈과 연계되어 있다면 eGovFrame 4.3 전환과 별개로 보안 모듈 전환 또는 제거 검토가 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 중간 |
| 설정파일    | 중간 |
| pom.xml | 낮음 |
| 운영환경    | 높음 |

### 비고

* eGovFrame 4.3 전환 자체의 필수 라이브러리는 아니다.
* 다만 공공기관 시스템에서는 로그인, 인증, 개인정보 암호화와 연결될 가능성이 높아 운영 위험도가 높다.
* 사용자의 전환 범위 중 **암·복호화 플러그인 제거 및 Java API 전환**과 관련될 수 있으므로 별도 영향도 분석 대상이다.
* `xecure-7.jar`가 ActiveX 또는 클라이언트 보안 플러그인과 연계되어 있다면 단순 유지가 아니라 제거/대체 검토가 필요하다.
* 실제 소스에서 `Xecure`, `xecure`, `softforum`, `encrypt`, `decrypt`, `sign` 사용 여부를 반드시 확인해야 한다.
