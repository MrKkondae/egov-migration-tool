## 2. 라이브러리 개요

### 주요 기능

* AOP(Aspect Oriented Programming) 표준 인터페이스 제공
* Method Interceptor 인터페이스 정의
* Invocation Context 인터페이스 정의
* Spring AOP 및 Aspect Framework 연동 지원

### 사용 목적

AOP Alliance는 AOP 관련 표준 인터페이스만 제공하는 경량 라이브러리이다.

실제 AOP 기능을 구현하지 않으며, Framework가 구현해야 하는 인터페이스 규약만 정의한다.

주요 인터페이스는 다음과 같다.

* MethodInterceptor
* MethodInvocation
* Interceptor
* Joinpoint

전자정부프레임워크 및 Spring Framework에서는 내부 AOP 처리 시 해당 인터페이스를 활용한다.

### 대표 사용 예시

```java
import org.aopalliance.intercept.MethodInterceptor;
import org.aopalliance.intercept.MethodInvocation;

public class LoggingInterceptor
        implements MethodInterceptor {

    @Override
    public Object invoke(
            MethodInvocation invocation)
            throws Throwable {

        System.out.println("Before");

        Object result = invocation.proceed();

        System.out.println("After");

        return result;
    }
}
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 제외

### 전환 사유

AOP Alliance는 전자정부프레임워크 전용 라이브러리가 아니며 Spring Framework 내부 AOP 처리에 사용되는 표준 인터페이스 라이브러리이다.

전자정부프레임워크 4.3이 사용하는 Spring Framework 5.3 환경에서도 동일하게 사용된다.

따라서 eGovFrame 3.x → 4.3 전환 과정에서 별도 변경이 필요하지 않다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 없음 |
| 설정파일    | 없음 |
| pom.xml | 없음 |
| 운영환경    | 없음 |
