## 2. 라이브러리 개요

### 주요 기능

* Property 파일(.properties) 통합 관리
* XML 기반 Property 설정 관리
* 환경별 설정값 조회
* Spring Bean 기반 PropertyService 제공
* 시스템 공통 설정정보 관리

### 사용 목적

전자정부프레임워크 3.x에서 애플리케이션 설정정보를 중앙에서 관리하기 위한 라이브러리이다.

일반적으로 다음과 같은 설정값 관리에 사용된다.

* 파일 업로드 경로
* 배치 실행 경로
* 외부 시스템 URL
* 암호화 설정
* 시스템 환경설정

### 대표 사용 예시

```java
@Resource(name="propertiesService")
protected EgovPropertyService propertiesService;

String uploadPath =
    propertiesService.getString("Globals.fileStorePath");

int pageUnit =
    propertiesService.getInt("pageUnit");
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상

### 전환 사유

전자정부프레임워크 4.3에서도 동일한 기능이 제공된다.

다만 패키지명이 변경되므로 import 변경이 필요하다.

#### AS-IS

```java
import egovframework.rte.fdl.property.EgovPropertyService;
```

#### TO-BE

```java
import org.egovframe.rte.fdl.property.EgovPropertyService;
```

### 영향 범위

| 구분      | 영향        |
| ------- | --------- |
| 소스      | import 변경 |
| 설정파일    | 일부 검토     |
| pom.xml | 변경 필요     |
| 운영환경    | 영향 없음     |

---

## 5. 전환 시 검토사항

### 변경 필요 사항

#### 1) Import 패키지 변경

```java
egovframework.rte.fdl.property
    ↓
org.egovframe.rte.fdl.property
```

#### 2) Maven Dependency 변경

AS-IS

```xml
<dependency>
    <groupId>egovframework.rte</groupId>
    <artifactId>egovframework.rte.fdl.property</artifactId>
    <version>3.1.0</version>
</dependency>
```

TO-BE

```xml
<dependency>
    <groupId>org.egovframe.rte</groupId>
    <artifactId>org.egovframe.rte.fdl.property</artifactId>
    <version>4.3.0</version>
</dependency>
```

#### 3) Property 설정 파일 검토

일반적으로 다음 설정은 그대로 유지된다.

```properties
Globals.fileStorePath=
Globals.pageUnit=
Globals.pageSize=
```

별도 소스 수정은 거의 발생하지 않는다.

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | ----- |
| OpenRewrite | 가능    |
| Python Rule | 가능    |
| LLM 보정      | 불필요   |

### 예상 난이도

* 낮음
