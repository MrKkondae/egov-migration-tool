# 2. 라이브러리 개요

## 주요 기능

* 객체 Pool 관리
* Connection Pool 구현 기반 제공
* GenericObjectPool 제공
* KeyedObjectPool 제공
* 객체 재사용을 통한 성능 향상

## 사용 목적

Commons Pool은 생성 비용이 큰 객체를 재사용하기 위한 Object Pool 라이브러리이다.

주로 다음과 같은 라이브러리의 내부 의존성으로 사용된다.

* Commons DBCP
* Jedis (Redis Client)
* LDAP Connection Pool
* FTP Client Pool

전자정부프레임워크 3.x 환경에서는 대부분 Commons DBCP 1.x가 내부적으로 Commons Pool 1.x를 사용한다.

## 대표 사용 예시

```java
GenericObjectPool pool = new GenericObjectPool();

Object obj = pool.borrowObject();

try {
    // 업무 처리
} finally {
    pool.returnObject(obj);
}
```

# 4. eGovFrame 4.3 영향도 분석

## 전환 대상 여부

* 조건부 대상

## 전환 사유

Commons Pool 자체는 전자정부프레임워크와 직접적인 연관은 없다.

그러나 현재 프로젝트에서 확인된

* commons-dbcp-1.4.jar
* commons-pool-1.5.4.jar

조합은 매우 일반적인 구성이다.

eGovFrame 4.3는 기본적으로 DBCP 1.x를 사용하지 않으며 Spring 5.x 환경에서는 HikariCP 사용이 권장된다.

따라서 DBCP 제거 시 Commons Pool도 함께 제거될 가능성이 높다.

## 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 낮음 |
| 설정파일    | 중간 |
| pom.xml | 높음 |
| 운영환경    | 중간 |

# 5. 전환 시 검토사항

## 변경 필요 사항

### 경우 1 : DBCP 제거

현재

```text
commons-dbcp-1.4
    └─ commons-pool-1.5.4
```

구조라면

```text
HikariCP
```

로 전환하면서 Commons Pool 제거 가능

### 경우 2 : 직접 사용 중

다음 코드가 존재하면 검토 필요

```java
GenericObjectPool
ObjectPool
PoolableObjectFactory
```

Commons Pool 2.x API로 변경 필요

### 경우 3 : 외부 라이브러리 사용

Jedis

LDAP

FTP Pool

등이 사용 중이면 버전 호환성 확인 필요

## 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | ----- |
| OpenRewrite | 가능    |
| Python Rule | 가능    |
| LLM 보정      | 필요    |

## 예상 난이도

* 직접 사용 없음 → 낮음
* 직접 사용 있음 → 중간

# 7. 위험요소

## 보안

Commons Pool 1.x 자체의 치명적 CVE는 많지 않으나 다음 문제가 존재한다.

* 장기간 유지보수 종료(EOL)
* 최신 JDK 환경 검증 부족
* Pool 고갈 관련 버그 수정 미반영

## 운영

Connection Pool 설정이 잘못될 경우

* Connection 누수
* Pool Exhaustion
* 응답 지연

발생 가능

## 호환성

| 항목            | 영향                 |
| ------------- | ------------------ |
| JDK 17        | 직접 사용 시 검토 필요      |
| Spring 5.3    | 영향 없음              |
| eGovFrame 4.3 | DBCP 제거 시 함께 제거 가능 |

## 비고

현재 프로젝트에서 이미 확인된 `commons-dbcp-1.4.jar`와 함께 사용되는 경우가 가장 유력하다.

따라서 본 라이브러리는 독립적으로 전환을 검토하기보다 **commons-dbcp-1.4.jar 분석 결과와 연계하여 판단하는 것이 적절**하다.

일반적인 eGovFrame 3.x → 4.3 전환 프로젝트에서는

```text
commons-dbcp-1.4
commons-pool-1.5.4
        ↓
HikariCP
```

구조로 변경되면서 함께 제거되는 사례가 가장 많다.
