## 2. 라이브러리 개요

### 주요 기능

* Java Bean Property 접근
* 객체 간 속성(Property) 복사
* Reflection 기반 필드 처리
* Request Parameter 바인딩

### 사용 목적

BeanUtils는 Java Bean 객체의 Property를 동적으로 처리하기 위한 라이브러리이다.

주로 다음과 같은 작업에 사용된다.

* VO → VO 복사
* DTO → VO 복사
* Request Parameter → VO 바인딩
* Reflection 기반 Property 접근

과거 전자정부프레임워크 2.x~3.x, Spring 2.x/3.x, Struts 기반 시스템에서 널리 사용되었다.

### 대표 사용 예시

```java
BeanUtils.copyProperties(targetVo, sourceVo);

String userNm = BeanUtils.getProperty(user, "userNm");

BeanUtils.setProperty(user, "userNm", "홍길동");
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 제외

### 전환 사유

전자정부프레임워크 4.3은 Apache Commons BeanUtils에 의존하지 않는다.

또한 eGovFrame 3.x → 4.3 전환 과정에서 BeanUtils 관련 API 변경이나 프레임워크 전환 규칙은 존재하지 않는다.

따라서 본 라이브러리는 전환 대상 라이브러리로 분류하지 않는다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 없음 |
| 설정파일    | 없음 |
| pom.xml | 없음 |
| 운영환경    | 없음 |

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text

```

또는

```text

```

### TO-BE 버전

```text
commons-beanutils 1.9.4 이상 권장
```

### 주요 변경사항

* BeanUtils 1.8.3 → 1.9.x 업그레이드 시 주요 API 유지
* 보안 취약점 개선
* JDK 호환성 개선
* Reflection 처리 안정성 향상

대표 API는 대부분 동일하게 사용 가능하다.

```java
copyProperties()
getProperty()
setProperty()
populate()
```

### 소스 수정 여부

* 일반적으로 없음
* 특수 Reflection 처리 로직은 검증 필요

## 7. 위험요소

### 보안

BeanUtils 1.8.3은 오래된 버전으로 다음과 같은 취약점 이력이 존재한다.

```text
CVE-2014-0114
```

ClassLoader 조작과 관련된 보안 취약점으로 알려져 있으며 운영 환경에서는 최신 버전 사용이 권장된다.

### 운영

* 사용 여부 확인 없이 제거 금지
* Reflection 기반 처리 로직 검증 필요

### 호환성

| 항목            | 영향       |
| ------------- | -------- |
| JDK 8         | 지원       |
| JDK 17        | 일반적으로 동작 |
| Spring 5.3    | 영향 없음    |
| eGovFrame 4.3 | 영향 없음    |
