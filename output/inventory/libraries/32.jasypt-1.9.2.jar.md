## 2. 라이브러리 개요

### 주요 기능

* 문자열 암호화 및 복호화
* Property 파일 내 민감정보 암호화
* PBE(Password Based Encryption) 기반 암호화 처리
* 암호화 알고리즘, 암호화 키, 반복 횟수, Salt Generator 설정 지원
* Spring 설정과 연계한 DB 접속정보, 외부 연계 비밀번호 등 민감정보 보호

### 사용 목적

Jasypt는 Java 애플리케이션에서 비교적 간단한 방식으로 암호화 기능을 적용하기 위한 라이브러리이다.

전자정부프레임워크 3.1 기반 시스템에서는 주로 다음과 같은 목적으로 사용될 가능성이 있다.

* `context-*.xml`, `globals.properties`, `jdbc.properties` 등 설정파일의 DB 비밀번호 암호화
* 외부 연계 시스템 비밀번호 암호화
* 업무 데이터 중 일부 문자열 필드 암호화
* 배치 또는 운영 스크립트에서 암호화된 설정값 복호화

본 항목은 `jasypt-spring3-1.9.2.jar`와 함께 사용되는 경우가 많으므로, 33번 항목과 연계 검토가 필요하다.

### 대표 사용 예시

```java
import org.jasypt.encryption.pbe.StandardPBEStringEncryptor;

public class JasyptSample {
    public static void main(String[] args) {
        StandardPBEStringEncryptor encryptor = new StandardPBEStringEncryptor();
        encryptor.setPassword("encryptionKey");
        encryptor.setAlgorithm("PBEWithMD5AndDES");

        String encrypted = encryptor.encrypt("plainPassword");
        String decrypted = encryptor.decrypt(encrypted);

        System.out.println(encrypted);
        System.out.println(decrypted);
    }
}
```

실제 운영환경에서는 암호화 키를 소스 또는 설정파일에 평문으로 저장하지 않고, 환경변수, WAS 시스템 프로퍼티, 별도 보안 저장소 등으로 분리 관리해야 한다.

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 대상
* 버전업 / 검토

### 전환 사유

`jasypt-1.9.2.jar`는 민감정보 암호화와 복호화에 사용될 수 있는 보안성 라이브러리이다. 전자정부프레임워크 4.3 전환 시 Spring Framework 5.3.x 및 JDK 1.8 이상 환경으로 변경되므로 기존 Jasypt 설정이 정상 동작하는지 확인해야 한다.

Matrix 기준 TO-BE 후보는 `jasypt-1.9.3` 또는 호환 설정 유지이다. `jasypt-1.9.3`은 Jasypt 1.x 계열의 후속 버전이며, 기존 1.9.2와 API 호환성이 비교적 높아 우선 검토 대상이다.

다만 암호화 라이브러리는 단순 버전업보다 기존 암호문 복호화 호환성이 더 중요하다. 알고리즘, Salt Generator, Provider, 키 관리 방식, 출력 인코딩, 암호화 키 전달 방식이 달라지면 기존 운영 암호문을 복호화하지 못할 수 있다.

### 영향 범위

| 구분      | 영향 |
| ------- | ---- |
| 소스      | 직접 암복호화 API 사용 시 영향 가능. `StandardPBEStringEncryptor`, `PooledPBEStringEncryptor`, `StringEncryptor` 사용처 확인 필요 |
| 설정파일    | 영향 있음. Spring XML, Property Placeholder, `ENC(...)` 값, 암호화 키 주입 방식 확인 필요 |
| pom.xml | 영향 있음. `jasypt-1.9.2`를 `jasypt-1.9.3`으로 버전업하거나 기존 호환 설정 유지 |
| 운영환경    | 영향 있음. 암호화 키를 전달하는 JVM 옵션, 환경변수, WAS 설정, 배포 스크립트 확인 필요 |

## 5. 전환 시 검토사항

### 변경 필요 사항

* `jasypt-1.9.2.jar` 직접 사용 여부 확인
* `jasypt-spring3-1.9.2.jar`와의 연계 여부 확인
* 기존 암호화 알고리즘 확인
  * 예: `PBEWithMD5AndDES`
  * 예: `PBEWithMD5AndTripleDES`
  * 예: `PBEWithHMACSHA512AndAES_256`
* 기존 암호문 복호화 테스트 수행
* 암호화 키 저장 위치 확인
  * 소스코드 하드코딩 여부
  * Property 파일 평문 저장 여부
  * JVM 옵션 또는 환경변수 사용 여부
  * WAS 보안 설정 사용 여부
* 운영/개발/검증 환경별 키 관리 방식 확인
* Java Cryptography Extension 정책 영향 확인
* FIPS, 국정원 검증필 암호모듈 등 별도 보안 요구사항 존재 여부 확인
* 신규 암호화 방식 적용 시 기존 암호문 재암호화 계획 수립

### 자동 변환 가능 여부

| 구분          | 가능 여부 |
| ----------- | -------- |
| OpenRewrite | 제한적 가능 |
| Python Rule | 가능 |
| LLM 보정      | 필요 |

### 자동화 적용 의견

* 사용처 검색, 의존성 버전 변경, `ENC(...)` 사용 현황 추출은 자동화 가능
* 암호화 알고리즘 변경, 키 관리 방식 변경, 운영환경 변수 전환은 자동화만으로 처리하기 어려움
* 기존 암호문 복호화 검증은 반드시 실제 환경값 또는 검증용 샘플 암호문으로 수행 필요

### 예상 난이도

* 중간

단순히 `1.9.2`에서 `1.9.3`으로 올리는 작업은 난이도가 높지 않지만, 운영 비밀번호·연계 비밀번호·암호화된 업무 데이터가 포함될 경우 장애 영향도가 커지므로 검증 난이도는 중간 이상으로 본다.

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
org.jasypt:jasypt:1.9.2
```

### TO-BE 버전

```text
org.jasypt:jasypt:1.9.3
```

또는 기존 암호화 설정과의 호환성 문제가 있을 경우 다음과 같이 판단한다.

```text
호환 설정 유지 + 사용처 및 보안 검토
```

### 주요 변경사항

* Jasypt 1.x 계열 내 최신 안정 버전 후보로 `1.9.3` 검토
* 기존 API 호환성은 비교적 높으나, 암호화 결과 호환성은 설정값에 따라 달라질 수 있음
* Spring 연계 설정은 33번 `jasypt-spring3-1.9.2.jar` 항목과 함께 검토 필요
* Spring Boot 전환이 아닌 XML 기반 Spring MVC 구조라면 `jasypt-spring-boot`를 무조건 적용하지 않음

### 소스 수정 여부

* 없음 ~ 일부 수정 필요

다음 조건에서는 소스 수정이 발생할 수 있다.

* 암복호화 클래스를 업무 소스에서 직접 사용하는 경우
* 암호화 알고리즘 또는 Provider를 변경하는 경우
* 암호화 키 주입 방식을 변경하는 경우
* Spring XML 기반 Property Placeholder 설정을 변경하는 경우
* `jasypt-spring-boot` 방식으로 전환하는 경우

## 7. 위험요소

### 보안

* 암호화 키가 소스, Property 파일, 배포 스크립트에 평문으로 저장되어 있을 수 있음
* 구식 알고리즘 사용 가능성 있음
  * `PBEWithMD5AndDES` 등은 현재 보안 기준에서 취약하거나 부적절할 수 있음
* 기존 암호문과 신규 암호화 방식의 호환성 문제 발생 가능
* 운영·검증·개발 환경의 암호화 키가 다르면 배포 후 복호화 오류 발생 가능
* 암호화 알고리즘 변경 시 기존 데이터 재암호화 또는 이중 복호화 전략 필요

### 운영

* DB 비밀번호 복호화 실패 시 애플리케이션 기동 실패 가능
* 외부 연계 비밀번호 복호화 실패 시 연계 업무 장애 가능
* 암호화 키 누락 시 운영 배포 직후 장애 발생 가능
* 운영자 매뉴얼, 배포 스크립트, WAS JVM 옵션과 함께 변경 관리 필요

### 호환성

| 항목            | 영향 |
| --------------- | ---- |
| JDK 1.8 이상     | 영향 가능. 암호화 알고리즘 및 JCE 정책 확인 필요 |
| Spring 5.3.x     | Spring 연계 모듈 설정 확인 필요 |
| eGovFrame 4.3    | 직접 제공 라이브러리라기보다 업무 보안 설정 영역으로 검토 필요 |
| WAS              | JVM 옵션, 시스템 프로퍼티, 환경변수 전달 방식 확인 필요 |
| 기존 암호문       | 반드시 복호화 호환성 테스트 필요 |

### 비고

* 본 라이브러리는 기능 전환보다 운영 보안 설정 검증이 중요하다.
* 기존 암호문을 복호화하지 못하면 DB 접속, 외부 연계, 배치 실행 등 핵심 업무 장애로 이어질 수 있다.
* 암호화 키는 소스 및 설정파일에 평문 저장하지 않는 것을 원칙으로 한다.
* Spring XML 기반 프로젝트에서는 `jasypt-spring-boot` 방식으로 단순 변경하지 말고, 기존 Spring 설정 구조에 맞는 수동 Bean 설정을 우선 검토한다.
* 33번 `jasypt-spring3-1.9.2.jar`와 함께 통합 검토해야 한다.
* 최종 판정은 실제 `ENC(...)` 사용 여부, 암호화 API 직접 사용 여부, 운영 키 관리 방식 확인 후 확정한다.
