## 2. 라이브러리 개요

### 주요 기능

* TSA(Time Stamping Authority) 서버 연계
* 전자문서 또는 전자서명 데이터의 시점확인 요청
* 타임스탬프 토큰(Time-Stamp Token) 발급 요청 및 응답 처리
* RFC 3161 기반 Timestamp Request / Timestamp Response 처리 가능성
* PDF 전자서명, 전자문서 진본확인, 부인방지 기능과 연계 가능성

### 사용 목적

`lib.magic.tsaclient-1.1.jar`는 파일명 기준으로 TSA 타임스탬프 서버와 연계하기 위한 클라이언트 모듈로 추정된다.

TSA는 전자문서나 전자서명 데이터가 특정 시점에 존재했음을 제3의 신뢰기관이 증명하기 위한 기능이다. RFC 3161은 Time Stamping Authority에 요청을 보내고 응답을 받는 형식을 정의하며, TSA는 데이터가 특정 시점에 존재했음을 나타내는 Time-Stamp Token을 생성한다.

공개 Maven 저장소에서 표준 오픈소스 라이브러리로 확인되는 항목은 아니므로, 일반 라이브러리 버전업이 아니라 공급사, 제품명, 라이선스, 운영 TSA 서버, 인증서, 네트워크 정책을 함께 확인해야 하는 검토 대상이다.

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 검토

### 전환 사유

본 라이브러리는 eGovFrame 표준 실행환경 라이브러리라기보다, 업무 보안 기능 또는 외부 TSA 서비스 연계를 위해 반입된 공급사 모듈로 추정된다.

전자정부프레임워크 4.3 전환 자체보다 JDK 8, WAS, 네트워크, 인증서, PDF 전자서명 모듈, TSA 서버와의 호환성 영향이 더 크다.

따라서 단순 버전업 또는 제거로 판단하면 안 되며, 다음 항목을 먼저 확인해야 한다.

* 실제 사용 업무
* 공급사 및 제품명
* 라이선스 및 유지보수 계약
* JDK 8/WAS 호환 버전 존재 여부
* TSA 서버 주소 및 통신 방식
* 인증서 저장소 및 정책 OID
* 운영망/폐쇄망 반입 가능 여부
* PDF 전자서명 모듈과의 연계 방식

### 영향 범위

| 구분      | 영향 |

| 소스      | 사용 API가 있으면 영향 있음. 공급사 API 변경 시 수정 필요 |

## 7. 위험요소

### 보안

* TSA는 전자문서 시점 증명과 연결되는 보안 기능이므로 영향도 높음
* 인증서 만료, 폐기, 신뢰체인 오류 발생 가능
* TSA 응답 검증 실패 시 전자문서 증빙 기능 실패 가능
* TLS 프로토콜, 암호화 알고리즘, 인증서 저장소 호환성 확인 필요
* 운영망 TSA 서버 접근 통제 및 방화벽 정책 확인 필요

### 운영

* TSA 서버 장애 또는 네트워크 지연 시 문서 발급/서명 업무 지연 가능
* 운영/검증 TSA 서버가 분리되어 있을 수 있음
* 폐쇄망 환경에서는 외부 TSA 서버 접근 방식 확인 필요
* 서버 시간 동기화, 로그 보관, 장애 재처리 기준 필요
* 라이선스 파일 또는 공급사 런타임 모듈 누락 시 운영 장애 가능

### 호환성

| 항목              | 영향 |
| --------------- | ---- |
| JDK 8           | 공급사 모듈 호환성 확인 필요 |
| Spring 5.3      | 직접 영향은 낮으나 Bean 등록 방식이 있으면 확인 필요 |
| eGovFrame 4.3   | 직접 표준 라이브러리는 아니며 업무 보안 기능으로 별도 검토 필요 |
| WAS             | 공통 lib 배치, ClassLoader, TLS 설정 영향 가능 |
| PDF 전자서명 모듈 | 41번 `lib.dstspdfSig-1.1.jar`와 연계 가능성 높음 |
| 네트워크          | TSA 서버 URL, 포트, 방화벽, 프록시 설정 영향 큼 |

### 비고

* 공개 Maven 저장소에서 `lib.magic.tsaclient`의 표준 오픈소스 좌표는 확인되지 않았다.
* 파일명과 공개 제품 정보 기준으로 DreamSecurity Magic TSA 계열 또는 유사 TSA 클라이언트 모듈 가능성이 있다.
* DreamSecurity의 Magic TSA는 타임스탬프 토큰을 통해 전자서명이나 전자적 행위의 시점을 증명하는 솔루션으로 소개된다.
* RFC 3161은 TSA에 보내는 요청과 TSA가 반환하는 응답 형식을 정의한다.
* 한국정보인증 등 국내 TSA 서비스도 RFC 3161 기반 타임스탬프 토큰 생성·발급·검증 기능을 제공하는 것으로 안내된다.
* 본 항목은 단순 라이브러리 버전업보다 업무 기능, 공급사, 인증서, TSA 서버, 운영 네트워크 검증이 핵심이다.
* 전자문서 증빙 또는 PDF 전자서명 업무와 연결되어 있을 경우 전환 난이도와 운영 위험도가 높다.
