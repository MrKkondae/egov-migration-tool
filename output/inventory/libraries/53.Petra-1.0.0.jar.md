## 2. 라이브러리 개요

### 주요 기능

* 데이터 암호화 및 복호화
* 개인정보 암호화 처리
* 대칭키 암호화 지원
* 문자열 암호화 API 제공
* 보안 모듈 연계 기능

### 사용 목적

업무 데이터 또는 개인정보에 대한 암호화 및 복호화 기능을 제공하기 위해 사용된다.

전자정부 프로젝트에서는 주민등록번호, 연락처, 계좌번호, 이메일 등의 개인정보 암호화 처리에 사용되는 경우가 많다.

※ Petra는 전자정부프레임워크 표준 라이브러리가 아니며, 기관 또는 솔루션 업체에서 제공하는 상용 암호화 모듈인 경우가 많다.

### 대표 사용 예시

```java
String encValue = Petra.encrypt(value);

String decValue = Petra.decrypt(encValue);
```

또는

```java
PetraManager.encrypt(data);
```

※ 실제 API는 제품 버전에 따라 다를 수 있음

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 검토 대상

### 전환 사유

Petra-1.0.0.jar는 전자정부프레임워크 Runtime 또는 Spring Framework 구성요소가 아니므로 eGovFrame 4.3 전환 시 직접적인 변경 대상은 아니다.

다만 암호화 라이브러리는 JDK 버전, WAS 버전, 보안 정책 및 암호화 알고리즘 지원 여부에 영향을 받을 수 있으므로 호환성 검토가 필요하다.

### 영향 범위

| 구분      | 영향 |
| ------- | -- |
| 소스      | 중간 |
| 설정파일    | 낮음 |
| pom.xml | 낮음 |
| 운영환경    | 높음 |

---
