## 2. 라이브러리 개요

### 주요 기능

* Dataset 생성 및 관리
* XML 기반 데이터 처리
* XPlatform 데이터 송수신
* Dataset 직렬화/역직렬화
* UI Framework 연계

### 사용 목적

`xdataset-1.0.1.jar`는 일반적으로 TOBESOFT의 XPlatform 또는 MiPlatform 계열 제품에서 사용하는 Dataset 처리 라이브러리이다.

주로 서버와 화면 간 Dataset 기반 데이터 교환을 처리하기 위해 사용된다.

전자정부 프로젝트에서는 다음과 같은 환경에서 발견되는 경우가 많다.

* XPlatform
* MiPlatform
* XInternet
* Legacy UI Framework
* 구형 업무시스템

### 대표 사용 예시

```java id="h4w8my"
Dataset ds =
    new Dataset("dsUser");

ds.addColumn(
    "USER_ID",
    DataTypes.STRING
);
```

또는

```java id="q8f1sx"
VariableList varList =
    platformData.getVariableList();

DatasetList dsList =
    platformData.getDatasetList();
```

## 4. eGovFrame 4.3 영향도 분석

### 전환 대상 여부

* 검토 대상

### 전환 사유

`xdataset-1.0.1.jar`는 전자정부프레임워크 라이브러리가 아니며 XPlatform/MiPlatform 전용 데이터 처리 라이브러리이다.

따라서 eGovFrame 3.x → 4.3 전환 자체와 직접적인 관련은 없다.

그러나 사용자의 프로젝트처럼 전자정부프레임워크 전환과 함께 화면(UI Framework) 전환을 검토하는 경우에는 매우 중요한 영향 분석 대상이 된다.

특히 다음 환경에서는 영향도가 높다.

* XPlatform 사용
* MiPlatform 사용
* XInternet 사용
* Dataset 기반 통신 사용

### 영향 범위

| 구분      | 영향    |
| ------- | ----- |
| 소스      | 매우 높음 |
| 설정파일    | 중간    |
| pom.xml | 낮음    |
| 운영환경    | 높음    |

### 비고

* eGovFrame 4.3 전환 자체보다는 **화면(UI Framework) 전환 여부가 핵심 판단 기준**이다.
* 사용자의 전환 프로젝트에서 자주 등장하는 **XPlatform → xFrame5**, **XPlatform → WebSquare** 전환 시 핵심 분석 대상 라이브러리이다.
* 만약 프로젝트가 JSP + Spring MVC 구조라면 영향이 거의 없을 수 있다.
* 반대로 XPlatform 기반 업무시스템이라면 Controller, Service, Dataset 처리 로직까지 영향을 받을 수 있다.
* 실제 분석 시 다음 클래스가 발견되면 우선 검토 대상으로 분류하는 것이 좋다.

```java id="t2h7wr"
PlatformData
Dataset
DatasetList
VariableList
HttpPlatformRequest
HttpPlatformResponse
```

* 사용자의 eGovFrame 3.x → 4.3 전환 프로젝트 기준으로는 **Spring Framework 전환 대상이 아니라 "UI Framework 연계 영향도 분석 대상"** 으로 분류하는 것이 가장 적절하다.
